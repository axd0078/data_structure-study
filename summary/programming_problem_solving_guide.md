# 程序设计题解题思路：按算法类型分类

本文根据 `programming` 目录中的题目类型，总结做程序设计题时的判断方法和常见解题套路。重点不是逐题背答案，而是看到题面后先判断“它像哪一类问题”，再选择合适的建模方式和维护信息。

## 总体做题流程

1. **读操作和目标**：先区分题目是在求最值、计数、可行性、路径、区间统计，还是模拟过程。
2. **找数据结构形状**：把题目对象转成数组、区间、字符串、矩阵、图、树或状态表。
3. **判断是否重复计算**：如果查询很多、每次现算很贵，就考虑预处理、维护统计量或使用树状数组、堆、索引表等结构。
4. **先写朴素思路再优化**：朴素解能帮助确认正确性，优化时通常围绕“减少枚举范围”或“把查询变快”展开。
5. **检查边界和不变量**：排序规则、下标起点、空输入、重复元素、区间闭合方式、树根和访问标记都要单独确认。

## 双指针与排序贪心

代表目录：`programming/double-pointer-and-greedy`

代表文件：`a+b+c=0.cpp`、`merge.cpp`、`meeting.cpp`、`rain.cpp`、`rain-plus.cpp`

### 适用信号

- 题目和数组、区间、左右边界、两端收缩有关。
- 需要找两个数、三个数、最大面积、合并区间或统计会议室数量。
- 排序后相邻关系、左右端点关系会变清楚。

### 建模方法

- 把原问题整理成有序数组或有序区间。
- 用两个指针维护当前候选范围，例如 `left/right` 或两个数组扫描指针。
- 对区间题，通常先按左端点或右端点排序，再维护当前合并区间或当前可用资源。

### 解题步骤

1. 先排序，让元素之间的大小关系固定下来。
2. 根据目标移动指针：和太小就左指针右移，和太大就右指针左移；区间重叠就合并，不重叠就开新区间。
3. 对接雨水、盛水这类题，维护左右两边的限制条件，移动较短或较弱的一侧。
4. 输出累计答案或最终合并后的结构。

### 易错点

- 三数之和要跳过重复元素，否则答案重复。
- 区间合并要明确端点相等时是否算重叠。
- 接雨水类题要区分“盛最多水的容器”和“接雨水”：前者求两条边围成的最大面积，后者求所有位置能接住的总水量。
- 双指针循环条件一般是 `left < right`，不要让两个指针越界或重复统计。

## 二分

代表目录：`programming/binary-search`

代表文件：`find-mid.cpp`、`lis-binary.cpp`、`minEatingSpeed.cpp`、`shipWithDays.cpp`

### 适用信号

- 题目要求在有序结构中找位置、找边界或找最小可行值。
- 答案本身有单调性：速度越大越容易完成，容量越大越容易装下。
- 暴力枚举答案可以做，但答案范围很大。

### 建模方法

- 普通二分：在有序数组或有序划分中查找目标位置。
- 二分答案：把“求最优值”转成“判断某个值是否可行”。
- LIS 二分优化：用 `tails[i]` 表示长度为 `i + 1` 的递增子序列的最小结尾值。

### 解题步骤

1. 明确二分对象是下标、数值答案，还是状态数组中的插入位置。
2. 写出 `check(mid)`：它只回答 `mid` 是否满足条件，不直接求最终答案。
3. 根据 `check(mid)` 的真假收缩边界。
4. 循环结束后返回边界值，并用小样例验证边界是否偏一位。

### 易错点

- 二分答案必须先证明单调性，否则不能二分。
- `left`、`right` 的含义要稳定：是闭区间、左闭右开，还是保留候选答案。
- `mid = left + (right - left) / 2` 可以避免溢出。
- `check` 中的计数要注意上取整，例如香蕉速度和运货天数问题都容易写错。

## 动态规划

代表目录：`programming/dynamic-programming`

代表文件：`coinChange.cpp`、`lis.cpp`、`eraseOverlapIntervals.cpp`、`lcis.cpp`、`maxSubArray.cpp`、`maxEnvelopes.cpp`、`numSquare.cpp`

### 适用信号

- 题目有“最少、最多、最长、最大和、方案数”等最优子结构。
- 当前选择会影响后续结果，但可以用一个状态概括前面的历史。
- 暴力递归会重复计算同一类子问题。

### 建模方法

- 先定义 `dp[i]` 或 `dp[i][j]` 的含义，必须能用一句话说清楚。
- 找状态转移：当前答案由哪些更小规模状态推出来。
- 明确初始化：空状态、第一项、不可达状态分别如何表示。
- 能一维化时再优化，不要一开始就把状态压缩到难以理解。

### 解题步骤

1. 写出状态定义，例如 `dp[i]` 表示到第 `i` 个元素为止的最优结果。
2. 写出转移方程，例如最大子数组和中 `dp[i] = max(nums[i], dp[i - 1] + nums[i])`。
3. 确定遍历顺序，保证转移时依赖的状态已经算出。
4. 根据题意决定答案是 `dp[n]`、某一行列，还是所有状态中的最大值。

### 易错点

- `LIS` 和 `LCIS` 不同：前者不要求连续，后者要求连续。
- 完全背包一维优化时，容量通常正序遍历；01 背包通常倒序遍历。
- 不可达状态要初始化成足够大的值，避免参与 `min` 时污染答案。
- 排序 + DP 的题要特别检查排序规则，例如俄罗斯套娃信封要避免同宽信封互相套。

## 图搜索

代表目录：`programming/graph-search`

代表文件：`dijkstra.cpp`、`numOfLand.cpp`、`numOfLand2.cpp`、`sizeOfLand.cpp`、`sonOfLand.cpp`、`numOfDiffLand.cpp`

### 适用信号

- 题目中有网格、连通块、岛屿、路径、最短距离、可达性。
- 一个位置会向上下左右或相邻节点扩展。
- 需要统计有多少块、面积多大、是否封闭、路径最短是多少。

### 建模方法

- 网格题把每个格子看成图上的一个点，上下左右是边。
- 一般图题用邻接表或邻接矩阵存边。
- 最短路题根据边权选择 BFS、Dijkstra 或 Floyd；当前目录中 `dijkstra.cpp` 使用邻接矩阵版 Dijkstra。

### 解题步骤

1. 建图或确认网格的移动方向。
2. 准备 `visited` 或直接修改原网格，避免重复访问。
3. 对每个未访问起点发起 DFS/BFS，并在搜索过程中统计数量、面积或形状。
4. 对最短路题，维护距离数组，每轮选择当前最近的未确定点并松弛邻居。

### 易错点

- 网格边界判断必须在访问数组或网格内容之前。
- DFS 淹没岛屿时，要确认修改的是当前题目允许破坏的数组。
- 封闭岛屿通常要先处理边界，再统计内部连通块。
- 递归 DFS 在数据很大时可能爆栈，必要时改成显式栈或 BFS。

## 贪心

代表目录：`programming/greedy`

代表文件：`arrayPairSum.cpp`、`candy.cpp`、`eraseOverlapIntervals.cpp`、`findMinArrowShots.cpp`、`increasingDigits.cpp`、`jump.cpp`、`jump2.cpp`、`maxProfit.cpp`、`maxSubArray.cpp`、`meeting.cpp`、`wiggleMaxLength.cpp`

### 适用信号

- 题目要求最少次数、最多保留、最大收益、最远可达。
- 每一步只需要维护当前最优信息，不需要保存完整历史。
- 排序后按某个准则选择，局部选择能推出全局最优。

### 建模方法

- 区间贪心：按右端点排序，优先保留结束早的区间。
- 跳跃问题：维护当前能到的最远位置，或当前层的覆盖边界。
- 股票问题：把总收益拆成每一段上升差值。
- 摆动序列：只保留趋势变化点。

### 解题步骤

1. 先找贪心指标：右端点、最远位置、当前收益、当前趋势。
2. 排序或线性扫描，让每一步选择都只依赖当前维护量。
3. 每次遇到更优选择就更新维护量，遇到冲突就按贪心规则舍弃。
4. 用反例检查贪心规则是否站得住，例如只按左端点排序是否会失败。

### 易错点

- 贪心题最容易选错排序规则，区间调度通常优先考虑右端点。
- `jump2.cpp` 的“最少跳数”要按层更新边界，不能每个位置都直接加步数。
- `candy.cpp` 这类左右约束题，单向扫描通常不够，要从两个方向修正。
- 数位贪心如 `increasingDigits.cpp`，修改前一位后，后缀通常要统一置成 `9`。

## 字符串索引

代表目录：`programming/string-index`

代表文件：`countMatchingSubsequences.cpp`

### 适用信号

- 需要反复判断多个单词是否是同一个长字符串的子序列。
- 对每个单词都从头扫描长字符串会重复计算。
- 查询次数多，适合提前建立索引。

### 建模方法

- 对长字符串预处理：记录每个字符出现的所有位置。
- 判断单词时，维护当前匹配到的位置 `pos`。
- 对每个字符，用 `upper_bound` 找到严格大于 `pos` 的下一个出现位置。

### 解题步骤

1. 建立字符到位置列表的倒排索引。
2. 遍历每个待匹配单词。
3. 对单词中的每个字符，在对应位置列表中二分查找下一个合法位置。
4. 如果所有字符都能找到，就计入答案。

### 易错点

- 子序列要求相对顺序，不要求连续。
- 每次二分要找的是大于当前 `pos` 的位置，不是大于等于。
- 字符不存在时要立刻判失败。
- `pos` 初始值可以设为 `-1`，方便匹配第一个字符。

## 数学与模拟

代表目录：`programming/math-and-simulation`

代表文件：`countPrimes.cpp`、`set-matrix-zeroes.cpp`

### 适用信号

- 题目规则明确，按步骤执行即可，但直接做可能重复或污染状态。
- 出现质数、倍数、矩阵标记、原地修改等关键词。
- 需要把数学性质转成循环边界和标记数组。

### 建模方法

- 筛法：用数组标记每个数是否可能为质数，从质数的倍数开始排除。
- 矩阵模拟：先记录哪些行列需要修改，再统一写回，避免边扫描边修改影响后续判断。

### 解题步骤

1. 先确定规则是否会互相影响。
2. 如果会影响，就分成“标记阶段”和“执行阶段”。
3. 数学题要尽量减少无意义枚举，例如筛法从 `i * i` 开始标记。
4. 最后统计或输出修改后的结果。

### 易错点

- `countPrimes.cpp` 通常统计的是小于 `n` 的质数，不包括 `n` 本身。
- `i * i` 可能溢出，范围变大时要用更大的整数类型。
- 矩阵置零不能一边发现 0 一边整行整列置零，否则新产生的 0 会干扰判断。
- 标记数组大小要和题目下标范围一致。

## 补充综合题

代表目录：`programming/test`

代表文件：`abbreviation-check.cpp`、`bestAttract.cpp`、`boss-time.cpp`、`deliver_takeout.cpp`、`greedy_multi_round_selection.cpp`、`safe-cell.cpp`、`stack_sequence_builder.cpp.cpp`、`nursing-center.cpp`、`nursing-center-plus.cpp`、`nursing-center-pro.cpp`

### 适用信号

- 题目通常不是单一模板，而是需要先把业务描述翻译成结构。
- 可能混合字符串解析、区间排序、图论、树上路径、栈队列模拟或动态统计。
- 输入输出细节和状态维护比算法名字更重要。

### 建模方法

- 区间时间题：把时间转成统一单位，再排序扫描。
- Floyd 题：把人物关系建成距离矩阵，求任意两点最短距离。
- 树上路径题：用父节点、深度、访问标记维护已经走过的路径。
- 栈队列模拟题：分别对应题目里的传送带、盒子、输出序列。
- 树上子树统计题：把管理结点建成树，老人当前直属结点作为动态位置。

### 解题步骤

1. 先提取实体和关系，例如人、时间段、管理结点、路径、队列元素。
2. 再确定操作会改变什么状态，查询需要什么结果。
3. 如果查询频繁，就维护统计量，避免每次从头扫描。
4. 对综合题保留一个清晰的不变量，例如“`belong[x]` 永远表示老人 `x` 当前直属管理结点”。

### 易错点

- 字符串编号和数字编号混在一起时，要先判断类型再转换。
- 时间区间要统一成秒或分钟，避免字符串直接比较造成错误。
- 栈队列模拟要严格按题目优先级执行，不能凭直觉调换步骤。
- 题目保证合法输入时可以简化判断；如果自己调试，要警惕不存在的 key 被 `map` 或 `unordered_map` 自动插入。

## 案例：树上子树统计与动态维护

代表文件：`programming/test/nursing-center.cpp`、`programming/test/nursing-center-plus.cpp`、`programming/test/nursing-center-pro.cpp`

### 题目建模

- 老人编号为 `1..N`。
- 管理结点使用字符串名称，需要映射成整数编号。
- 管理结点之间构成一棵树。
- 每位老人直属于某个管理结点。
- `T x S` 表示把老人 `x` 转移到管理结点 `S`。
- `Q S` 表示查询管理结点 `S` 的整棵子树中共有多少老人。

### 解法演进

1. **朴素做法**：每次查询遍历所有老人，从老人直属结点一路向上找祖先，判断是否经过目标结点。查询复杂度是 `O(N * 树高)`，修改只要改 `belong[x]`。
2. **维护子树计数**：维护 `cnt[u]` 表示结点 `u` 子树内老人数量。转移老人时，沿旧祖先链全部 `-1`，沿新祖先链全部 `+1`。查询变成 `O(1)`，修改是 `O(树高)`。
3. **DFS 序 + 树状数组**：把树拍平成 DFS 序，使结点 `u` 的整棵子树对应连续区间 `[in[u], out[u]]`。老人直属于结点 `u` 时，就在 `in[u]` 位置加 `1`。查询 `Q S` 就是求区间 `[in[S], out[S]]` 的和。

### 最终解题步骤

1. 用 `unordered_map<string, int>` 把管理结点名称映射成整数编号。
2. 读入初始关系，建立 `tree[u]` 儿子列表、`parent[u]` 父节点数组和 `belong[x]` 老人归属数组。
3. 找到 `parent[root] == 0` 的根结点，从根开始 DFS，求出每个结点的 `in[u]` 和 `out[u]`。
4. 遍历所有老人，在 `in[belong[i]]` 上用树状数组加 `1`。
5. 处理转移时，先在旧位置 `add(in[belong[x]], -1)`，再更新 `belong[x]`，最后在新位置 `add(in[belong[x]], 1)`。
6. 处理查询时，输出 `rangeSum(in[node], out[node])`。

### 调试经验

- 树状数组下标必须从 `1` 开始，`add(0, v)` 会死循环，因为 `0 += 0 & -0` 仍然是 `0`。
- `DFS` 完成后不要再用 `getId` 新建管理结点，否则新结点没有 `in/out` 区间。
- 查询时如果使用 `mp[s]` 访问不存在的字符串，会自动插入值 `0`，可能引出 `in[0]` 这类错误。
- 更稳妥的写法是用 `mp.find(s)` 判断是否存在；如果题目保证操作合法，也要保证输入阶段已经完整建树。
- 看到“超时”时不要只怀疑复杂度，也要检查是否有死循环、下标为 0、循环变量不变等问题。

## 复盘清单

- 这题的对象能不能转成数组、区间、图、树、字符串索引或状态表？
- 查询是否比修改更频繁？是否值得维护一个统计量？
- 排序规则是按左端点、右端点、数值大小，还是自定义优先级？
- 如果用了二分，`check` 是否真的单调？
- 如果用了 DFS/BFS，访问标记是否会漏掉或重复？
- 如果用了树状数组、前缀和、下标映射，是否保证下标从合法位置开始？
