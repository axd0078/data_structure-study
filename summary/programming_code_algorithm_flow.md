# programming 代码算法流程总结

本文按 `programming` 目录的现有分组，逐题整理源码中的算法流程。每个文件都从三件事看：先把题目对象建模成什么数据结构，再看输入输出怎样接住，最后看算法步骤和容易写错的细节。

## 双指针与排序贪心

### `double-pointer-and-greedy/a+b+c=0.cpp`

- **思路 / 数据结构设计**：用 `int nums[]` 存所有数，先排序；用 `Node { int a, b, c; }` 保存每组三元组答案。固定第一个数 `nums[i]`，剩下两个数用 `left/right` 在有序数组中夹逼。
- **输入输出处理**：C 风格 `scanf` 读 `n` 和 `n` 个整数，答案存入动态分配的 `Node* arr`，最后逐行 `printf("%d %d %d\n", ...)`。
- **算法细节问题**：排序后如果 `nums[i] > 0` 可以提前结束；固定数和左右指针都要跳过重复值，避免输出重复三元组；`result < 0` 时左指针右移，`result > 0` 时右指针左移。时间复杂度 `O(n^2)`。

### `double-pointer-and-greedy/merge.cpp`

- **思路 / 数据结构设计**：用 `interval { int l, r; }` 表示区间，数组 `intervals[]` 保存全部区间，按左端点排序后从前往后合并。
- **输入输出处理**：`scanf` 读区间数量和每个区间端点；先输出合并后的区间个数，再逐行输出每个区间。
- **算法细节问题**：当前实现发现重叠后把后面的区间整体前移，属于就地删除，复杂度可能到 `O(n^2)`；复用时更推荐维护一个写入下标或答案数组。重叠判断要明确闭区间边界，常见写法是 `next.l <= cur.r`。

### `double-pointer-and-greedy/meeting.cpp`

- **思路 / 数据结构设计**：把会议拆成两个数组 `begin[]` 和 `end[]`，分别保存所有开始时间和结束时间。排序后用两个指针模拟时间线上会议开始和结束事件。
- **输入输出处理**：`scanf` 读 `n` 和 `n` 行 `[start, end]`，`printf` 输出最少会议室数量。
- **算法细节问题**：如果下一个开始时间小于最早结束时间，说明需要新增会议室；否则释放一个会议室。这里用 `begin[i] < end[j]`，表示同一时刻结束和开始可以共用会议室。复杂度 `O(n log n)`。

### `double-pointer-and-greedy/rain.cpp`

- **思路 / 数据结构设计**：用 `vector<int> height` 保存高度，左右指针分别站在数组两端，当前容量由较短边和宽度决定。
- **输入输出处理**：`cin` 读 `n` 和高度数组，`cout` 输出最大容量。
- **算法细节问题**：每次移动较短的一边，因为容量受短板限制，移动长边不会让当前宽度下的短板变高。数组长度小于 2 直接返回 0。复杂度 `O(n)`。

### `double-pointer-and-greedy/rain-plus.cpp`

- **思路 / 数据结构设计**：仍用高度数组和左右指针，但额外维护 `leftMax/rightMax`，表示当前位置两侧见过的最高柱子。
- **输入输出处理**：`cin` 读 `n` 和高度数组，`cout` 输出总接水量。
- **算法细节问题**：每次更新两侧最大值，移动较低一侧；如果左侧较低，左侧可接水为 `leftMax - height[left]`，右侧同理。长度小于 3 不可能接水。复杂度 `O(n)`。

## 二分

### `binary-search/find-mid.cpp`

- **思路 / 数据结构设计**：两个数组分别排序后，把总数组的左半部分切成 `i + j = k` 个元素。二分较短数组的切分点 `i`，另一个数组切分点由 `j = k - i` 决定。
- **输入输出处理**：`scanf` 读两个数组长度和元素，先 `qsort` 排序，再输出一位小数格式的中位数。
- **算法细节问题**：用 `INT_MIN/INT_MAX` 处理切在数组边界的情况；合法切分要求 `nums1Left <= nums2Right && nums2Left <= nums1Right`。总长度奇数取左半最大值，偶数取左半最大和右半最小的平均值。复杂度 `O(log min(m,n))`。

### `binary-search/lis-binary.cpp`

- **思路 / 数据结构设计**：`tails[i]` 表示长度为 `i + 1` 的递增子序列中最小的结尾值。每个新数用二分找第一个 `>= target` 的位置。
- **输入输出处理**：`scanf` 读 `n` 和数组，`printf` 输出 LIS 长度。
- **算法细节问题**：找到的位置如果等于当前长度，就扩展 `tails`；否则替换原位置，使同长度子序列的结尾尽量小。二分区间是 `[0, maxLen)`。复杂度 `O(n log n)`。

### `binary-search/minEatingSpeed.cpp`

- **思路 / 数据结构设计**：二分答案 `speed`。`getTime` 计算给定速度吃完所有香蕉需要多少小时。
- **输入输出处理**：`cin` 读 `pilesSize h` 和数组，用 `vector<int>` 接收，再把 `data()` 传给函数，`cout` 输出最小速度。
- **算法细节问题**：速度下界为 1，上界为最大堆香蕉数；`getTime` 中每堆耗时要向上取整。速度越大，耗时越小，满足单调性，可以二分最小可行值。复杂度 `O(n log M)`。

### `binary-search/shipWithDays.cpp`

- **思路 / 数据结构设计**：二分船的承载能力 `capacity`。`getDays` 顺序装货，超过容量就开启新一天。
- **输入输出处理**：`cin` 读货物数量、天数和每个重量，用 `vector<int>` 保存，输出最小运载能力。
- **算法细节问题**：容量下界是单件最大重量，上界是所有重量总和；容量越大，所需天数越少。`getDays` 初始天数为 1，遇到装不下时先 `day++` 再重置当天载重。复杂度 `O(n log S)`。

## 动态规划

### `dynamic-programming/coinChange.cpp`

- **思路 / 数据结构设计**：完全背包模型，`dp[x]` 表示凑出金额 `x` 需要的最少硬币数。`amount + 1` 作为不可达哨兵。
- **输入输出处理**：第一行用 `getline(cin >> ws, line)` 整行读取硬币列表，`parseCoins` 把非数字和非负号字符替换为空格，再用 `stringstream` 提取整数；第二行读 `amount`。
- **算法细节问题**：`dp[0] = 0`；枚举每个硬币后正序遍历金额，表示硬币可以重复使用；非正硬币跳过。最后如果仍为哨兵则输出 `-1`。复杂度 `O(amount * coinCount)`。

### `dynamic-programming/eraseOverlapIntervals.cpp`

- **思路 / 数据结构设计**：先把“删除最少区间”转成“保留最多互不重叠区间”。按右端点排序，`dp[i]` 表示前 `i` 个区间最多能保留多少个。
- **输入输出处理**：`cin` 读 `n` 和 `n` 个区间；对非法或空输入做提前返回；输出最少删除数。
- **算法细节问题**：`ends[]` 保存排序后的右端点，对当前区间起点 `start` 用 `upper_bound` 找前面有多少区间满足 `end <= start`。转移为 `dp[i] = max(dp[i-1], dp[j] + 1)`。复杂度 `O(n log n)`。

### `dynamic-programming/lcis.cpp`

- **思路 / 数据结构设计**：只需要两个变量，`pre` 表示以当前元素结尾的连续递增长度，`res` 表示答案。
- **输入输出处理**：`cin` 读 `n`，如果 `n == 0` 输出 0；之后读数组并输出 LCIS 长度。
- **算法细节问题**：当前数大于前一个数时 `pre++`，否则重置为 1。LCIS 要求连续，不能像 LIS 一样跳过元素。复杂度 `O(n)`。

### `dynamic-programming/lis.cpp`

- **思路 / 数据结构设计**：文件中保留两种做法。朴素 DP 用 `dp[i]` 表示以 `nums[i]` 结尾的 LIS 长度；优化版用 `dp[end]` 保存各长度的最小结尾值。
- **输入输出处理**：`cin` 读 `n` 和数组，当前 `main` 调用二分优化版并输出长度。
- **算法细节问题**：朴素版双层循环判断 `nums[j] < nums[i]`；二分版遇到更大值就扩展，否则找第一个 `>= nums[i]` 的位置替换。注意是严格递增，不能用 `>` 和 `>=` 混乱。复杂度分别为 `O(n^2)` 和 `O(n log n)`。

### `dynamic-programming/maxEnvelopes.cpp`

- **思路 / 数据结构设计**：二维信封先排序降维，再在高度上做 LIS。宽度升序，宽度相同时高度降序，防止同宽信封互相嵌套。
- **输入输出处理**：`cin` 读 `n` 和每个信封的宽高，空输入或 `n == 0` 直接输出 0。
- **算法细节问题**：排序后只比较第二维高度；源码中的 `dp` 存的是信封下标，不是直接存高度，所以比较时要访问 `envelopes[dp[mid]][1]`。复杂度 `O(n log n)`。

### `dynamic-programming/maxSubArray.cpp`

- **思路 / 数据结构设计**：Kadane 算法，`pre` 表示以当前元素结尾的最大子数组和，`res` 保存全局最大值。
- **输入输出处理**：`cin` 读 `n` 和数组，`n <= 0` 时直接结束。
- **算法细节问题**：转移为 `pre = max(nums[i], pre + nums[i])`，表示要么从当前数重新开始，要么接在前一段后面。全负数场景要用首元素初始化。复杂度 `O(n)`。

### `dynamic-programming/numSquare.cpp`

- **思路 / 数据结构设计**：`dp[i]` 表示组成整数 `i` 的最少完全平方数个数。
- **输入输出处理**：`cin` 读一个整数 `n`，输出 `dp[n]`。
- **算法细节问题**：`dp[0] = 0`；对每个 `i` 枚举 `j*j <= i` 的平方数，转移 `dp[i] = min(dp[i], dp[i - j*j] + 1)`。源码使用 `pow`，复用时可改成 `j * j` 减少浮点参与。复杂度 `O(n sqrt n)`。

## 图搜索

### `graph-search/dijkstra.cpp`

- **思路 / 数据结构设计**：`Graph` 结构体保存顶点数、边数和邻接矩阵 `edge[][]`；`distance[]` 保存起点到每个点的最短距离，`visited[]` 标记最短路已确定的点。
- **输入输出处理**：`scanf` 读顶点数、边数和每条边，矩阵先初始化为对角线 0、其余 `INF`；输出从 0 号点到最后一个点的距离，不可达输出 `-1`。
- **算法细节问题**：每轮从未访问点中找距离最小的点，再用它松弛所有邻接点。当前代码写入边时用了 `G.edge[-x][--y] = num`，说明它依赖特定题面下标或符号约定，移植时必须先统一成 0 基或 1 基下标。邻接矩阵版复杂度 `O(n^2)`。

### `graph-search/numOfLand.cpp`

- **思路 / 数据结构设计**：字符网格 `char** grid`，`'1'` 表示陆地，`'0'` 表示水。DFS 把访问过的陆地直接改成水。
- **输入输出处理**：`scanf` 读行列，逐行 `scanf("%s", grid[i])` 读字符矩阵，输出岛屿数量。
- **算法细节问题**：遍历每个格子，遇到 `'1'` 就答案加一并 DFS 淹没整个连通块。边界判断必须在访问数组前完成。复杂度 `O(row * col)`。

### `graph-search/numOfLand2.cpp`

- **思路 / 数据结构设计**：整数网格，`1` 表示陆地。先从边界出发 DFS，把所有连到边界的陆地清掉，剩下的才是封闭岛屿。
- **输入输出处理**：`scanf` 读行列和矩阵，动态分配二维 `int**`，输出封闭岛屿数量并释放内存。
- **算法细节问题**：要先处理第一行、最后一行、第一列、最后一列，再统计内部剩余连通块。DFS 访问后置 0 防止重复。复杂度 `O(row * col)`。

### `graph-search/sizeOfLand.cpp`

- **思路 / 数据结构设计**：和封闭岛屿类似，额外用全局变量 `size` 累加内部封闭陆地的总面积。`flag` 控制当前 DFS 是否计入面积。
- **输入输出处理**：`scanf` 读矩阵，输出面积总和。
- **算法细节问题**：边界连通陆地用 `flag = 0` 清掉但不计面积；内部陆地用 `flag = 1`，DFS 到一个格子就 `size++`。全局变量复用多组数据时需要手动清零。复杂度 `O(row * col)`。

### `graph-search/sonOfLand.cpp`

- **思路 / 数据结构设计**：两个整数网格 `grid1/grid2`。先把 `grid2` 中落在 `grid1` 水域上的连通块全部清掉，剩下的 `grid2` 岛屿就是子岛屿。
- **输入输出处理**：`scanf` 读行列，依次读两个矩阵，输出子岛屿数量并释放两张网格。
- **算法细节问题**：第一遍条件是 `grid1[i][j] == 0 && grid2[i][j] == 1`，这些岛不可能是子岛，要整块删除；第二遍统计 `grid2` 剩余连通块。复杂度 `O(row * col)`。

### `graph-search/numOfDiffLand.cpp`

- **思路 / 数据结构设计**：用 `vector<vector<int>> grid` 存网格，用 `path` 记录一个岛屿中每个格子相对起点的坐标，再把 `path` 放入 `set` 去重。
- **输入输出处理**：`cin` 读 `m n` 和整数网格，`cout` 输出不同岛屿形状数量。
- **算法细节问题**：相同形状出现在不同位置时，绝对坐标不同但相对坐标相同；DFS 方向顺序固定，路径表示才稳定。访问过的格子置 0。复杂度 `O(mn log K)`，`K` 为不同形状数。

## 贪心

### `greedy/arrayPairSum.cpp`

- **思路 / 数据结构设计**：排序后相邻两两配对，每对较小值就是贡献。用 `vector<int>` 保存数组。
- **输入输出处理**：`cin` 读 `n` 和数组，`cout` 输出最大贡献和。
- **算法细节问题**：升序后应取下标 `0, 2, 4...` 的元素；源码从 `i = 1` 开始累加 `min(nums[i], nums[i-1])`，效果等价。复杂度 `O(n log n)`。

### `greedy/candy.cpp`

- **思路 / 数据结构设计**：不用开糖果数组，维护上一人糖果数 `pre`、最近递增段长度 `inc`、递减段长度 `dec` 和总数 `ans`。
- **输入输出处理**：`cin` 读评分数组，输出最少糖果数。
- **算法细节问题**：评分上升时 `pre + 1`，相等时重置为 1；下降时递减段长度增加，如果 `dec == inc`，峰顶需要补一颗糖，所以 `dec++`。复杂度 `O(n)`。

### `greedy/eraseOverlapIntervals.cpp`

- **思路 / 数据结构设计**：区间按右端点升序排序，优先保留结束最早的区间，统计最多能保留多少个。
- **输入输出处理**：`cin` 读区间数量和端点，空输入输出 0。
- **算法细节问题**：当前区间起点 `>= end` 才能接在已选区间后面；最终答案是总数减去保留数。复杂度 `O(n log n)`。

### `greedy/findMinArrowShots.cpp`

- **思路 / 数据结构设计**：把每个气球看成区间，箭的位置 `pos` 放在当前可覆盖区间组的最小右端点。
- **输入输出处理**：`cin` 读 `n` 和每个气球左右边界，输出最少箭数。
- **算法细节问题**：按右端点排序；如果下一个气球左端点 `> pos`，当前箭射不到，需要新箭并更新 `pos`。边界相等时仍可被同一支箭射中。复杂度 `O(n log n)`。

### `greedy/increasingDigits.cpp`

- **思路 / 数据结构设计**：把数字转成 `string`，方便逐位修改。
- **输入输出处理**：`cin` 读整数，`cout` 输出不超过它的最大单调递增数字。
- **算法细节问题**：先找到第一处下降；再向左回退，遇到前一位仍大于后一位就前一位减 1；最后把修改位之后全部置为 `'9'`，保证结果最大。复杂度 `O(d)`。

### `greedy/jump.cpp`

- **思路 / 数据结构设计**：维护 `maxReached`，表示当前扫描到的位置能够覆盖到的最远下标。
- **输入输出处理**：`cin` 读数组，直接输出 `bool` 的 `0/1`。
- **算法细节问题**：如果扫描位置 `i > maxReached`，说明出现断点不可达；只要 `maxReached >= len - 1` 就可以提前返回 true。复杂度 `O(n)`。

### `greedy/jump2.cpp`

- **思路 / 数据结构设计**：把一次跳跃能覆盖的范围看成一层。`current` 是当前步数能覆盖的右边界，`next` 是再跳一步能覆盖的最远位置。
- **输入输出处理**：`cin` 读数组，输出最少跳数。
- **算法细节问题**：遍历到 `current` 时说明当前层结束，必须跳一次并把边界更新为 `next`；循环只到 `len - 2`，避免到终点后多加一步。复杂度 `O(n)`。

### `greedy/maxProfit.cpp`

- **思路 / 数据结构设计**：股票多次买卖可以把总利润拆成所有上升相邻差值之和。
- **输入输出处理**：`cin` 读价格数组，输出最大利润。
- **算法细节问题**：只要 `prices[i+1] > prices[i]` 就累加差值，相当于低点买入、高点卖出。复杂度 `O(n)`。

### `greedy/maxSubArray.cpp`

- **思路 / 数据结构设计**：用 `sum` 保存当前连续段贡献，`ans` 保存最大值。
- **输入输出处理**：`cin` 读数组，输出最大子数组和。
- **算法细节问题**：如果 `sum >= 0`，当前数接上前段更好；如果 `sum < 0`，从当前数重新开始。`ans` 用首元素初始化以支持全负数组。复杂度 `O(n)`。

### `greedy/meeting.cpp`

- **思路 / 数据结构设计**：按会议开始时间排序，用小根堆保存当前正在使用的会议室结束时间。
- **输入输出处理**：`cin` 读区间，`priority_queue<int, vector<int>, greater<int>>` 维护最早结束时间，输出最大堆大小。
- **算法细节问题**：处理新会议前，把所有 `end <= start` 的会议室弹出；再压入当前会议结束时间。堆的最大大小就是会议室数。复杂度 `O(n log n)`。

### `greedy/wiggleMaxLength.cpp`

- **思路 / 数据结构设计**：只记录上一个非零差值 `prev` 和当前差值 `curr`，遇到趋势反转就计数。
- **输入输出处理**：`cin` 读数组，输出最长摆动子序列长度。
- **算法细节问题**：相邻差为 0 不构成摆动；判断条件是 `(curr > 0 && prev <= 0) || (curr < 0 && prev >= 0)`。复杂度 `O(n)`。

## 字符串索引

### `string-index/countMatchingSubsequences.cpp`

- **思路 / 数据结构设计**：建立倒排索引 `index[26]`，每个字符保存它在主串中出现的所有下标。

### `test/gplt/2026/traffic-monitoring.cpp`

- **思路 / 数据结构设计**：用数组保存每个监测点流量，同时在线维护最大值、最小值和总和。
- **输入输出处理**：`cin` 读监测点数量和流量，先输出最大值、最小值、整数平均值，再输出超过平均值两倍的监测点编号或 `Normal`。
- **算法细节问题**：监测点编号按输入下标从 1 开始；异常条件是 `traffic[i] > ave * 2`，不是大于等于。复杂度 `O(n)`。
