# Day 20：并查集与最小生成树

## ▶ CSP 考情分析

并查集（Union-Find）和最小生成树（MST）在 CSP 第 4 题中反复出现。这两者紧密关联——Kruskal 算法就是并查集 + 排序边。

**CSP 并查集/MST 真题：**
| 真题 | 考点 |
|------|------|
| 最优灌溉(201412-4) | 最小生成树（裸 Kruskal） |
| 数据中心(201812-4) | 最小生成树（MST 的瓶颈性质） |
| 地铁修建(201703-4) | 二分 + 并查集连通性判断 |
| 公路(2023-J T2) | 并查集维护连通块 |

**CSP 爱考 MST 的原因：** MST 算法本身简单（Kruskal ~20 行），但 CSP 会包装一层"应用题面"让你识别出这是 MST。

## ▶ 核心技巧

### 并查集模板

```cpp
int parent[MAXN];

int find(int x) {
    return parent[x] == x ? x : parent[x] = find(parent[x]);
    // 路径压缩：让每个节点直接指向根
}

void unite(int a, int b) {
    int ra = find(a), rb = find(b);
    if (ra != rb) parent[ra] = rb;
}

// 初始化
for (int i = 1; i <= n; i++) parent[i] = i;
```

### Kruskal 最小生成树

```cpp
struct Edge { int u, v, w; };
bool cmp(Edge a, Edge b) { return a.w < b.w; }

int kruskal() {
    sort(edges, edges + m, cmp);
    int total = 0, cnt = 0;
    for (Edge& e : edges) {
        if (find(e.u) != find(e.v)) {
            unite(e.u, e.v);
            total += e.w;
            cnt++;
            if (cnt == n - 1) break;
        }
    }
    return total;  // MST 的总权重
}
```

## ▶ 真题 1：最优灌溉

**试题编号：201412-4（第3次认证第4题）**

N 块麦田，可以在两块田之间修水渠（有成本），也可以在某块田打井（有成本）。求灌溉所有麦田的最小总成本。

**思路：** 添加虚拟节点 0（水源），打井 = 连边 (0, i, cost)。在这个 N+1 个节点的图上求 MST。

**CSP 考察点：** "加虚拟节点"的技巧——把"点上的成本"转化为"边上的成本"，变成标准 MST。

## ▶ 真题 2：数据中心

**试题编号：201812-4（第17次认证第4题）**

n 个节点，m 条边，每条边有传输时间。要构建一棵树形结构，使根到所有节点的最大传输时间最小。

**思路：** 求 MST，MST 中最大的那条边就是答案。因为 MST 的"瓶颈性质"：最小生成树能最小化最大边权。

## ▶ 真题 3：地铁修建

**试题编号：201703-4（第6次认证第4题）**

n 个隧道施工点，m 条可选隧道（有修建天数）。要确保所有点连通，同时最小化最长隧道的修建天数。

**思路：** 方法一：Kruskal → 当连通分量数 = 1 时最后加入的边权就是答案。
方法二：二分答案 + 并查集验证（边权 ≤ mid 的边能否使图连通）。
