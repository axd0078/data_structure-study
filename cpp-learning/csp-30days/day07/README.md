# Day 7：STL vector 与层次模拟

## ▶ CSP 考情分析

CSP 第 2-3 题常出现"层级/顺序会动态变化"的模拟。这类题的核心数据结构是 `vector`，核心操作是 **erase + push_back（移到底层/顶层）**。

**CSP 层次模拟题的特征信号：**
- "把某个元素移到最上面" → erase + push_back
- "从最上面开始找第一个满足条件的" → 倒序遍历
- "保持从底到顶的顺序" → vector 下标 0=底，末尾=顶

| 真题 | 考查重点 |
|------|---------|
| 窗口(201403-2) | 命中后置顶 |
| 未初始化警告(202203-1) | set 维护已初始化变量 |
| 推荐系统(201909-4) | 优先队列 + 删除标记 |

## ▶ 核心技巧

### vector 置顶操作

```cpp
vector<Item> items;  // 0=底层，back()=顶层

// 把命中的元素移到顶层
Item hit = items[pos];
items.erase(items.begin() + pos);  // 从原位置删除
items.push_back(hit);              // 加入顶层
```

### 从顶层开始查找

```cpp
for (int j = items.size() - 1; j >= 0; j--) {
    if (contains(items[j], x, y)) {
        // 命中！
        break;
    }
}
```

**CSP 易错点：** erase 后所有后续元素的下标都前移。如果你用 for 循环遍历同时 erase，需要从后往前遍历（i--）。

## ▶ 真题 1：窗口

**试题编号：201403-2（第1次认证第2题）**

N 个窗口（N ≤ 10），每个用 (x1,y1)-(x2,y2) 定义。M 次鼠标点击，输出点击到的最顶层窗口编号。命中的窗口被移到最顶层。未命中输出 IGNORED。

| 输入样例 | 输出样例 |
|---------|---------|
| 3 4\n0 0 4 4\n1 1 5 5\n2 2 6 6\n1 1\n0 0\n4 4\n0 5 | 2\n1\n1\nIGNORED |

**思路：** vector 存窗口（编号+坐标），每次从后往前找命中窗口，erase 后 push_back 置顶。

## ▶ 真题 2：未初始化警告

**试题编号：202203-1（第25次认证第1题）**

n 个变量，k 条赋值语句。如果某赋值语句右值中的变量尚未被赋值过，产生一条警告。求警告总数。

| 输入样例 | 输出样例 |
|---------|---------|
| 10 7\n1 2\n3 4\n5 6\n7 8\n9 10\n2 3\n4 5 | 5 |

**思路：** 维护 `unordered_set<int>` 存已赋值的变量。遍历每句赋值，检查右值是否在 set 中，不在则计数+1。处理完后把左值加入 set。

**CSP 考察点：** 表面是模拟，实际在考你用什么结构快速判断"是否出现过"——set/hash 的典型应用。

## ▶ 真题 3：推荐系统（进阶了解）

**试题编号：201909-4（第17次认证第4题）**

有 m 类商品，每类最多选 k 个。支持增、删、查最高分商品。商品用 (id, score) 表示，同类按分数排序。

**思路：** 每类商品用一个 `set<pair<int,int>>`（分数→id），全局再用一个 `set` 维护各类分数最高的商品。增删时同步更新两层容器。

**CSP 考察点：** 多级索引的同步维护——单个 `set` 不够用，因为查询维度和删除维度不同。
