# Day 11：多容器索引与区间维护

## ▶ CSP 考情分析

CSP 中有一类题，同一个数据集有**两种以上查询维度**。比如既要"按左端点查找"又要"按长度排序"——单用一个容器无法同时高效支持。这是 CSP 第 3-4 题中重要的设计思想。

**CSP 多容器索引的典型信号：**
- 需要按不同 key 查找（如按左端点、按长度、按 id）
- 插入/删除时要同步更新所有容器
- 题目明确说"选择最短的"同时"与左右合并"

| 真题 | 容器组合 | 用途 |
|------|---------|------|
| 区间分配(第41次T3) | map + set | map按左端点、set按长度 |
| 集合竞价(201412-3) | map + vector | 双向查找 |
| 推荐系统(201909-4) | set嵌套 | 按类别+分数双层索引 |

## ▶ 核心技巧

### 为什么一个容器不够？

```cpp
// map: 按左端点快速找相邻段 → 适合合并
map<long long, long long> freeByL;

// set: 按长度排序 → 适合找最短满足条件的段
set<Segment, SegmentCmp> freeByLen;

// 增删时必须同步更新两者！
void addFree(long long l, long long r) {
    freeByL[l] = r;
    freeByLen.insert({l, r});
}
```

### 合并相邻段（释放时）

释放区间 [l, r] 时，检查左右是否与已有空闲段相邻：
1. `lower_bound(l)` 找右邻居 → 如果 `r+1 == right.l` 则合并
2. `prev(it)` 找左邻居 → 如果 `left.r+1 == l` 则合并
3. 移除旧段，插入合并后的新段

## ▶ 真题：区间分配

**来源：第 41 次认证第 3 题**

支持三种操作：
- `new p L`：为进程 p 分配长 L 的空间（最短且最左），输出起始地址
- `delete p i`：释放进程 p 的第 i 次分配
- `send p`：循环输出进程 p 队列的下一个地址

**思路：**
1. `freeByL` (map)：按左端点，释放时找左右合并
2. `freeByLen` (set)：按长度排序，分配时二分查最短可用段
3. 每个进程维护 queue，记录每次分配的区间和当前输出指针

**CSP 考察点：** 多容器索引 + 区间合并 + 进程状态管理。这是 CSP 第 3 题的巅峰难度。

## ▶ 类似真题：命令行选项

**试题编号：201403-3（第1次认证第3题）**

解析命令行参数：带参数选项 vs 不带参数选项，处理不合法的选项。

**思路：** 用 `map<char, string>` 存选项→参数映射，用 `unordered_set<char>` 存合法选项。两种查询维度自然地用了两个容器。
