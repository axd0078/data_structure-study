# Day 14：贪心算法

## ▶ CSP 考情分析

贪心算法在 CSP 中的考察频率仅次于动态规划，尤其是**第 2-4 题**。CSP 的贪心题通常不是裸贪心，而是**"贪心 + 其他算法"的组合**。

**CSP 贪心题的常见形态：**
| 形态 | 真题 |
|------|------|
| 纯贪心（按收益排序） | 最优配餐(201409-4) |
| 贪心 + 背包 | 背包贪心(第41次T2) |
| 贪心 + 前缀和 | 公路(2023-J T2) |
| 贪心 + 区间调度 | 出行计划(202203-2) |

**CSP 贪心题的陷阱：** 不是所有"选最优"都是贪心。贪心成立的条件是——局部最优能推到全局最优。如果题目描述中有"选与不选都有后续影响"，可能是 DP。

## ▶ 核心技巧

### 分数贪心（按单位收益排序）

```cpp
sort(tasks.begin(), tasks.end(), [](Task& a, Task& b) {
    return 1.0 * a.value / a.cost > 1.0 * b.value / b.cost;
});

double gain = 0, left = budget;
for (Task& t : tasks) {
    double take = min(left, (double)t.cost);
    gain += take * t.value / t.cost;
    left -= take;
}
```

### 物品可分割 → 贪心 / 不可分割 → 背包

这是 CSP 第 2 题的经典考点：题目说"可以按比例分配"就是用贪心，"只能整数份"就是用背包。

### 混合类型：枚举 + 贪心

当一部分任务用贪心、一部分用背包时，核心做法是：**枚举分给背包的资源量，剩余给贪心**。

```
for used = 0 to totalBudget:
    best = max(best, knapSack(used) + greedy(totalBudget - used))
```

## ▶ 真题 1：两种任务的最优分配

**来源：第 41 次认证第 2 题（简化）**

n 个任务，类型 o=0 可分（比例减时、贪心），o=1 不可分（整数咖啡、背包）。有 m 杯咖啡，求最短总时间。

**思路：**
1. 不可分任务：0/1 背包求 dp[0..m]（dp[j] = 用 j 杯咖啡最大减时量）
2. 可分任务：按 b/a 收益率排序
3. 枚举 used ∈ [0, m]：总减时 = dp[used] + greedy(m - used)
4. 答案 = 原始总耗时 - max(总减时)

## ▶ 真题 2：最优配餐

**试题编号：201409-4（第2次认证第4题）**

n×m 网格，有 k 个起点（分店）、d 个障碍、c 个客户（位置+订餐量）。求所有分店到所有客户的总配送费用最小。

**思路：** 多源 BFS（所有分店同时入队作起点），每个格子记录到最近分店的距离。每个客户的费用 = 距离 × 订餐量。

**为什么是贪心？** BFS 保证找到的每个格子"到最近分店的距离"是最优的——这是局部最优到全局最优的体现。
