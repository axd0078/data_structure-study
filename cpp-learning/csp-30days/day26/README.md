# Day 26：状态压缩 DP

## ▶ CSP 考情分析

状态压缩 DP（状压 DP）在 CSP 中出现的频率在上升，尤其是 2020 年之后。CSP 的状压 DP 题通常出现在第 4-5 题。

**CSP 状压 DP 真题：**
| 真题 | 考点 |
|------|------|
| 收集卡牌(202109-4) | 期望 DP + 状压 |
| 食材运输(202012-4) | 二分 + 树形 DP + 状压 DP |
| 彩色路径(202309-5) | 折半搜索 + 状压 |

**CSP 状压 DP 的识别信号：**
- n ≤ 20（元素个数小到可以用 2^n 表示子集）
- "选一些元素"、"子集覆盖"
- 状态中需要记录"哪些已经选过了"

## ▶ 核心技巧

### 用 bit 表示集合

```cpp
// 第 i 位 = 1 表示第 i 个元素在集合中
int mask = 0;
mask |= (1 << i);        // 加入元素 i
mask &= ~(1 << i);       // 移除元素 i
if (mask & (1 << i)) {}  // 检查元素 i 是否在集合中
int full = (1 << n) - 1; // 全集（所有 n 个元素）
```

### 常见状压 DP 模式

```cpp
// dp[mask] = 已经选了 mask 集合后的最优值
for (int mask = 0; mask < (1 << n); mask++) {
    for (int i = 0; i < n; i++) {
        if (!(mask & (1 << i)))  // i 还没选
            dp[mask | (1 << i)] = min(dp[mask | (1 << i)], dp[mask] + cost(mask, i));
    }
}
```

### 子集枚举

```cpp
// 枚举 mask 的所有子集
for (int sub = mask; sub; sub = (sub - 1) & mask) {
    // sub 是 mask 的一个子集
}
```

## ▶ 真题 1：收集卡牌

**试题编号：202109-4（第23次认证第4题）**

有 n 种卡牌（n ≤ 16），每次抽卡获得一种随机卡牌。集齐所有卡牌时可以兑换硬币（兑换率 = 已收集数量）。问抽卡次数的期望最小值。

**思路：** 状压 DP + 期望。
- mask 表示已收集的卡牌种类
- dp[mask] = 从当前已收集 mask 到集齐的期望抽卡次数
- 转移：抽到已拥有的 → 立即兑换（跳过），抽到新的 → dp[mask | newBit]

**CSP 考察点：** 状压 + 概率期望的结合——CSP 近年常考概率期望，且 n ≤ 16 强烈暗示状压。

## ▶ 真题 2：食材运输（状压部分）

**试题编号：202012-4（第19次认证第4题，综合题）**

树形 DP 计算出"从节点 i 出发能覆盖哪些食材种类"（用 bit mask 表示）。然后状压 DP：选 m 个起点，确保它们的 mask 之并 = 全集。

```cpp
// dp[t][mask] = 选了 t 个起点，能覆盖 mask 的可行性
for (int node : validStarts) {
    for (int t = m - 1; t >= 0; t--)
        for (int mask = 0; mask < full; mask++)
            if (dp[t][mask])
                dp[t+1][mask | cover[node]] = true;
}
```

**CSP 考察点：** 当 n 小（食材种类数小）时，状压 DP 是处理"覆盖/选择"子集问题的标准工具。

---

## ▶ 状压 DP 常见优化

- **折半搜索（Meet-in-the-Middle）：** n 较大时，分两半各自状压，再合并
- **滚动数组：** 如果只依赖前一层，可以降维
- **预计算 transfer 表：** 避免在循环内重复计算
