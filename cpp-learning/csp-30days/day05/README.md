# Day 5：数列与状态变化

## ▶ CSP 考情分析

CSP 第 1-2 题中，**"遍历序列 + 维护状态变量"** 是最常见的出题模式之一。题目通常会描述一个"过程"，你需要维护一个或多个状态变量在遍历中更新。

**CSP 中的经典状态变量类型：**

| 状态变量 | 真题示例 | 管什么 |
|---------|---------|--------|
| prev（上一个值） | 数列分段(201509-1) | 判断"一段"是否结束 |
| streak（连续次数） | 跳一跳(201803-1) | 连续命中的递增值 |
| curSum（当前和） | 非零段划分(202109-2) | 累计到当前的和 |

**CSP 爱考的坑：** 把"重置状态"和"更新状态"的顺序写反。核心原则——**先判断当前输入触发了什么变化，再决定状态是累积还是重置。**

## ▶ 核心技巧

### 套路 1：判断新段

```cpp
int prev = -1, seg = 0;
for (int i = 0; i < n; i++) {
    int x; cin >> x;
    if (i == 0 || x != prev) seg++;  // 跟上一个不同 → 新段
    prev = x;
}
```

### 套路 2：连续得分/递增

```cpp
int streak = 0, total = 0;
while (cin >> x && x != 0) {
    if (x == 2) {
        streak += 2;      // 连续 +2
        total += streak;
    } else if (x == 1) {
        streak = 0;       // 中断重置
        total += 1;
    }
}
```

## ▶ 真题 1：数列分段

**试题编号：201509-1（第5次认证第1题）**

数列中连续相同的整数算一段，问共有多少段。

| 输入样例 | 输出样例 |
|---------|---------|
| 8\n8 8 8 0 12 12 8 0 | 5 |

说明：`8 8 8` 第一段，`0` 第二段，`12 12` 第三段，`8` 第四段，`0` 第五段。

**CSP 考察点：** 你能否意识到"只需要记住上一个数"就够了——不需要把整个数组存下来再处理。

## ▶ 真题 2：跳一跳

**试题编号：201803-1（第13次认证第1题）**

输入 1/2/0 序列：
- 1：跳到方块没到中心，得 1 分
- 2：跳到中心，连续则每跳递增 2 分（2,4,6,8...），中断则从 2 重新开始
- 0：游戏结束

| 输入样例 | 输出样例 |
|---------|---------|
| 1 1 2 2 2 1 1 2 2 0 | 22 |

得分过程：1 + 1 + 2 + 4 + 6 + 1 + 1 + 2 + 4 = 22

**CSP 考察点：** `lastScore` 的状态维护——什么时候重置为 1、什么时候从 2 开始、什么时候递增 +2。三种情况的切换逻辑是 CSP 最爱考的。

## ▶ 真题 3：非零段划分（进阶）

**试题编号：202109-2（第23次认证第2题）**

给定数组 A，可以选一个正数 p，把 A 中小于 p 的变为 0。问如何选 p 使非零段数量最大。

| 输入样例 | 输出样例 |
|---------|---------|
| 11\n3 1 2 0 0 2 0 4 5 0 2 | 5 |

当 p=2 时：[3,1,2] 变成 [3,0,2] → 3 个非零段 + [2] 、[4,5]、[2] → 共 5 段。

**CSP 考察点：** 这是状态变化题的质变——不再只维护"上一个值"，而是需要分析"水位上升时段的合并与分裂"。用扫描法 + 分类讨论每个元素前后的状态。
