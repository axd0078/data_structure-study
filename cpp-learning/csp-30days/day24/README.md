# Day 24：区间 DP

## ▶ CSP 考情分析

区间 DP 在 CSP 第 4-5 题中出现频率不高但一旦出现往往就是关键题。CSP 的区间 DP 题有一个明显特征：**问题可以在"某个断点"处分成两个子问题**。

**CSP 区间 DP 真题：**
| 真题 | 考点 |
|------|------|
| 压缩编码(201612-4) | 石子合并模型（区间 DP 经典题） |
| 拼图(201409-5) | 区间 DP + 矩阵快速幂 |

**CSP 区间 DP 的识别信号：**
- "相邻合并"、"合并成一段"
- 问题描述类似"每次选两个相邻的合并，代价为..."
- 答案与"某种合并顺序"有关

## ▶ 核心技巧

### 区间 DP 模板

```cpp
// dp[l][r] = 区间 [l, r] 的最优解
for (int len = 2; len <= n; len++) {          // 枚举区间长度
    for (int l = 0; l + len - 1 < n; l++) {   // 枚举左端点
        int r = l + len - 1;
        for (int k = l; k < r; k++) {         // 枚举分割点
            dp[l][r] = min(dp[l][r], dp[l][k] + dp[k+1][r] + cost(l, r));
        }
    }
}
```

**三重循环顺序：** 长度 → 左端点 → 分割点。不能颠倒。

### 石子合并模型（CSP 考过的）

N 堆石子排成一行，每次合并相邻两堆，代价为两堆石子数之和。求最小总代价。

```cpp
// dp[l][r] = min_{k=l..r-1} (dp[l][k] + dp[k+1][r]) + sum(l, r)
// sum(l, r) 用前缀和 O(1) 求
```

## ▶ 真题 1：压缩编码

**试题编号：201612-4（第8次认证第4题）**

给定 n 个词的词频。压缩时两个相邻词可以合并成一个新词（新词频 = 两者之和），合并代价为新词频。求最小编码长度。

**分析：** 这就是石子合并的模型。每次合并相邻两段，代价 = 频次之和。用前缀和维护频次和，区间 DP 求解。

**和哈夫曼编码的区别：** 石子合并是相邻合并（区间 DP），哈夫曼编码可以任意合并（贪心/优先队列）。CSP 爱考这个区别。

## ▶ 真题 2：拼图（扩展了解）

**试题编号：201409-5（第2次认证第5题）**

用 1×2 的骨牌拼 n×m 的矩形。n ≤ 10^18，m ≤ 5。

**思路：** 按列 DP + 状态压缩。dp[col][mask] 表示填充到第 col 列，当前列缺口状态为 mask 的方案数。由于 n 巨大，转移矩阵用**矩阵快速幂**优化。

**CSP 考察点：** 区间 DP 思想（逐列推进）+ 矩阵快速幂——CSP 第 5 题的典型组合。

## ▶ 区间 DP 的常见变体

```
dp[l][r][0/1] → 考虑区间两端"选不选"的状态
环形区间 DP → 复制数组 a[0..2n-1] 取长度为 n 的最优区间
回文相关    → dp[l][r] = 删掉两端或不删
```
