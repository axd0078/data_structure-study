## ▶ CSP 考情分析

**动态规划是 CSP 第 4-5 题的绝对核心**，占比约 25%-30%。CSP 的 DP 题不会出偏门 DP 模型，考的都是经典类型，但包装了实际场景。

**CSP DP 题的识别信号：** "最少/最多" 、 "方案数" 、 "选择一些物品" 、 "决策有后效性"。

## ▶ 核心技巧

### DP 三要素

1. **状态定义：** dp[i] 表示什么？（最难的一步）
2. **转移方程：** dp[i] 从哪些之前的状态推过来？
3. **初始值/边界：** dp[0] = ?

### 线性 DP 简单例子

```cpp
// 最大连续子段和
int maxSum = a[0], cur = a[0];
for (int i = 1; i < n; i++) {
    cur = max(a[i], cur + a[i]);
    maxSum = max(maxSum, cur);
}
```

### 线性 DP 的常见状态

```
dp[i]     → 前 i 个元素的最优解
dp[i][0/1] → 第 i 个元素选/不选的两种状态
dp[i][k]  → 前 i 个元素，选了 k 个的最优解
```

## ▶ 真题 1：有趣的数

**试题编号：201312-4（第1次认证第4题）**

求 n 位数字中"有趣的数"的数量。一个数"有趣"的定义：
1. 只含 0, 1, 2, 3
2. 所有 0 必须出现在 1 前面，所有 2 必须出现在 3 前面
3. 最高位不为 0

**思路：** 线性 DP，按"使用过哪些数字"分 6 种状态：
0: 只用 2
1: 用了 2,0
2: 用了 2,3
3: 用了 2,0,1
4: 用了 2,0,3
5: 用了全部 2,0,1,3

dp[i][state] = 前 i 位，状态为 state 的方案数。

**CSP 考察点：** "按约束分阶段"的 DP 建模——不直接 dp[i]=答案，而是用状态机表示"当前满足了多少约束"。
