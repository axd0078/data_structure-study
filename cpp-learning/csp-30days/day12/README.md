# Day 12：矩阵与坐标变换

## ▶ CSP 考情分析

矩阵操作是 CSP 第 1-3 题的常客，尤其在图像处理、数据重塑、矩阵运算等场景。

**CSP 矩阵考什么？**
| 考点 | 真题 |
|------|------|
| 旋转/翻转 | 图像旋转(201503-1) |
| Z 字形遍历 | Z 字形扫描(201412-2) |
| 矩阵重塑 | 矩阵重塑(202406-1/2) |
| 子矩阵操作 | 字符矩阵逆向(第40次T3)、消除类游戏(201512-2) |
| 矩阵运算 | 矩阵运算(202303-1) |

**CSP 矩阵题的核心思想：** 不要真的去旋转/移动数据（O(n²) 开销大），而是用**坐标映射函数**让你"看起来"旋转了。

## ▶ 核心技巧

### 旋转 90° 的坐标映射

```
顺时针 90°：新(x,y) = 旧(y, n-1-x)
逆时针 90°：新(x,y) = 旧(n-1-y, x)
旋转 180°：新(x,y) = 旧(n-1-x, n-1-y)
```

### 延迟旋转法（用 rot 变量）

```cpp
int rot = 0;  // 当前旋转角度（0/1/2/3 表示 0°/90°/180°/270°）

pair<int,int> pos(int x, int y) {
    if (rot == 0) return {x, y};
    if (rot == 1) return {Z - y - 1, x};
    if (rot == 2) return {Z - x - 1, Z - y - 1};
    return {y, Z - x - 1};
}

// 所有读写走 pos() 而不是直接访问 g[x][y]
char getCell(int x, int y) { auto p = pos(x, y); return g[p.first][p.second]; }
```

**优势：** 多个旋转操作叠加时只需 `rot = (rot + r) % 4`，不用真的转矩阵。

## ▶ 真题 1：图像旋转

**试题编号：201503-1（第4次认证第1题）**

n×m 矩阵逆时针旋转 90° 输出。

| 输入样例 | 输出样例 |
|---------|---------|
| 2 3\n1 5 3\n3 2 4 | 3 4\n5 2\n1 3 |

**思路：** 输出时：第 j 行 = 原矩阵的第 m-1-j 列（从上到下）。不需要创建新矩阵，直接按变换后的坐标输出。

## ▶ 真题 2：Z 字形扫描

**试题编号：201412-2（第3次认证第2题）**

n×n 矩阵按 Z 字形（对角线方向交替）输出所有元素。

| 输入样例 | 输出样例 |
|---------|---------|
| 4\n1 5 3 9\n3 7 5 6\n9 4 6 4\n7 3 1 3 | 1 5 3 9 7 3 9 5 4 7 3 6 6 4 1 3 |

**思路：** 枚举每条对角线 `i+j = 0..2n-2`，根据对角线编号的奇偶决定遍历方向（从上到下 vs 从下到上）。

**CSP 考察点：** 矩阵题关键是找清楚行列的循环顺序——哪个是外层循环、哪个是内层。

## ▶ 真题 3：消除类游戏

**试题编号：201512-2（第6次认证第2题）**

n×m 棋盘，如果一行或一列中有连续 3 个及以上相同数字，这些位置被消除（变为 0）。消除可能同时发生，最终输出棋盘。

**思路：** 用标记数组记录哪些位置要消除（不能边遍历边消除，因为可能同时属于行消除和列消除）。先标记所有应消除位置，再统一置 0。

**CSP 考察点：** 二维矩阵 + 行列分别处理 + 标记后处理——这是 CSP 矩阵题的经典模式。
