# Day 13：进制转换与编码解码

## ▶ CSP 考情分析

进制转换在 CSP 中是"单看简单，组合起来能出大题"的能力。CSP 第 2-3 题多次出现编码解码类题目，难度从简单到复杂跨度很大。

**CSP 进制/编码题的难度分层：**
| 难度 | 真题 | 考察内容 |
|------|------|---------|
| T1 | 数位之和(201512-1) | 十进制取位 |
| T2 | 二进制位统计(第41次T1) | 除 2 统计 0/1 |
| T3 | 登机牌条码(202112-3) | 多进制混合编码 |
| T3 | 短消息解码(第38次T3) | 38 进制 + 混合进制 + 散列 |

**CSP 编码题必考点：** 你能区分"编码"和"解码"的互逆关系——最低位是第一个被 % 出来的。

## ▶ 核心技巧

### 编码（逐位乘）

```cpp
long long encode(string s, int base) {
    long long sum = 0;
    for (char c : s)
        sum = sum * base + charToValue(c);
    return sum;
}
```

### 解码（逐位取模，从后往前）

```cpp
string decode(long long x, int base, int len) {
    string s(len, ' ');
    for (int i = len - 1; i >= 0; i--) {
        int v = x % base;   // 最低位先被取出来
        x /= base;
        s[i] = valueToChar(v);
    }
    return s;
}
```

**关键记忆：** 个位是最低权重（base^0），所以第一个被 `%` 出来。

### 混合进制解码

如果不同位有不同进制（如低 3 位是 26，往上 1 位是 10，再往上 1 位是 36）：
```
value → %26, %26, %26, %10, %36, 剩余
```

**判断拆解顺序的方法：** 看原编码公式里哪个字符乘的是 26^0——它第一个被 %。

### 散列值不可逆

CSP 编码题中，散列/哈希值通常**无法数学反推**，必须维护历史表：
```cpp
unordered_map<int, string> hashTable;
// 每遇到直接出现的代码 → 算散列加入 hashTable
// 解码遇到散列值 → 查 hashTable，查不到输出 "###"
```

## ▶ 真题 1：短消息解码

**来源：第 38 次认证第 3 题（简化版）**

72 位二进制消息，分两种类型。需要按位提取字段、38 进制解码普通代码、混合进制解码典型代码。

**思路框架：**
1. 封装 `getBits(s, start, len)` 提取位字段
2. 先写好编码/解码正反函数
3. 混合进制按 `26,26,26,10,36` 顺序拆
4. 维护散列历史表（12 位和 25 位两张表）
5. **注意处理顺序：** 先解当前消息 → 输出 → 再把本消息的代码加入历史
