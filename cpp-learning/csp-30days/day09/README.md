# Day 9：小状态空间预处理

## ▶ CSP 考情分析

**"先算所有状态，再 O(1) 回答查询"** 是 CSP 第 2-4 题中一个重要的优化策略。当状态总数很小（≤ 10^6）但查询次数可能很多时，预处理可以显著降低总复杂度。

**CSP 中考察预处理的信号：**
- 输入数据范围很小（如 0~511、9 位二进制）
- 每个询问都要执行相同的变换序列
- 变换操作有规律可复用

| 真题 | 状态空间 | 做法 |
|------|---------|------|
| 状态预处理(第40次T2) | 512 种 | 对每种状态预计算变换结果 |
| 优秀的拆分(2020-J T1) | 2^24 内 | 预处理 2 的幂 |
| 收集卡牌(202109-4) | 2^16 | 状压 DP + 记忆化 |

## ▶ 核心技巧

### 预处理的本质

```cpp
// 不用预处理（每次查询都完整模拟）：
for each query:
    result = initial
    for each operation: result = transform(result, op)
// 复杂度：O(Q × M)

// 用预处理（所有状态提前算好）：
for each possible state s:
    ans[s] = apply_all_operations(s)
// 预处理复杂度：O(States × M)，之后每个查询 O(1)
```

**判断依据：** 状态数 × 操作数 < Q × M 时，预处理更优。

### CSP 常见套路

1. **枚举所有状态：** 一个简单的 for 循环遍历整个状态空间
2. **倒序应用操作：** 如果题目要求"依次逆序应用"，预处理时也要倒序
3. **查表输出：** 预处理好后，查询直接输出 `ans[input]`

## ▶ 真题：小状态空间预处理

**来源：第 40 次认证第 2 题**

有 n 个输入数（每个 ≤ 511），m 个操作（每个用一个整数 k 表示）。每个输入数需要逆序应用全部 m 个操作。定义：
- f(x, k) = ((x² + k²) & 7) ^ k
- g(x, k)：将 x 拆成三段(b,q,r)，经 f 变换后重组

**输入：** n m + m 个 k + n 个数（0~511）
**输出：** n 个处理后的结果

**思路：** 状态只有 512 个。对每个 state ∈ [0, 511]，倒序应用 m 次 g 函数，存入 `ans[state]`。查询时直接输出 `ans[num]`。

**CSP 考察点：** 不是考你算法，而是考你"是否意识到状态空间很小"——很多人会逐个输入去模拟 m 次操作，导致 O(n×m) 超时。

## ▶ 相似真题扩展

**优秀的拆分（2020 CSP-J 第 1 题）**

判断一个数能否拆成若干个不同的 2 的幂之和。如果能，从大到小输出这些 2 的幂。

**思路：** 预处理 2^0 到 2^24，从大到小判断 n 是否 ≥ 当前幂。

**CSP 考察点：** 预计算常用值（2 的幂次表），避免反复计算 `pow(2, i)`。
