# Day 16：栈与表达式求值

## ▶ CSP 考情分析

栈在 CSP 中主要出现在两个场景：
1. **表达式求值/解析**（第 3 题难度）：如 逻辑表达式(2022-J T3)、梯度求解(202306-3)
2. **括号匹配/层级嵌套**（第 2 题难度）：如 CSP 中考过的各类嵌套格式解析

CSP 考察栈的核心不是"会不会用 stack 容器"，而是**能不能把题目描述的嵌套规则翻译成栈的 push/pop 逻辑**。

**CSP 栈题的典型特征：** 题目中有"嵌套"、"层级"、"先处理内部再处理外部"的描述。

## ▶ 核心技巧

### 栈的基本操作

```cpp
#include <stack>
stack<int> stk;
stk.push(x);     // 入栈
stk.pop();       // 出栈（不返回值）
stk.top();       // 查看栈顶
stk.empty();     // 判空
```

### 表达式求值模板

```cpp
// 遇到操作数 → 入栈
// 遇到运算符 → 弹出栈顶 1-2 个，计算后压回
stack<int> nums;
for (char c : expr) {
    if (c >= '0' && c <= '9') {
        nums.push(c - '0');
    } else {
        int b = nums.top(); nums.pop();
        int a = nums.top(); nums.pop();
        if (c == '+') nums.push(a + b);
        if (c == '*') nums.push(a * b);
    }
}
```

### 后缀表达式（逆波兰式）

CSP 中表达式的考察通常用后缀表达式（运算符在操作数之后），因为不需要处理括号优先级。

```
中缀：3 + 4 × 2   →   后缀：3 4 2 × +
```

## ▶ 真题 1：逻辑表达式

**来源：2022 CSP-J 第 3 题（简化描述）**

给定一个逻辑表达式，包含 `&`（与）、`|`（或）、`0`、`1`。`&` 优先级高于 `|`。同时统计"短路"次数：当 `a & b` 中 a=0 时可以跳过 b，`a | b` 中 a=1 时可以跳过 b。

**思路：**
1. 中缀表达式转后缀（处理优先级：`&` > `|`）
2. 后缀表达式求值
3. 在求值过程中统计短路次数

**CSP 考察点：** 表达式求值是 CSP 第 3 题的经典题材，2023 年 CSP-J T3 和 CSP 认证的梯度求解(202306-3)都考了。

## ▶ 真题 2：梯度求解

**试题编号：202306-3（第31次认证第3题）**

给定一个数学表达式的逆波兰表示（含变量 x_i），对指定变量求偏导数值。

**思路：** 后缀表达式求值 + 符号微分。对每个变量维护"值"和"导数值"，按运算符规则同时更新两者。

**CSP 考察点：** 这是栈+微分的结合——CSP T3 的典型难度：不是考单一算法，而是把多个概念组合。
