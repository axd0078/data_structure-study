# Day 21：最短路径（Dijkstra）

## ▶ CSP 考情分析

最短路径是 CSP 第 4-5 题中考察频率最高的图论算法。CSP 不会考裸的最短路，而是**包装一层"实际场景"让你建模成图**。

**CSP 最短路题的特点：**
- 不是直接给图，需要你从题目描述中提取节点和边
- 常涉及"拆点"技巧（同一个物理位置在不同状态下是不同的节点）
- 有时是最短路变体：最长路、第 k 短路、瓶颈路

| 真题 | 算法 | 技巧 |
|------|------|------|
| 交通规划(201609-4) | Dijkstra | 最短路树 |
| 行车路线(201712-4) | Dijkstra 拆点 | 按"最后走的小路长度"拆状态 |
| 317号子任务(201903-5) | SPFA/多源 BFS | 按星球类型分类 |
| 星际网络II(202303-4) | Floyd | 全源最短路 |

## ▶ 核心技巧

### Dijkstra 模板（优先队列）

```cpp
vector<pair<int,long long>> G[MAXN]; // {邻居, 距离}
long long dist[MAXN];
bool done[MAXN];

// 初始化
fill(dist, dist + n + 1, LLONG_MAX);
dist[start] = 0;
priority_queue<pair<long long,int>> pq; // {距离, 节点}，注意是负距离
pq.push({0, start});

while (!pq.empty()) {
    int u = pq.top().second; pq.pop();
    if (done[u]) continue;
    done[u] = true;
    for (auto& [v, w] : G[u]) {
        if (dist[u] + w < dist[v]) {
            dist[v] = dist[u] + w;
            pq.push({-dist[v], v});
        }
    }
}
```

### 拆点技巧（CSP 高频）

当"同一个位置在不同状态下代价不同"时，把 (位置, 状态) 作为一个整体建节点。

例：行车路线中，从 u 到 v 可能走大路或小路。小路连续走会影响疲劳值。建点 `(city, lastTrail)` 表示"到达 city 时，最后连续走的小路长度为 lastTrail"。

## ▶ 真题 1：交通规划

**试题编号：201609-4（第11次认证第4题）**

n 个路口，m 条道路。从首都（1号）到每个城市的最短路径是 d[i]。现在要修建一些道路，使所有城市到首都的最短距离保持不变，同时总长度最小。

**思路：** 用 Dijkstra 求最短路时，同时记录到达每个节点的"前驱边中长度最小的"。对每个节点，选择那个让它 `dist[v] = dist[u] + w` 的最小 w。

**CSP 考察点：** 最短路树（Shortest Path Tree）——所有最短路的并集，选边时取最小边权。

## ▶ 真题 2：行车路线

**试题编号：201712-4（第8次认证第4题）**

n 个路口，m 条路（分大路和小路）。走大路不累，走小路产生疲劳值 = 连续走的小路长度之和的平方。求从 1 到 n 的最小疲劳值。

**思路：** 拆点。状态 = (路口, 连续小路长度)。对每条边：
- 大路：dist[v][0] = min(dist[u][any] + w)
- 小路：dist[v][len+w] = min(dist[u][len] + (len+w)² - len²)

**CSP 考察点：** 这是 CSP 最短路的经典难题——连续小路的代价不是独立的，需要记录"连续走了多长"这个额外状态。

## ▶ 扩展：第 k 短路（次短路）

CSP 2024-S 考察了次短路。基本思路：Dijkstra 中每个节点维护两个距离（最短路和次短路），松弛时两者都要更新。
