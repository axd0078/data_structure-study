# Day 3：基础模拟

## ▶ CSP 考情分析

**模拟是 CSP 最重要的题型类别，没有之一。** 从第 1 次到第 41 次认证，每次考试的第 1-3 题中**至少有一道模拟题**。

CSP 模拟题的出题思路是：给一段自然语言描述的规则，让你逐条翻译成代码。**CSP 模拟不像算法题需要你知道特定算法，但极其考验代码翻译的精准度。**

**CSP 模拟题的难度分层：**
- T1 级：单条规则，10-20 行。如 移动(36 次)、报数(201912-1)
- T2 级：多条规则 + 状态切换。如 跳一跳(201803-1)、数组推导(202109-1)
- T3 级：复杂规则 + 大量业务逻辑。如 DHCP 服务器(202104-3)、登机牌条码(202112-3)

## ▶ 核心技巧：模拟三要素

```
1. 定义状态变量 → 题目里什么东西会变？
   位置 → int x, y
   分数 → int score
   方向 → int dir

2. 逐条翻译规则 → if (指令 == 'f') 往前走一步

3. 边界检查 → 越界了怎么办？跳过？循环？
```

**CSP 模拟题最容易翻车的地方：** 状态变量更新时机错了。必须先判断 → 再更新，不能反过来。

## ▶ 真题 1：移动

**来源：第 36 次认证第 1 题**

n×n 网格，起点 (x,y)，执行 f(前)/b(后)/l(左)/r(右) 指令。如果移动后越界则忽略该指令。

| 输入样例 | 输出样例 |
|---------|---------|
| 5 4\n2 3\nf f r f | 3 4 |

说明：起点 (2,3) → f → (2,4) → f → (2,5) → r → (3,5) → f → (3,4)

**思路：** 写 `inBounds(x, y)` 判断越界，对每个指令先算新坐标，不越界才更新。

**CSP 套路：** 网格 + 方向指令 + 边界判断——这种题在 CSP 里反复出现。

## ▶ 真题 2：报数

**试题编号：201912-1（第19次认证第1题）**

4 人从 1 开始轮流报数，7 的倍数或含数字 7 的数跳过，报够 n 个数为止。统计每人跳过次数。

| 输入样例 | 输出样例 |
|---------|---------|
| 20 | 2\n1\n1\n0 |

**思路：**
1. 写 `skip(x)` 判断是否含 7 或是 7 的倍数
2. 维护 `cnt[4]`，遇到跳过 → `cnt[i % 4]++`，否则 `reported++`
3. `reported == n` 时结束

**CSP 考察点：** 含数字判断 + 多人轮流状态 + 双重计数（"报出的数"和"总尝试的数"是两回事）。

## ▶ 真题 3：数组推导（进阶）

**试题编号：202109-1（第23次认证第1题）**

给定一个非降序数组 B，B[i] = max(A[1..i])，求 A 元素和的最大值与最小值。

| 输入样例 | 输出样例 |
|---------|---------|
| 6\n0 0 5 5 10 10 | 30 15 |

最大和：每个 B[i] 都取满（sum = ΣB[i]）。最小和：B[i] 与 B[i-1] 相同时取 0，不同时取 B[i]。

**CSP 考察点：** 这道题是"模拟+分析"的结合——不需要真的构造 A，直接根据 B 的规则推导最值。
