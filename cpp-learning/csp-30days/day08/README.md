# Day 8：广度优先搜索 BFS

## ▶ CSP 考情分析

BFS 在 CSP 中是 **第 4 题和第 5 题的核心工具**，尤其在以下场景反复出现：

| 考察方向 | CSP 真题 |
|---------|---------|
| 最短步数 / 最少操作 | 游戏(201604-4)、马步可达(第38次T2) |
| 连通性 / 可达性 | I'm stuck(201312-5)、DFS+BFS 双搜 |
| 分层 / 按距离扩展 | 网络延时(201503-4)、树的直径 |
| BFS + 状态拆点 | 行车路线(201712-4) |

**CSP 为什么考 BFS？** 因为 CSP 第 4 题的定位是"中等难度算法"，BFS 的难度正好匹配——需要建模能力，但算法本身不复杂。

**CSP BFS 题的典型特征：** 网格 + 方向移动 + 最少步数 → 用 BFS，不是 DFS。

## ▶ 核心技巧

### BFS 标准模板

```cpp
queue<State> q;
bool visited[MAXN][MAXN] = {false};
// 方向数组：4 方向或 8 方向（马走日）
int dx[] = {-1, 1, 0, 0};
int dy[] = {0, 0, -1, 1};

q.push(start);
visited[start.x][start.y] = true;

while (!q.empty()) {
    State cur = q.front(); q.pop();
    for (int d = 0; d < 4; d++) {
        int nx = cur.x + dx[d];
        int ny = cur.y + dy[d];
        if (inBounds(nx, ny) && !visited[nx][ny]) {
            visited[nx][ny] = true;  // ⚠️ 入队时立刻标记！
            q.push({nx, ny, cur.step + 1});
        }
    }
}
```

**CSP 最大坑：** visited 标记的位置。如果在 `q.pop()` 时标记，会有大量重复入队导致超时。**入队即标记。**

## ▶ 真题 1：马步可达

**来源：第 38 次认证第 2 题**

n×n 棋盘，马走日（8 个方向），求 k 步内能到达的不同格子数（含起点）。

| 输入样例 | 输出样例 |
|---------|---------|
| 8 1\n4 4 | 9 |

起点 + 8 个方向的格子 = 9 个。

**马走日 8 方向：** `(±2, ±1)` 和 `(±1, ±2)`

**思路：** 队列存 (x, y, step)，只有当 step < k 时才继续扩展新格子。

## ▶ 真题 2：游戏（经典 T4）

**试题编号：201604-4（第7次认证第4题）**

n×m 网格，有些格子危险（在 b 时间段内不能走），从 (1,1) 到 (n,m) 最少时间。

**思路：** BFS + 时间维度拆点。visited[x][y][t] 表示在时刻 t 是否访问过 (x,y)。由于危险时间段只到 10^4，可以按时间轴 BFS。

**CSP 考察点：** 这道题是 BFS 的状态拓展——从二维 (x,y) 拓展到三维 (x,y,t)。所有"随时间变化"的 BFS 题都可以用这思路。

## ▶ 真题 3：I'm stuck（经典 T5）

**试题编号：201312-5（第1次认证第5题）**

R×C 网格，每个格子有方向（上下左右），人物按方向移动。起点 S 可到达一些格子。问有多少个格子满足：从 S 可达该格子，且从该格子无法到达终点 T。

**思路：** 两次 BFS：
1. 从 S 正向 BFS 找到所有可达点
2. 从 T 反向 BFS（沿方向的反方向）找到所有能到 T 的点
3. 计数：正向可达 ∧ 反向不可达

**CSP 考察点：** 正反两次 BFS + 边的反向建图——这是 CSP 图论题的基础套路。
