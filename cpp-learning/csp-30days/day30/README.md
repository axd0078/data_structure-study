### 考前 checklist

```
□ 数组开 N+5（多开几个防边界）
□ long long 防止 int 溢出
□ ios::sync_with_stdio(false); cin.tie(nullptr);
□ 全局数组 > 10^6 时开全局变量区（不在 main 里）
□ string 操作：注意 find() 返回 string::npos
□ vector 下标：size() 返回的是 size_t（无符号），与 0 比较安全
□ 输出格式：空格 vs 换行，最后有无空行
□ 边界测试：n=1, n=0, n=最大值
□ 样例测试：至少 3 组（题面样例 + 自己造的 + 边界）
```

### CSP 常见翻车原因（来自真题分析）

1. **状态更新顺序反了** —— 应先判断旧状态再更新
2. **循环变量的范围** —— 从 0 还是 1，到 n 还是 n-1
3. **int 溢出** —— 乘法、平方、大数累加需要 long long
4. **cin 后 getline 多读换行** —— 要用 cin.ignore()
5. **浮点精度** —— 用 double 而非 float，输出保留精度
6. **BFS 标记时机** —— 入队即标记，不能弹出时标记
7. **DP 循环方向** —— 0/1 背包 j 从大到小
8. **组合数取模忘了** —— CSP 有时不要求取模但数很大

### 偷分策略（T4-T5 部分分的获取）

- **暴力枚举**：n 小但有满分数据 → 对 n ≤ 10 写暴力拿部分分
- **子任务拆解**：T4 通常有子任务标注 → 分析每个子任务做针对性写法
- **二分答案**：即使不会正解，二分+暴力验证通常能多拿 10-20 分
- **特殊性质利用**：题目常给特殊性质（链/树/无环/升序），针对性质简化代码
