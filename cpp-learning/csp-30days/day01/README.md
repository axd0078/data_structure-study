# Day 1：数组计数与哈希思想

## ▶ CSP 考情分析

CSP 第 1 题（最简单的一题）在 41 次认证中，**至少有 8 次直接考察了"统计/计数"能力**。这也是 CSP 最基础但必考的能力点。

**CSP 中的计数题分两类：**

| 类型 | 典型真题 | 做法 |
|------|---------|------|
| 值域小（≤10^6） | 门禁系统(201412-1)、数列分段(201509-1) | 数组计数 `cnt[x]++` |
| 值域大/需要快速查找 | 相反数(201403-1)、词频统计(202403-1) | `unordered_set` / `unordered_map` |

**CSP 第 1 题的考察重点不是你"会不会写循环"，而是——能不能边读边处理，不额外开 O(n) 的空间存输入数据。**

## ▶ 核心技巧

### 数组计数（CSP 最高频用法）

当数据范围小（比如值 ≤ 10000），直接用数组做"值→次数"的映射：

```cpp
int cnt[10010] = {0};  // 下标 = 数的值
for (int i = 0; i < n; i++) {
    int x; cin >> x;
    cnt[x]++;
}
```

CSP 真题中值域通常 ≤ 1000（门禁系统）、≤ 10000（数列分段），数组足够。

### 哈希表（值域大的备选）

当值域到 10^9 或只需要判断"是否存在"：

```cpp
#include <unordered_set>
unordered_set<int> s;
s.insert(x);    // 插入
s.count(x);     // 1=存在, 0=不存在
```

**选择口诀：值小用数组（O(1) 无哈希开销），值大/需要去重用哈希。**

## ▶ 真题 1：门禁系统

**试题编号：201412-1（第3次认证第1题）**

每位读者有一个编号，按来访顺序输出每条记录中该读者是第几次出现。

**输入：** n + n 个整数（读者编号）
**输出：** n 个整数（第几次出现）

| 输入样例 | 输出样例 |
|---------|---------|
| 5\n1 2 1 1 3 | 1 1 2 3 1 |

**思路：** 开 `cnt[1010]`，读一个 `x` → `cnt[x]++` → 输出 `cnt[x]`

**CSP 考察点：** 边读边处理，不是一个读完再遍历的模式

## ▶ 真题 2：相反数

**试题编号：201403-1（第1次认证第1题）**

N 个互不相同的非零整数，问有多少对相反数（a 和 -a）。

| 输入样例 | 输出样例 |
|---------|---------|
| 5\n1 2 3 -1 -2 | 2 |

**思路：**
- 方法一（计数数组）：遍历，`cnt[abs(x)]++`，最后统计 `cnt[i] == 2` 的个数（值域 ≤ 1000 时用）
- **方法二（哈希集合，推荐）：** 遍历每个数 x，查 `s.count(-x)`，有则 ans++，无则把 x 插入

**CSP 考察点：** 这是一道"判断配对"的题。很多 CSP 第 1 题表面上是计数，实际上在考察你用什么数据结构存"已经出现过的值"。

## ▶ 真题 3：词频统计（进阶）

**试题编号：202403-1（第33次认证第1题）**

统计各单词在多少个文档中出现，以及总出现次数。

**CSP 考察点：** 这是第 1 题"进化"的方向——不再是简单的一维计数，而是二维信息（单词→文档×次数）的统计。需要 `map<string, set<int>>` 或者 `unordered_map` + 去重逻辑。
