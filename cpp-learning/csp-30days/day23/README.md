# Day 23：背包问题详解

## ▶ CSP 考情分析

背包问题是 CSP 第 2-4 题中最常考的 DP 模型。CSP 考背包通常不是裸题，而是**"识别出这是背包问题"本身就需要理解**。

**CSP 背包题的包装方式：**
| 包装 | 真题 | 本质 |
|------|------|------|
| "资源分配" | 背包贪心(第41次T2) | 0/1 背包 |
| "货物调度" | 货物调度(202406-4) | 多重背包 + 贪心 |
| "咖啡减时" | 背包贪心(第41次T2) | 0/1 + 分数背包混合 |

**CSP 背包题的识别：** 题目给了一个"总容量/总量"，每个物品有"重量"和"价值"，问如何选择使总价值最大。

## ▶ 核心技巧

### 0/1 背包（每件最多选一次）

```cpp
vector<int> dp(capacity + 1, 0);
for (auto& item : items) {
    for (int j = capacity; j >= item.weight; j--) {
        dp[j] = max(dp[j], dp[j - item.weight] + item.value);
    }
}
// ⚠️ j 从大到小！保证每件物品只用一次
```

### 完全背包（每件可选无限次）

```cpp
for (auto& item : items) {
    for (int j = item.weight; j <= capacity; j++) {
        dp[j] = max(dp[j], dp[j - item.weight] + item.value);
    }
}
// j 从小到大！允许多次使用
```

### 0/1 vs 完全，只差循环方向

| 类型 | j 循环方向 | 原因 |
|------|-----------|------|
| 0/1 | 从大到小 | 防止重复使用 |
| 完全 | 从小到大 | 允许重复使用 |

**CSP 最爱考这个区别。**

## ▶ 真题 1：货物调度

**试题编号：202406-4（第34次认证第4题）**

有 n 个货物，每个有送达时间和重量。需要在限定装载能力下安排运输，使所有货物按时送达。

**思路：** 按送达时间分批，每批内是背包问题（选择哪些货物同批运送）。贪心 + DP 的组合。

## ▶ 真题 2：背包 + 分数贪心

**来源：第 41 次认证第 2 题**

部分任务不可分（背包），部分任务可分（贪心）。枚举分配给背包的资源量，剩余给贪心。

```
for used = 0 to totalBudget:
    ans = max(ans, dp[used] + greedy(totalBudget - used))
```

**CSP 考察点：** 把"混合类型"拆成"枚举分界线 + 两种策略"——这是 CSP 面对混合约束的标准做法。

## ▶ 扩展：多重背包

每件物品有限定的个数。可以二进制拆分（1,2,4,8...）转化为 0/1 背包。

```cpp
// 将 count 个物品拆成 log(count) 组
int k = 1;
while (count >= k) {
    items.push_back({weight * k, value * k});
    count -= k;
    k <<= 1;
}
if (count > 0) items.push_back({weight * count, value * count});
```
