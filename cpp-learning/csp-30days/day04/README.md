# Day 4：字符串处理

## ▶ CSP 考情分析

字符串处理在 CSP 的 **T1-T2 中高频出现**，同时也是 **T3 大模拟题的常见元素**。

CSP 字符串题的出题特点：
- T1 级：数字字符转换（数位之和 201512-1）、简单校验（ISBN 201312-2）
- T2-T3 级：格式化解析（模板生成 201509-3）、编码解码（命令行选项 201403-3、登机牌条码 202112-3）

**CSP 对字符串的考察重点不是算法（KMP 很少考），而是"按格式拆字段"和"字符与数值互相转换"。**

## ▶ 核心技巧

### 字符 ↔ 数字 转换

```cpp
char c = '7';
int d = c - '0';       // d = 7

int d = 7;
char c = d + '0';      // c = '7'

// 遍历字符串中每个数字求和
string s; cin >> s;
int sum = 0;
for (char c : s) sum += c - '0';
```

### 跳过分隔符

当字符串有固定分隔符（如 ISBN 的 `-`）：

```cpp
string s; cin >> s;
for (char c : s) {
    if (c == '-') continue;
    // 处理数字
}
```

### 读一整行 vs 读一个词

```cpp
cin >> s;           // 读一个词（遇空格/换行停）
getline(cin, s);    // 读一整行（含空格）
```

**CSP 常见坑：** `cin >> n` 后 `getline` 之前需要 `cin.ignore()` 吃掉换行符。

## ▶ 真题 1：数位之和

**试题编号：201512-1（第6次认证第1题）**

输入整数 n（可达 10^9），输出各位数字之和。

| 输入样例 | 输出样例 |
|---------|---------|
| 20151220 | 13 |

**两种写法：**
- 字符串法：`cin >> s`，遍历字符 `sum += c - '0'`
- 整数法：`while (n) { sum += n % 10; n /= 10; }`

**CSP 考察点：** 这题看似简单，但考察了你是否知道 `c - '0'` 这个基础操作。n 可达 10^9，用 int 足够，但如果在第 2-3 题出现大数就需要 long long。

## ▶ 真题 2：ISBN 号码

**试题编号：201312-2（第3次认证第2题）**

校验 ISBN：格式 `x-xxx-xxxxx-x`，前 9 位乘 1-9 加权和 % 11 = 识别码（余数 10 用 X 表示）。判断输入是否正确，不正确则输出正确的。

| 输入样例 | 输出样例 |
|---------|---------|
| 0-670-82162-4 | Right |
| 0-670-82162-0 | 0-670-82162-4 |

**思路：**
1. 读入字符串，跳过 `-`，前 9 位加权求和
2. `code = sum % 11`，10 → 'X'
3. 比较最后一位，不同则替换后输出

**CSP 考察点：** 跳过分隔符 + 字符数字互转 + 特殊值 X 的处理。这是 CSP 第 2 题的典型难度。

## ▶ 真题 3：模板生成系统（选做）

**试题编号：201509-3（第5次认证第3题）**

给定模板文本（含 `{{ 变量名 }}` 占位符），根据给定变量值替换所有占位符输出。

这是 CSP **第 3 题大模拟的代表作**——考察字符串查找替换、map 存变量、处理边界情况（变量不存在、嵌套引用）。
