# 第 5 课：STL 基础容器、顺序维护与模拟

## 本课目标

这一课不只是学容器名字，而是学会一件更重要的事：

> 当题目要求你维护“先后顺序、进出顺序、当前最值”时，应该立刻想到容器。

本课要熟练四种容器：

- `stack`
- `queue`
- `deque`
- `priority_queue`

## 1. stack

栈是后进先出。

```cpp
stack<int> st;
st.push(1);
st.push(2);
cout << st.top() << endl; // 2
st.pop();
```

常见场景：

- 括号匹配
- 表达式处理
- 撤销操作
- 从“最近压入”的东西开始回退

## 2. queue

队列是先进先出。

```cpp
queue<int> q;
q.push(1);
q.push(2);
cout << q.front() << endl; // 1
q.pop();
```

常见场景：

- 排队服务
- 宽搜
- 事件按到达顺序处理

## 3. deque

双端队列两头都能进出。

```cpp
deque<int> dq;
dq.push_back(1);
dq.push_front(2);
dq.pop_back();
dq.pop_front();
```

## 4. priority_queue

优先队列默认是大根堆，也就是最大值先出。

```cpp
priority_queue<int> pq;
pq.push(3);
pq.push(10);
cout << pq.top() << endl; // 10
```

如果要最小值先出，写成：

```cpp
priority_queue<int, vector<int>, greater<int>> pq;
```

## 容器题最容易错的地方

- 变量重名，比如 `string s; stack<char> s;`
- 忘记判断空容器就 `top()` / `front()`
- 以为 `pop()` 会返回值
- 分不清“当前元素”和“已经存进去的顺序结构”

## 本课练习

### 练习 1：括号匹配

输入一个只包含 `(`、`)`、`[`、`]` 的字符串，判断括号是否匹配。匹配输出 `yes`，否则输出 `no`。

### 练习 2：排队服务

输入若干操作。`IN x` 表示编号 `x` 的人入队，`OUT` 表示队首出队并输出编号。如果队列为空时 `OUT`，输出 `empty`。

### 练习 3：取最小任务

输入若干任务耗时，每次取出当前耗时最小的任务并输出，直到全部输出。

## 长题面训练方向

这一课最适合做一类题：

- 有明显的先后顺序
- 状态会随着操作变化
- 题面会反复提到“当前最上面”“当前最前面”“当前最小值”

做这种题时，先别急着写代码，先写下面 4 行：

- 输入是什么
- 输出是什么
- 哪个量在变化
- 我准备用什么容器维护顺序

做完后你要能回答：

1. 这题最核心的状态量是什么？
2. 你第一次写错的地方在哪？
3. 如果我让你口头讲题意，你会怎么用 3 句话讲清楚？
