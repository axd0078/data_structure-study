# 第 3 课：vector、数组替代与前缀和意识

## 本课目标

你已经会写最基本的数组读入了，这一课不再只讲“怎么声明 `vector`”，而是讲三件更重要的事：

1. 什么时候该用 `vector`。
2. 什么时候该边读边统计，不必真的存整个数组。
3. 什么时候应该顺手预处理前缀和。

## 为什么刷题更推荐 vector

`vector` 比手写数组更适合做题和练习，原因不是“高级”，而是更不容易出边界问题：

- 长度由数据决定，不需要提前拍脑袋估上限。
- 可以直接传给函数。
- 可以和 `sort`、`reverse`、`lower_bound` 一起用。
- 二维 `vector` 比动态二维数组更稳。

最常见写法：

```cpp
int n;
cin >> n;
vector<int> a(n);
for (int i = 0; i < n; i++) {
    cin >> a[i];
}
```

## 三种常见写题方式

### 1. 必须存数组

后面还要多次访问、排序、查找、做前缀和，就要存：

```cpp
vector<int> a(n);
```

### 2. 边读边统计

如果题目只要求总和、最大值、正数个数，就没必要一定存：

```cpp
int ans = 0;
for (int i = 0; i < n; i++) {
    int x;
    cin >> x;
    if (x > 0) ans++;
}
```

### 3. 存数组并同时维护信息

很多题目最稳的写法不是“先读，再算”，而是边读边更新：

```cpp
vector<int> a(n);
int maxPos = 0;
for (int i = 0; i < n; i++) {
    cin >> a[i];
    if (a[i] > a[maxPos]) {
        maxPos = i;
    }
}
```

## 范围 for 和引用

只读：

```cpp
for (int x : a) {
    cout << x << " ";
}
```

修改原数组：

```cpp
for (int &x : a) {
    x *= 2;
}
```

这里的 `&` 很关键，不写 `&` 只是改副本。

## 二维 vector

```cpp
int n, m;
cin >> n >> m;
vector<vector<int>> grid(n, vector<int>(m));
```

这类写法适合：

- 矩阵输入
- 网格题
- 需要按行列访问的数据

## 前缀和意识

如果题目后面会多次问区间和，不要每次重新累加，而是先做一遍预处理：

```cpp
vector<int> prefix(n + 1, 0);
for (int i = 1; i <= n; i++) {
    prefix[i] = prefix[i - 1] + a[i - 1];
}
```

这样区间 `[l, r]`（下标从 1 开始）就可以 O(1) 求和：

```cpp
prefix[r] - prefix[l - 1]
```

这一课你不一定立刻用前缀和解题，但要先形成这种意识。

## 这课最容易错的地方

- `int ans;` 没初始化就直接累加或计数。
- `vector<int> a; a[0] = 1;`，此时还没有元素。
- `a[n]` 越界，最后一个元素是 `a[n - 1]`。
- 最大值位置题里把“第一次出现”写成了“最后一次出现”。
- 二维输入时把 `n` 和 `m` 的循环顺序写反。
