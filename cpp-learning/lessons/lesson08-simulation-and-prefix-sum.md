# 第 8 课：模拟、前缀和与多步骤规则

## 本课目标

掌握两类高频问题：

1. 规则多、状态会变化的模拟题
2. 频繁求闭区间和的问题

## 模拟题思路

模拟题不是套复杂算法，而是把题目规则翻译成代码：

1. 明确状态变量
2. 按输入顺序更新状态
3. 每次更新后检查边界条件
4. 按题目格式输出

常见状态变量：分数、余额、当前位置、方向、计数器、当前模式。

## 前缀和

如果要频繁求区间和，可以先预处理。建议使用 1-based 写法：

```cpp
vector<long long> a(n + 1), prefix(n + 1, 0);
for (int i = 1; i <= n; i++) {
    cin >> a[i];
    prefix[i] = prefix[i - 1] + a[i];
}
```

如果题目查询的是 1-based 闭区间 `[l, r]`，也就是包含 `l` 和 `r`，区间和为：

```cpp
prefix[r] - prefix[l - 1]
```

这不是左开右闭，也不是左闭右开。输入给你的 `[l, r]` 是闭区间；公式里的 `prefix[l - 1]` 只是为了扣掉左边不属于查询范围的部分。

## 本课练习

### 练习 1：累计得分

输入 `n` 和 `n` 次得分变化，输出最终得分。得分变化可以是小数。

测试输入：

```text
5
3 -2 10 0.5 -1.5
```

测试输出：

```text
10
```

### 练习 2：区间和查询

输入 `n`、`q` 和一个长度为 `n` 的整数数组。接下来 `q` 行，每行给出 `l r`，表示查询 1-based 闭区间 `[l, r]` 的元素和，输出每次查询结果。

测试输入：

```text
5 4
1 2 3 4 5
1 3
2 5
3 3
1 5
```

测试输出：

```text
6
14
3
15
```

### 练习 3：余额模拟

第一行输入初始余额 `money`，第二行输入操作数 `n`。接下来 `n` 行，每行两个整数 `op change`：

- `op = 1` 表示收入，余额增加 `change`
- `op = 0` 表示支出，若支出后余额小于 `0`，本次支出失败，余额不变

输出最终余额和失败次数。

测试输入：

```text
10
5
1 5
0 3
0 20
1 4
0 10
```

测试输出：

```text
6 1
```

## 复盘点

- 区间题必须先确认下标从 0 还是 1 开始。
- 闭区间 `[l, r]` 包含两个端点。
- 前缀和推荐开 `n + 1`，让 `l = 1` 时公式仍然统一。
