# Day 018：集合判定与二进制计数

## 学习内容

Day 018 是第一阶段 T1 的收束。两道题分别训练“真假关系是否一致”和“只看有效二进制位”。这两类题都不难，但很容易因为读题顺序或定义细节丢分。

**集合 (set)** 要判断小 C 的异或哈希做法在每个询问中是否正确。正确不是问 `S` 和 `T` 是否相等，而是问“小 C 用 `f(S)==f(T)` 得出的结论”是否和真实集合相等一致。也就是：

```text
(S == T) == (f(S) == f(T))
```

一致输出 `correct`，不一致输出 `wrong`。题面先输入所有 `S`，再输入所有 `T`，所以必须先保存每个 `S` 的元素序列和异或值。

**平衡数**要统计二进制表示里 `0` 和 `1` 个数相等的正整数。这里的二进制表示从最高位的 `1` 开始，不考虑更高位补零。因此只需要在 `while (x > 0)` 里统计每一位。

今天的最低复杂度目标：

- 集合 (set)：`O(输入元素总数)` 时间，保存所有 `S` 需要 `O(S总大小)` 空间。
- 平衡数：`O(n log A)` 时间，`O(1)` 空间。

## 题解

### 第40次 T1 集合 (set)

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给定长度为 `n` 的序列 `a`。对集合 `S` 定义：

```text
f(S) = xor(a_i), i in S
```

每个询问给出两个集合 `S_i,T_i`，需要判断小 C 使用 `f(S_i)==f(T_i)` 来判断集合相等是否会得出正确结论。

**知识点**

真实集合相等可以直接比较元素序列，因为题目保证每个集合内部严格递增。小 C 的判断结果是异或值是否相等。只要两个布尔值一致，小 C 就“正确”：

```text
same_set = (S_i == T_i)
same_xor = (xor(S_i) == xor(T_i))
answer = same_set == same_xor
```

**解法**

1. 读入 `n,m` 和序列 `a`。
2. 读入所有 `S_i`：
   - 保存元素序列。
   - 同时计算 `xor(S_i)`。
3. 再读入所有 `T_i`：
   - 计算当前 `T_i` 的元素序列和异或值。
   - 比较真实相等和异或相等是否一致。
4. 一致输出 `correct`，否则输出 `wrong`。

**易错点**

- 输入顺序是所有 `S` 在前，所有 `T` 在后，不是一行 `S` 紧跟一行 `T`。
- 输出的是小 C 的做法是否正确，不是简单输出集合是否相等。
- 集合元素严格递增，所以 `vector<int>` 可以直接比较。
- `a_i < 2^31`，异或结果用 `int` 可以，但 `unsigned int` 或 `int` 都要保持一致。

**复杂度**

设输入集合元素总数为 `E`。时间复杂度 `O(E)`，空间复杂度 `O(S总大小)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    vector<int> value(n + 1);
    for (int i = 1; i <= n; ++i) {
        cin >> value[i];
    }

    vector<vector<int>> sets(m);
    vector<int> set_xor(m, 0);
    for (int i = 0; i < m; ++i) {
        int len;
        cin >> len;
        sets[i].resize(len);
        for (int j = 0; j < len; ++j) {
            cin >> sets[i][j];
            set_xor[i] ^= value[sets[i][j]];
        }
    }

    for (int i = 0; i < m; ++i) {
        int len;
        cin >> len;
        vector<int> current(len);
        int current_xor = 0;
        for (int j = 0; j < len; ++j) {
            cin >> current[j];
            current_xor ^= value[current[j]];
        }

        bool same_set = (sets[i] == current);
        bool same_xor = (set_xor[i] == current_xor);
        cout << (same_set == same_xor ? "correct" : "wrong") << '\n';
    }

    return 0;
}
```

### 第41次 T1 平衡数（balance）

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

一个正整数的二进制表示中，如果 `0` 的个数和 `1` 的个数相等，它就是平衡数。输入 `n` 个正整数，统计其中平衡数数量。

**知识点**

二进制表示不包含前导零，所以只统计 `x` 变成 0 之前经过的位。每次看最低位：

```text
x & 1 == 1  ->  当前位是 1
x & 1 == 0  ->  当前位是 0
```

然后右移一位 `x >>= 1`。

**解法**

1. 对每个数初始化 `zero=0, one=0`。
2. 当 `x > 0`：
   - 如果最低位是 1，`one++`；否则 `zero++`。
   - 右移一位。
3. 判断 `zero == one`。
4. 统计满足条件的个数。

**易错点**

- 不考虑机器字长补出来的前导零。
- 输入保证是正整数，不需要处理 0。
- `10^9` 的二进制位数只有 30 位左右，逐位处理足够快。

**复杂度**

时间复杂度 `O(n log A)`，其中 `A` 是最大输入值；额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

bool is_balanced(int x) {
    int zero = 0;
    int one = 0;

    while (x > 0) {
        if (x & 1) {
            ++one;
        } else {
            ++zero;
        }
        x >>= 1;
    }

    return zero == one;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    int answer = 0;
    while (n--) {
        int x;
        cin >> x;
        if (is_balanced(x)) {
            ++answer;
        }
    }

    cout << answer << '\n';
    return 0;
}
```
