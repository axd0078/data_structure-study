## 题解

### 第26次 T2 寻宝！大冒险！

**题意抓手**

大地图边长可达 `1e9`，只给出树的坐标。藏宝图是 `(S+1)x(S+1)` 的 01 模式，问它能在大地图中匹配多少个左下角位置。匹配要求模式内每个格子的 0/1 都和大地图对应区域一致。

**知识点**

稀疏点集匹配。大图只需要支持“某坐标是否有树”的查询，用 `set<pair<long long,long long>>` 即可。由于藏宝图左下角一定为 1，候选左下角只可能是已有树坐标。

**解法**

读入所有树坐标并放入集合。藏宝图输入是从上到下给的：第 1 行是 `B[S][0..S]`，最后一行才是 `B[0][0..S]`，读入时要倒着存。枚举每棵树 `(x,y)` 作为候选，如果 `x+S>L` 或 `y+S>L` 则越界跳过。否则枚举 `0<=i,j<=S`，检查 `(x+i,y+j)` 是否有树与 `B[i][j]` 是否为 1 完全一致。

**易错点**

- 藏宝图输入行顺序和坐标中的第一维是反的，要倒序存。
- 不仅要检查 `B=1` 的位置有树，还要检查 `B=0` 的位置没有树。
- 候选区域不能超过大地图边界。

**复杂度**

候选最多 `n` 个，每个候选检查 `(S+1)^2` 个格子，集合查询 `O(log n)`。时间复杂度 `O(n S^2 log n)`，空间复杂度 `O(n+S^2)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, S;
    long long L;
    cin >> n >> L >> S;

    vector<pair<long long, long long>> trees(n);
    set<pair<long long, long long>> has_tree;
    for (auto& [x, y] : trees) {
        cin >> x >> y;
        has_tree.insert({x, y});
    }

    vector<vector<int>> pattern(S + 1, vector<int>(S + 1));
    for (int row = S; row >= 0; --row) {
        for (int col = 0; col <= S; ++col) {
            cin >> pattern[row][col];
        }
    }

    int ans = 0;
    for (auto [x, y] : trees) {
        if (x + S > L || y + S > L) {
            continue;
        }
        bool ok = true;
        for (int i = 0; i <= S && ok; ++i) {
            for (int j = 0; j <= S; ++j) {
                bool tree = has_tree.count({x + i, y + j});
                if (tree != (pattern[i][j] == 1)) {
                    ok = false;
                    break;
                }
            }
        }
        ans += ok;
    }

    cout << ans << '\n';
    return 0;
}
```

### 第27次 T2 何以包邮？

**题意抓手**

从购物车中保留若干本书，使总价 `m >= x`，并让 `m` 尽量小。题目保证所有书总价至少为 `x`。

**知识点**

0/1 背包可达性。`dp[s]` 表示能否选出总价 `s`。每本书只能选一次，所以金额循环要从大到小。

**解法**

计算所有书总价 `total`，初始化 `dp[0]=true`。对每本书价格 `a`，倒序枚举已有金额 `s`，若 `dp[s]` 为真，则 `dp[s+a]=true`。最后从 `x` 到 `total` 找第一个可达金额输出。

**易错点**

- 金额循环必须倒序，否则同一本书会被重复使用。
- 输出的是最小可达总价，不是删去的价格。
- `n` 小但总价范围也很友好，背包比枚举 `2^n` 更稳。

**复杂度**

总价最多 `3e5`，时间复杂度 `O(n * total)`，空间复杂度 `O(total)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, x;
    cin >> n >> x;
    vector<int> price(n);
    int total = 0;
    for (int& value : price) {
        cin >> value;
        total += value;
    }

    vector<char> dp(total + 1, 0);
    dp[0] = 1;
    for (int value : price) {
        for (int sum = total - value; sum >= 0; --sum) {
            if (dp[sum]) {
                dp[sum + value] = 1;
            }
        }
    }

    for (int sum = x; sum <= total; ++sum) {
        if (dp[sum]) {
            cout << sum << '\n';
            return 0;
        }
    }

    return 0;
}
```
