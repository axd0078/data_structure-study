# Day 003：数组下标与出现性判断

## 学习内容

Day 003 继续训练 T1 保分题，但今天开始强调“用题目给出的数据范围选最低复杂度”。同样是线性题，能只扫一遍就不要排序；题目给出值域很小，就优先考虑计数桶。

今天三道题的最优复杂度分别是：

- **折点计数**：只需要看连续三天，单次扫描即可，时间 `O(n)`，额外空间 `O(1)`。
- **最大波动**：只比较相邻两天价格差，时间 `O(n)`，额外空间 `O(1)`。
- **中间数**：`ai <= 1000`，用计数桶统计每个值出现次数，时间 `O(n + V)`，其中 `V=1000`，比排序 `O(n log n)` 更低。

做这种题时，先判断答案依赖哪一段信息：

1. 只依赖相邻元素：保留前一个值即可。
2. 依赖长度为 3 的局部窗口：保留 `prev, cur, next` 即可。
3. 依赖全局大小关系且值域很小：用频次数组统计，再做前缀数量。

今天的边界样例重点：

- `n = 1` 或 `n = 2` 时没有折点。
- 最大波动可能为 0，例如所有价格相同。
- 中间数要忽略等于候选值的元素，只比较“小于它”和“大于它”的数量。
- 中间数不存在时输出 `-1`。

## 题解

### 第07次 T1 折点计数

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给出连续 `n` 天的销售量。如果某一天比前一天和后一天都高，或者比前一天和后一天都低，这一天就是折点。第一天和最后一天没有左右两侧，不可能是折点。

**知识点**

折点只由相邻三项决定。判断第 `i` 天是否是折点，只需要 `a[i-1]`、`a[i]`、`a[i+1]`。题目还保证相邻两天销售量不同，因此不用考虑相等时的特殊平台情况。

**解法**

最直接的数组写法是读入整个数组后枚举 `i=1..n-2`。不过这题可以做到额外空间 `O(1)`：读入并维护一个长度为 3 的滑动窗口。

流程：

1. 如果 `n <= 2`，直接输出 0。
2. 先读入 `prev = a1` 和 `cur = a2`。
3. 从第 3 个数开始，每次读入 `next`。
4. 判断 `cur` 是否同时大于两侧，或同时小于两侧。
5. 窗口右移：`prev = cur`，`cur = next`。

这已经是最优时间复杂度，因为每个输入至少要读一次。

**易错点**

- 第一天和最后一天不能算折点。
- 折点是严格大于两边或严格小于两边。
- `n = 1`、`n = 2` 时答案一定是 0。

**复杂度**

时间复杂度 `O(n)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    int prev;
    cin >> prev;
    if (n <= 2) {
        for (int i = 1; i < n; i++) {
            int unused;
            cin >> unused;
        }
        cout << 0 << '\n';
        return 0;
    }

    int cur;
    cin >> cur;
    int ans = 0;

    for (int i = 3; i <= n; i++) {
        int next;
        cin >> next;
        if ((cur > prev && cur > next) || (cur < prev && cur < next)) {
            ans++;
        }
        prev = cur;
        cur = next;
    }

    cout << ans << '\n';
    return 0;
}
```

### 第08次 T1 最大波动

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给出连续 `n` 天的股票收盘价，求相邻两天价格差的绝对值最大是多少。只比较连续两天，不比较任意两天。

**知识点**

这是相邻差值的在线统计。读到第 `i` 天价格时，只需要知道第 `i-1` 天价格，就能更新当前最大波动。

**解法**

1. 读入 `n`。
2. 先读第一天价格 `prev`。
3. 从第二天开始，每读入一个 `cur`，计算 `abs(cur - prev)`。
4. 用这个值更新答案。
5. 设置 `prev = cur`，继续扫描。

这同样是最优复杂度：每个价格必须读一次，除此之外不需要数组。

**易错点**

- 波动只看相邻两天，不是全局最大值减最小值。
- 价格可能相同，最大波动可以为 0。
- `n >= 2`，所以一定能读到第一天和第二天。

**复杂度**

时间复杂度 `O(n)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    int prev;
    cin >> prev;
    int ans = 0;

    for (int i = 1; i < n; i++) {
        int cur;
        cin >> cur;
        ans = max(ans, abs(cur - prev));
        prev = cur;
    }

    cout << ans << '\n';
    return 0;
}
```

### 第09次 T1 中间数

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

在一个整数序列中，如果某个数的“小于它的数的数量”等于“大于它的数的数量”，它就是中间数。等于它的数不参与两边数量比较。如果存在中间数，输出这个值；否则输出 `-1`。

**知识点**

这题看起来可以排序后找中位附近，但题目允许重复值，而且定义是比较严格小于和严格大于的数量。更重要的是，数据范围给出 `1 <= ai <= 1000`，所以计数桶是最低复杂度做法。

计数桶思路：

- `cnt[x]` 表示值 `x` 出现了多少次。
- 从小到大枚举候选值 `x`。
- `less` 维护当前值之前所有数的数量。
- `greater = n - less - cnt[x]`。
- 如果 `less == greater`，输出 `x`。

**解法**

1. 读入 `n`。
2. 开 `cnt[1001]`，统计每个值出现次数。
3. 从 `1` 到 `1000` 枚举候选值。
4. 如果当前值没出现过，跳过。
5. 计算小于和大于它的数量。
6. 相等则输出；枚举完还没有答案则输出 `-1`。

这个做法比排序 `O(n log n)` 更低。由于值域 `V=1000` 是题目常量，总体可以看作线性。

**易错点**

- 等于候选值的元素不算小于，也不算大于。
- 中间数不存在时输出 `-1`。
- 不要只取排序后中间位置。重复值会让“位置中间”和“数量相等”不是一回事。
- 候选值必须在原序列中出现过。

**复杂度**

时间复杂度 `O(n + V)`，其中 `V=1000`；额外空间复杂度 `O(V)`。在本题范围下，这是比排序更低的做法。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<int> cnt(1001, 0);
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        cnt[x]++;
    }

    int less = 0;
    for (int x = 1; x <= 1000; x++) {
        if (cnt[x] == 0) {
            continue;
        }
        int greater = n - less - cnt[x];
        if (less == greater) {
            cout << x << '\n';
            return 0;
        }
        less += cnt[x];
    }

    cout << -1 << '\n';
    return 0;
}
```
