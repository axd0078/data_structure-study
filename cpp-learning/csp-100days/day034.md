# Day 034：前后缀最值与完全背包

## 学习内容

今天两题看着是叙事题，但模型很干净。

梦境巡查可以转成“初始能量至少覆盖所有前缀缺口”。正常情况下第 `j` 段出发前的需求是 `sum(a0..aj)-sum(b1..bj)`。如果某个 `b_i` 被删除，只会影响 `j>=i` 的后缀需求，让它们整体增加 `b_i`。所以对每个 `i`，答案是 `max(前缀最大, 后缀最大+b_i)`。

机器人饲养指南是完全背包：每天可以投喂 `1..m` 个苹果，每种“投喂数量”可以使用任意多天，要求正好喂完 `n` 个苹果并最大化快乐值。`n<=1e4,m<=100`，直接 `dp[i]=max(dp[i-j]+A_j)`。

## 题解

### 第36次 T2 梦境巡查

**题意抓手**

巡查员从 0 号区域出发，依次经过 1 到 `n` 再回到 0。走第 `j` 段需要能量 `a_j`，到达第 `i` 号区域后获得补给 `b_i`。现在每次假设只有一个区域 `i` 的补给变为 0，求顺利完成巡查所需最小初始能量 `w(i)`。

**知识点**

最小初始能量等于所有前缀“净消耗需求”的最大值。正常补给下，第 `j` 段出发前必须满足：

```text
w >= a0 + a1 + ... + aj - (b1 + b2 + ... + bj)
```

记这个值为 `need[j]`。

**解法**

先计算正常情况下的 `need[0..n]`。若删除第 `i` 个补给：

- 对 `j<i` 的需求不变；
- 对 `j>=i` 的需求都会增加 `b_i`。

因此预处理 `prefixMax[i]=max(need[0..i])` 和 `suffixMax[i]=max(need[i..n])`，答案为：

```text
w(i)=max(prefixMax[i-1], suffixMax[i]+b_i)
```

**易错点**

- `a` 有 `n+1` 个，`b` 有 `n` 个。
- 删除 `b_i` 只影响从第 `i` 段之后的所有需求。
- 输出一行 `n` 个数。

**复杂度**

时间复杂度 `O(n)`，空间复杂度 `O(n)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<long long> a(n + 1), b(n + 1, 0);
    for (long long& value : a) cin >> value;
    for (int i = 1; i <= n; ++i) cin >> b[i];

    vector<long long> need(n + 1);
    long long sum_a = 0, sum_b = 0;
    for (int j = 0; j <= n; ++j) {
        sum_a += a[j];
        if (j > 0) sum_b += b[j];
        need[j] = sum_a - sum_b;
    }

    vector<long long> prefix_max = need;
    for (int i = 1; i <= n; ++i) {
        prefix_max[i] = max(prefix_max[i], prefix_max[i - 1]);
    }

    vector<long long> suffix_max = need;
    for (int i = n - 1; i >= 0; --i) {
        suffix_max[i] = max(suffix_max[i], suffix_max[i + 1]);
    }

    for (int i = 1; i <= n; ++i) {
        if (i > 1) cout << ' ';
        cout << max(prefix_max[i - 1], suffix_max[i] + b[i]);
    }
    cout << '\n';
    return 0;
}
```

### 第37次 T2 机器人饲养指南

**题意抓手**

一共有 `n` 个苹果。每天最多喂 `m` 个，如果某天喂 `i` 个苹果，收益为 `A_i`。问喂完所有苹果的最大总收益。

**知识点**

完全背包或线性 DP。每天的喂法可以重复使用，且必须正好喂完 `n` 个苹果。`dp[x]` 表示喂完 `x` 个苹果的最大收益。

**解法**

初始化 `dp[0]=0`。对 `apples=1..n`，枚举当天喂 `take=1..min(m,apples)` 个，转移：

```text
dp[apples] = max(dp[apples-take] + A_take)
```

**易错点**

- 收益序列不一定单调，不能贪心选收益最大的单日数量。
- 每天可以少喂，也可以多天重复同一种喂法。
- 输出喂完全部 `n` 个苹果的最大收益。

**复杂度**

时间复杂度 `O(nm)`，空间复杂度 `O(n)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;
    vector<long long> reward(m + 1, 0);
    for (int i = 1; i <= m; ++i) {
        cin >> reward[i];
    }

    vector<long long> dp(n + 1, 0);
    for (int apples = 1; apples <= n; ++apples) {
        long long best = 0;
        for (int take = 1; take <= min(m, apples); ++take) {
            best = max(best, dp[apples - take] + reward[take]);
        }
        dp[apples] = best;
    }

    cout << dp[n] << '\n';
    return 0;
}
```
