# Day 028：分段函数合并与差分统计

## 学习内容

今天的两题都可以看成“区间贡献”。

序列查询新解里，`f(x)` 和 `g(x)` 都是分段常数函数。`f(x)` 在相邻 `A_i` 之间不变，`g(x)=floor(x/r)` 在长度为 `r` 的区间内不变。因此不需要枚举 `0..N-1` 的每个 `x`，只要合并两类断点，计算每一段长度乘以误差即可。

出行计划是差分数组。一次出行计划 `(t,c)` 对检测时间 `q` 的有效区间是：

```text
t - k - c + 1 <= q <= t - k
```

把每个计划给这个区间加 1，最后前缀和即可回答所有查询。时间上界固定为 `2e5`，差分数组非常合适。

## 题解

### 第24次 T2 序列查询新解

**题意抓手**

有序序列 `A_0=0<A_1<...<A_n<N`。`f(x)` 是不超过 `x` 的最大 `A_i` 的下标；`g(x)=floor(x/r)`，其中 `r=floor(N/(n+1))`。要求 `sum_{x=0}^{N-1} |g(x)-f(x)|`。

**知识点**

`f` 和 `g` 都是阶梯函数。`f` 的断点是 `A_i`，`g` 的断点是 `r,2r,3r...`。在相邻断点之间，两者都不变，整段贡献可以一次算完。

**解法**

设当前处理到 `x`，用 `idx` 表示当前 `f(x)`。下一处可能变化的位置是 `A[idx+1]` 或 `(g(x)+1)*r`，取两者和 `N` 的最小值为 `next_x`。这一段 `[x,next_x)` 的误差固定为 `abs(g-f)`，贡献为 `(next_x-x)*abs(g-f)`。不断推进到 `N`。

**易错点**

- 输入不包含 `A_0`，需要手动补 0；末尾也可以补 `N` 作为边界。
- 结果可能很大，必须用 `long long`。
- `g(x)` 可能大于 `n`，按公式正常计算即可。

**复杂度**

每次推进至少跨过一个断点，时间复杂度约 `O(n + N/r)`，在约束下可视为 `O(n)`；空间复杂度 `O(n)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    long long N;
    cin >> n >> N;

    vector<long long> a(n + 2);
    a[0] = 0;
    for (int i = 1; i <= n; ++i) {
        cin >> a[i];
    }
    a[n + 1] = N;

    long long r = N / (n + 1);
    long long x = 0;
    int idx = 0;
    long long ans = 0;
    while (x < N) {
        while (idx + 1 <= n + 1 && a[idx + 1] <= x) {
            ++idx;
        }
        long long f = idx;
        long long g = x / r;
        long long next_x = min({N, a[idx + 1], (g + 1) * r});
        ans += (next_x - x) * llabs(g - f);
        x = next_x;
    }

    cout << ans << '\n';
    return 0;
}
```

### 第25次 T2 出行计划

**题意抓手**

如果在 `q` 时刻检测，`q+k` 时刻拿到结果。某计划在 `t` 时刻进入场所，要求 `c` 个单位时间内的核酸结果。问每个查询 `q` 能满足多少计划。

**知识点**

对单个计划反推可行检测时间。检测结果有效覆盖该计划需要：

```text
q + k <= t <= q + k + c - 1
```

移项得到 `q` 的有效区间 `[t-k-c+1, t-k]`。每个计划对这个区间内所有查询答案贡献 1，用差分处理。

**解法**

开长度 `200000+3` 的差分数组。对每个计划计算左右端点，裁剪到查询时间范围 `[1,200000]`。若区间非空，`diff[l]++`、`diff[r+1]--`。最后前缀和得到每个检测时间的可满足计划数，逐个输出查询位置的值。

**易错点**

- 右端点是 `t-k`，不是 `t`。
- 左端点有 `+1`：`t <= q+k+c-1` 推出 `q >= t-k-c+1`。
- 如果 `t-k < 1`，这个计划对正整数查询没有贡献。

**复杂度**

时间复杂度 `O(n + T + m)`，其中 `T=200000`；空间复杂度 `O(T)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m, k;
    cin >> n >> m >> k;

    const int MAX_TIME = 200000;
    vector<int> diff(MAX_TIME + 3, 0);
    for (int i = 0; i < n; ++i) {
        int t, c;
        cin >> t >> c;
        int left = t - c - k + 1;
        int right = t - k;
        if (right < 1) {
            continue;
        }
        left = max(left, 1);
        right = min(right, MAX_TIME);
        if (left <= right) {
            ++diff[left];
            --diff[right + 1];
        }
    }

    for (int i = 1; i <= MAX_TIME; ++i) {
        diff[i] += diff[i - 1];
    }

    while (m--) {
        int q;
        cin >> q;
        cout << diff[q] << '\n';
    }

    return 0;
}
```
