# Day 008：距离排序与线性加权

## 学习内容

Day 008 训练两种常见 T1 模型：按规则找最优项，以及线性加权求和。重点仍然是读清楚“比较规则”和“输出规则”。

- **检测点查询**：比较距离时不用开方，平方距离大小关系相同。并列时编号小的更近。只需要前三名，可以维护长度不超过 3 的候选表，做到 `O(n)`。
- **期末预测之安全指数**：直接累加 `w_i * score_i`，最后套 `max(0, sum)`。因为 `n` 可到 `10^5`，要用 `long long`。

今天的思维重点：

1. 如果只关心距离大小，优先使用平方距离，避免浮点数。
2. 比较规则可以写成 pair：`(距离, 编号)`。
3. 线性公式题先确定每项贡献，再统一处理最后的截断函数。

## 题解

### 第20次 T1 检测点查询

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给出市民位置 `(X,Y)` 和 `n` 个检测点坐标，输出距离市民最近的三个检测点编号。如果距离相同，编号小的排在前面。

**知识点**

欧氏距离为：

```text
sqrt((x-X)^2 + (y-Y)^2)
```

比较距离时平方根不会改变大小关系，所以直接比较平方距离：

```text
dist2 = (x-X)^2 + (y-Y)^2
```

并列规则可以合并进 `(dist2, index)` 的字典序比较。

**解法**

1. 读入 `n, X, Y`。
2. 维护一个最多 3 个元素的 `best` 候选表。
3. 对每个检测点计算 `(dist2, index)`，插入候选表。
4. 候选表按 pair 升序排列，超过 3 个就删除最后一个。
5. 扫描结束后输出三个编号。

本题 `n <= 200`，排序全部点也能过；但只维护前三名可以做到线性时间，空间也是常数级。

**易错点**

- 不要输出距离，输出编号。
- 距离相同时编号小者更近。
- 不需要开平方。
- 输出三行，每行一个编号。

**复杂度**

时间复杂度 `O(n)`，因为候选表长度最多 4，每次排序都是常数规模；额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    long long targetX, targetY;
    cin >> n >> targetX >> targetY;

    vector<pair<long long, int>> best;
    for (int i = 1; i <= n; i++) {
        long long x, y;
        cin >> x >> y;
        long long dx = x - targetX;
        long long dy = y - targetY;
        best.push_back({dx * dx + dy * dy, i});
        sort(best.begin(), best.end());
        if (best.size() > 3) {
            best.pop_back();
        }
    }

    for (const auto &[distance, index] : best) {
        (void)distance;
        cout << index << '\n';
    }

    return 0;
}
```

### 第21次 T1 期末预测之安全指数

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给出 `n` 个指标，每个指标有权重 `w_i` 和得分 `score_i`。安全指数为：

```text
y = max(0, sum(w_i * score_i))
```

输出 `y`。

**知识点**

这是线性加权求和。每一项互不影响，边读边累加即可。权重可能为负，最终总和可能为负，所以最后要用 ReLU 截断到 0。

`n <= 100000`，每项乘积最大绝对值约 `1000`，总和约 `1e8`，`int` 理论上够，但使用 `long long` 更稳。

**解法**

1. 读入 `n`。
2. 初始化 `sum = 0`。
3. 对每项读入 `w, score`，累加 `w * score`。
4. 输出 `max(0, sum)`。

**易错点**

- 输入顺序是 `w_i score_i`。
- 不要每一项单独取 `max(0, contribution)`，ReLU 只作用在总和上。
- 负权重会降低总和。

**复杂度**

时间复杂度 `O(n)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    long long sum = 0;
    for (int i = 0; i < n; i++) {
        long long w, score;
        cin >> w >> score;
        sum += w * score;
    }
    cout << max(0LL, sum) << '\n';
    return 0;
}
```
