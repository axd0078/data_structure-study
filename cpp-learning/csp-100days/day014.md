# Day 014：多维比较与去重计数

## 学习内容

Day 014 的两道题都和“计数前先定义清楚对象”有关。仓库规划要判断一个多维向量是否被另一个多维向量严格支配；词频统计要区分“总出现次数”和“出现过的文章数”。

**仓库规划**中，仓库 `j` 能成为仓库 `i` 的上级，当且仅当 `j` 的每一维编码都严格大于 `i`。如果多个仓库满足条件，选择编号最小的那个。因为 `n <= 1000`、`m <= 10`，按编号从小到大枚举候选上级，再逐维比较，是直接且稳定的做法。

**词频统计**中，每个词要输出两个数：出现过它的文章篇数，以及它在所有文章里的总出现次数。总次数每读到一次就加一；文章篇数不能在同一篇文章里重复加，所以需要记录当前文章中某个词是否已经出现过。可以用 `seen_stamp[word] == article_id` 做时间戳标记，比每篇文章创建集合更轻。

今天的最低复杂度目标：

- 仓库规划：`O(n^2 * m)` 时间，`O(nm)` 空间。
- 词频统计：`O(total_words + m)` 时间，`O(m)` 空间。

## 题解

### 第32次 T1 仓库规划

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

有 `n` 个仓库，每个仓库有一个 `m` 维整数编码。对仓库 `i`，要找编号最小的仓库 `j`，满足：

```text
code[j][k] > code[i][k]    对所有 k 都成立
```

如果没有这样的 `j`，输出 `0`。

**知识点**

这是多维严格支配判断。关键是“每一维都大于”，不是至少一维大于，也不是大于等于。因为题目要求编号最小，所以枚举候选上级时从 `j=0` 到 `n-1`，找到第一个满足条件的就可以停止。

**解法**

1. 读入 `n,m` 和所有仓库编码。
2. 对每个仓库 `i`：
   - 从小到大枚举候选仓库 `j`。
   - 跳过 `i==j`。
   - 检查所有维度 `k` 是否都有 `a[j][k] > a[i][k]`。
   - 第一个满足条件的 `j` 就是答案。
3. 若枚举完没有找到，输出 `0`。

**易错点**

- 条件是严格大于 `>`，不是 `>=`。
- 多个上级候选时取编号最小，所以不能取“编码最接近”或“维度和最小”。
- 编号输出从 1 开始，数组下标从 0 开始，输出时要 `j + 1`。
- 负数编码同样按普通大小比较。

**复杂度**

时间复杂度 `O(n^2 * m)`，空间复杂度 `O(nm)`。在 `n <= 1000`、`m <= 10` 下最多约一千万次维度比较，符合 T1 要求。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;
    vector<vector<int>> a(n, vector<int>(m));
    for (int i = 0; i < n; ++i) {
        for (int j = 0; j < m; ++j) {
            cin >> a[i][j];
        }
    }

    for (int i = 0; i < n; ++i) {
        int answer = 0;
        for (int j = 0; j < n; ++j) {
            if (i == j) {
                continue;
            }

            bool ok = true;
            for (int k = 0; k < m; ++k) {
                if (a[j][k] <= a[i][k]) {
                    ok = false;
                    break;
                }
            }

            if (ok) {
                answer = j + 1;
                break;
            }
        }
        cout << answer << '\n';
    }

    return 0;
}
```

### 第33次 T1 词频统计

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

输入 `n` 篇文章和词编号上限 `m`。每篇文章先给长度 `l_i`，后面跟 `l_i` 个词编号。对每个词 `1..m`，输出：

```text
出现过这个词的文章篇数  这个词的总出现次数
```

**知识点**

“总出现次数”和“文章篇数”不能混在一起。比如一个词在同一篇文章里出现 5 次：

- 总出现次数要加 5。
- 文章篇数只能加 1。

用 `seen_stamp[word]` 记录这个词最近一次出现在哪篇文章。处理第 `article` 篇文章时，如果 `seen_stamp[word] != article`，说明这是该词在本文中第一次出现，于是文章篇数加 1，并把时间戳设为当前文章编号。

**解法**

1. 建立 `doc_count[m+1]`、`total_count[m+1]`、`seen_stamp[m+1]`。
2. 对第 `article` 篇文章：
   - 读入长度 `len`。
   - 对每个词 `word`：
     - `total_count[word]++`。
     - 如果 `seen_stamp[word] != article`，则 `doc_count[word]++`，并更新 `seen_stamp[word] = article`。
3. 最后按 `1..m` 输出两个计数。

**易错点**

- 同一篇文章里重复出现的词，文章篇数只能加一次。
- 输出顺序是词编号从小到大，不是按出现次数排序。
- `m <= 100`，数组比哈希表更直接。
- `seen_stamp` 初始为 0，文章编号从 1 开始，正好区分“从未出现”。

**复杂度**

设所有文章总词数为 `S`。时间复杂度 `O(S + m)`，空间复杂度 `O(m)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    vector<int> doc_count(m + 1), total_count(m + 1), seen_stamp(m + 1);

    for (int article = 1; article <= n; ++article) {
        int len;
        cin >> len;
        for (int i = 0; i < len; ++i) {
            int word;
            cin >> word;

            ++total_count[word];
            if (seen_stamp[word] != article) {
                seen_stamp[word] = article;
                ++doc_count[word];
            }
        }
    }

    for (int word = 1; word <= m; ++word) {
        cout << doc_count[word] << ' ' << total_count[word] << '\n';
    }

    return 0;
}
```
