# Day 004：排序比较与频次统计

## 学习内容

Day 004 仍然是 T1 保分训练，但今天开始把“题目约束”当成算法选择的一部分。三道题看起来都很基础，真正要练的是：读清楚规则后，不套模板，直接选当前数据范围下最低复杂度的做法。

今天的三个核心模型：

- **顺序分组贪心**：题目已经规定必须按编号从小到大分蛋糕，不能重排。每块蛋糕只会进入当前朋友的一组，当前组重量达到 `k` 就立刻结算，因此单次扫描就是最优。
- **整数除法与取模**：打酱油不是模拟买卖过程，而是把钱先换成“原价可买瓶数”，再用 `5` 瓶优惠和 `3` 瓶优惠做分组。题目只有一个整数输入，最优解可以做到 `O(1)`。
- **小值域出现表**：最小差值通常可以先排序再看相邻差值，但本题所有数都不超过 `10000`，用 `seen[value]` 记录出现情况，再从小到大扫描值域，可以把复杂度降到 `O(n + V)`，其中 `V=10000`。

判断题目能不能降复杂度，可以按下面的顺序想：

1. **顺序是否固定**：如果题目说“总是先给编号最小的蛋糕”，就不能排序，也不需要搜索，直接按输入顺序处理。
2. **操作是否可合并成公式**：如果只有固定优惠规则，先把单位统一，再用整除和取模表示完整分组。
3. **值域是否很小**：如果 `ai <= 10000` 这种上界明确且不大，出现表或计数桶常常比排序更合适。

今天需要重点处理的边界：

- 分蛋糕最后一组即使不足 `k`，只要还有蛋糕，也算一个朋友分到了蛋糕。
- 打酱油输入金额是 `10` 的倍数，要先转成原价瓶数 `money / 10`。
- 最小差值如果有重复数字，答案立即是 `0`；否则答案来自升序后相邻两个出现过的值。

### 第10次 T1 分蛋糕

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

有 `n` 块蛋糕，编号从 `1` 到 `n`，重量依次为 `a1..an`。小明按编号从小到大分蛋糕。对每个朋友，他会一直给当前编号最小的蛋糕，直到这个朋友拿到的总重量达到 `k`，或者所有蛋糕都分完。问最后一共有多少个朋友拿到了蛋糕。

注意最后一个朋友不一定拿到重量 `>= k`。如果蛋糕用完时当前朋友已经拿到了一些蛋糕，即使不足 `k`，也要计入答案。

**知识点**

这题的关键词是“按编号从小到大”。顺序固定后，不存在选择空间：第 `i` 块蛋糕一定在第 `i-1` 块之后处理。当前朋友已经拿到的重量只需要用一个变量 `current` 记录。

什么时候能贪心结算？

- 当前朋友重量还没到 `k`：必须继续给下一块。
- 当前朋友重量达到 `k`：这个朋友已经满足要求，下一块蛋糕必须给下一个朋友。
- 输入结束但 `current > 0`：最后一个朋友拿到了蛋糕，也要计数。

**解法**

1. 读入 `n` 和 `k`。
2. 用 `current` 记录当前朋友拿到的总重量，初始为 0。
3. 依次读入每块蛋糕重量 `weight`，累加到 `current`。
4. 如果 `current >= k`，说明当前朋友结算，答案加 1，并把 `current` 清零。
5. 扫描结束后，如果 `current > 0`，说明还有不足 `k` 的最后一组，答案再加 1。

这个做法已经是最低时间复杂度，因为每块蛋糕的重量至少要读一次；额外空间也只用了两个整数。

**易错点**

- 不能把蛋糕排序，题目要求按编号从小到大分。
- 达到 `k` 后要立刻开始下一个朋友，不能继续往当前朋友手里放蛋糕。
- 最后一组不足 `k` 也算一个朋友，只要 `current > 0`。
- 如果某块蛋糕本身已经 `>= k`，它单独就能让一个朋友结算。

**复杂度**

时间复杂度 `O(n)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, k;
    cin >> n >> k;

    int friends = 0;
    int current = 0;
    for (int i = 0; i < n; i++) {
        int weight;
        cin >> weight;
        current += weight;
        if (current >= k) {
            friends++;
            current = 0;
        }
    }

    if (current > 0) {
        friends++;
    }

    cout << friends << '\n';
    return 0;
}
```

### 第11次 T1 打酱油

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

酱油每瓶 10 元。商家有两种优惠：

- 买 3 瓶送 1 瓶，也就是付 3 瓶的钱拿 4 瓶。
- 买 5 瓶送 2 瓶，也就是付 5 瓶的钱拿 7 瓶。

给定金额 `N`，且 `N` 是 10 的倍数，求最多能拿到多少瓶酱油。

**知识点**

先把金额转成“如果不优惠能买几瓶”：

```text
paid = N / 10
```

之后问题变成：用 `paid` 个付费瓶数，尽量组成 `5` 和 `3` 的优惠包，让最终瓶数最大。

对比两个优惠：

- 付 5 得 7，额外送 2。
- 付 3 得 4，额外送 1。

`5` 瓶优惠的赠送比例更高，所以优先使用 5 瓶优惠。剩下的 `0..4` 瓶中，只有剩余 `3` 或 `4` 时还能再使用一个 3 瓶优惠。剩余 `1` 或 `2` 只能原价购买。

需要确认“优先用 5 瓶优惠”不会因为余数变差：

- `5 + 1` 和 `3 + 3`：都付 6 瓶，最终都拿 8 瓶。
- `5 + 2` 和 `3 + 3 + 1`：都付 7 瓶，最终都拿 9 瓶。
- `5 + 3` 比 `3 + 3 + 2` 更好或相等。
- `5 + 4` 和 `3 + 3 + 3`：都付 9 瓶，最终都拿 12 瓶。

所以先尽量分 5 瓶优惠是正确的。

**解法**

1. 读入金额 `money`。
2. 计算原价可买瓶数 `paidBottles = money / 10`。
3. 每 `5` 个付费瓶数换成 `7` 瓶，答案加 `paidBottles / 5 * 7`。
4. 剩余瓶数 `paidBottles %= 5`。
5. 剩余部分每 `3` 个付费瓶数换成 `4` 瓶，不够 3 的按原价瓶数加入答案。

整个过程只做常数次整除和取模，因此是 `O(1)`。

**易错点**

- 输入是钱，不是瓶数，第一步必须除以 10。
- 不要只算赠品数量，输出的是最终拿到的总瓶数。
- 不能先用 3 瓶优惠再用 5 瓶优惠，否则有些情况会少拿。
- `N=10` 或 `N=20` 时没有优惠，答案分别是 1 和 2。

**复杂度**

时间复杂度 `O(1)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int money;
    cin >> money;

    int paidBottles = money / 10;
    int answer = paidBottles / 5 * 7;
    paidBottles %= 5;
    answer += paidBottles / 3 * 4 + paidBottles % 3;

    cout << answer << '\n';
    return 0;
}
```

### 第12次 T1 最小差值

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给定 `n` 个正整数，找出任意两个数的差的绝对值中的最小值并输出。两个相同的数差值为 0。

例如如果出现两个相同的 `3`，答案就是 0；如果没有重复值，就要找数值最接近的两个数。

**知识点**

如果把所有数升序排列，最小差值一定出现在相邻两个数之间。原因是：对任意 `x < y < z`，`z - x` 一定不小于 `y - x`，也不小于 `z - y`，所以跨过中间元素的差值不可能更小。

常规做法是排序，复杂度 `O(n log n)`。但本题还有一个更关键的条件：每个数都是不超过 `10000` 的正整数。可以用出现表 `seen[10001]` 记录每个值有没有出现，然后从 `1` 到 `10000` 扫描出现过的值，这等价于得到一个升序序列，但不需要比较排序。

处理重复值时：

- 读入时如果发现某个值已经出现过，答案必然是 0。
- 仍然可以把输入读完，再直接输出 0。

**解法**

1. 读入 `n`。
2. 建立 `seen[10001]`，初始全为 `false`。
3. 读入每个数 `value`：
   - 如果 `seen[value]` 已经是 `true`，记录存在重复值。
   - 否则把 `seen[value]` 设为 `true`。
4. 如果存在重复值，输出 0。
5. 否则从 `1` 到 `10000` 扫描所有出现过的值。
6. 用 `previous` 记录上一个出现过的值，用相邻差值更新答案。

这个做法利用了小值域，避免排序。因为 `V=10000` 是题目给出的固定上界，实际运行非常稳定。

**易错点**

- 有重复值时答案是 0，不需要再找其他差值。
- 不要只比较输入中相邻的两个数，题目要求任意两个数。
- 如果用排序，最小差值只需要比较排序后的相邻元素；如果用出现表，也只需要比较扫描时相邻出现的值。
- `n >= 2`，所以不存在无答案的情况。

**复杂度**

时间复杂度 `O(n + V)`，其中 `V=10000`；额外空间复杂度 `O(V)`。在本题约束下，这比排序的 `O(n log n)` 更低。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<bool> seen(10001, false);
    bool hasDuplicate = false;
    for (int i = 0; i < n; i++) {
        int value;
        cin >> value;
        if (seen[value]) {
            hasDuplicate = true;
        }
        seen[value] = true;
    }

    if (hasDuplicate) {
        cout << 0 << '\n';
        return 0;
    }

    int previous = -1;
    int answer = 10001;
    for (int value = 1; value <= 10000; value++) {
        if (!seen[value]) {
            continue;
        }
        if (previous != -1) {
            answer = min(answer, value - previous);
        }
        previous = value;
    }

    cout << answer << '\n';
    return 0;
}
```
