# Day 041：字符串 API 与 token 设计

## 学习内容

今天从“把输入文本变成结构化信息”开始练 T3。第09次 T3《权限查询》更偏映射建模：权限、角色、用户之间是多层关系，核心是把字符串权限统一拆成“权限名 + 可选等级”，再把用户最终拥有的最高权限预处理出来。第10次 T3《Markdown》更偏轻量解析：先识别块级结构，再处理行内标记，训练把复杂文本题拆成多个小解析器。

这类题的共同方法是先设计 token。不要一边读字符串一边混着业务逻辑硬写判断，最好先问三件事：

- 原始字符串可以拆成哪些稳定单位，例如 `disk:5`、`<int>`、`_em_`、`[text](url)`。
- 每个单位有哪些属性，例如权限是否带等级、Markdown 标记的开始结束位置。
- 后续查询需要什么形式的数据，例如 `map<string,int>` 存最高权限等级，或把 Markdown 分成块再递归处理行内片段。

《权限查询》里的权限分两类：不带等级的权限只有 true/false；带等级的权限需要保存最高等级。查询 `name:x` 时输出是否拥有至少该等级，查询 `name` 时如果是等级权限输出最高等级，否则输出 true/false。实现时可以用 `map<string, int>`，把普通权限记成一个特殊值，例如 `-1`，等级权限记成最大等级。

《Markdown》里的块级结构优先级比行内结构高。空行切块；块的第一行如果是若干 `#` 加空格，就是标题；如果每行都以 `* ` 开头，就是无序列表；否则是段落。行内结构只有强调和链接，题目保证格式合法，可以用递归函数处理 `_..._` 和 `[...](...)`，不同标记互相嵌套时只要每次找到最早出现的合法标记即可。

## 题解

### 第09次 T3 权限查询

**题意抓手**

系统中先给出权限定义，再给出角色包含的权限，再给出用户拥有的角色。最后每次查询某用户是否拥有某权限，或者查询某等级权限的最高等级。权限字符串有两种形式：`name` 和 `name:level`。

**知识点**

本题不是图搜索，角色只展开一层即可。重点是字符串拆分和映射聚合：把 `name:level` 拆成权限名和等级；用户继承多个角色时，对同名等级权限取最大等级，对普通权限只需记录存在。

**解法**

读入角色时，保存 `role -> permissions`。读入用户时，遍历用户的所有角色，把角色中的权限合并到 `user -> map<permission_name, max_level>`。普通权限用 `-1` 表示，等级权限用非负整数表示。查询时同样解析查询串：如果用户不存在或权限名不存在，输出 `false`；如果查询带等级，判断保存的等级是否不小于查询等级；如果查询不带等级，普通权限输出 `true`，等级权限输出最高等级。

**易错点**

- 权限定义部分不需要参与判断，真正有效的是角色和用户关系。
- 同一用户通过多个角色拿到同一个等级权限时必须取最大值。
- 查询 `perm` 和 `perm:3` 的输出规则不同：前者可能输出数字，后者只输出 `true/false`。
- 不存在的用户、不存在的权限、普通权限被按等级查询，都应输出 `false`。

**复杂度**

设角色权限总数为 `R`，用户继承关系展开后的权限总数为 `U`，查询数为 `Q`。预处理时间复杂度 `O(R+U)`，每次查询 `O(log P)` 或哈希表均摊 `O(1)`；空间复杂度 `O(R+U)`。

### 第10次 T3 Markdown

**题意抓手**

输入一段简化 Markdown 文本，输出对应 HTML。需要处理标题、无序列表、段落三类块级元素，以及强调 `_text_`、链接 `[text](url)` 两类行内元素。

**知识点**

本题适合分层解析：先按空行切分块，再判断块类型，最后对块内文本做行内转换。行内转换是一个小递归解析器，因为强调里可以有链接，链接文本里也可以有强调。

**解法**

先把所有输入行读入，连续非空行组成一个块。对每个块判断：

- 第一行形如 `#...# text`，其中 `#` 后紧跟空格，则输出 `<hN>...</hN>`。
- 块中每行都以 `* ` 开头，则输出 `<ul>`，每行去掉 `* ` 后作为一个 `<li>`。
- 其他情况是段落，把块内多行用换行拼接，外层包 `<p>`。

行内函数从左到右扫描字符串，遇到 `_` 找下一个 `_`，中间递归后包 `<em>`；遇到 `[` 找匹配的 `](` 和 `)`，链接文本递归后包 `<a href="url">`；普通字符原样输出。题目保证语法合法，不需要处理畸形闭合。

**易错点**

- 标题必须是若干 `#` 后跟一个空格，`#Title` 是普通段落。
- 无序列表的每一行都要以 `* ` 开头，列表外层要有单独的 `<ul>` 和 `</ul>`。
- 段落内多行之间保留换行，而不是直接拼接成一行。
- 行内链接的文本部分仍要递归处理强调。

**复杂度**

设全文长度为 `L`。块切分和行内扫描总时间复杂度 `O(L)`；递归只处理子串范围，实际实现若频繁 `substr` 会增加常数，仍可通过本题规模。空间复杂度 `O(L)`。
