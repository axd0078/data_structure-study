# Day 001：读题流程与边界样例

## 学习内容

Day 001 的三道题都是 T1 入门题，目标不是学习复杂算法，而是把“读题、圈数据范围、选最直接的数据结构、写边界样例”的流程固定下来。T1 通常可以用线性扫描、计数数组、排序或集合解决，真正容易丢分的地方往往是题意口径和输出格式。

做题时先按这个顺序处理：

1. 圈出输入规模和数值范围，判断能不能直接排序、开数组或用集合。
2. 找清楚“问的是数对、次数、还是每条记录的即时答案”。
3. 先手算最小样例和特殊样例，再写代码。
4. 输出前确认是否需要空格分隔、是否只有一行、末尾换行是否保留。

今天重点掌握三类基础模型：

- **集合查找**：当题目只关心某个值是否出现过，`set` / `unordered_set` 比双重循环更稳。
- **排序后相邻比较**：当题目问“差值正好为 1”的数对，排序可以把可能配对的元素变成相邻元素。
- **即时计数**：当题目要求每条记录是第几次出现，要边读边更新，不能等读完后只输出总频次。

建议今天额外手造这些边界：

- 只有 1 个数或 1 条记录。
- 没有任何答案的情况。
- 所有元素都能形成答案的情况。
- 包含负数、正数、读者编号重复多次的情况。

## 题解

### 第01次 T1 相反数

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给出 `N` 个互不相同的非零整数，统计有多少对数满足一个是另一个的相反数。比如 `1` 和 `-1` 算一对，`2` 和 `-2` 算一对。

**知识点**

这题的核心是“出现性判断”。如果已经知道所有数字是否出现，就能在 `O(1)` 或 `O(log n)` 的时间判断 `-x` 是否存在。由于题目只问配对数量，不需要保留原始顺序。

**解法**

朴素做法是枚举所有两两组合，判断它们和是否为 0，时间复杂度 `O(n^2)`。虽然本题 `n <= 500` 也能过，但这不是最清晰的模型。

更稳定的做法是：

1. 把所有数放进 `unordered_set<int>`。
2. 遍历集合里的每个数 `x`。
3. 只在 `x > 0` 时检查 `-x` 是否存在。
4. 如果存在，答案加一。

只统计正数的原因是避免重复计数。否则遍历到 `1` 会数一次，遍历到 `-1` 又会再数一次。

**易错点**

- 题目保证整数互不相同，所以不用处理重复元素。
- 题目说整数非零，`0` 不会出现；即使出现，也不能把 `0` 和自己算作一对。
- 只输出一个整数，不要输出配对本身。

**复杂度**

使用 `unordered_set` 时，期望时间复杂度 `O(n)`，空间复杂度 `O(n)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    unordered_set<int> seen;
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        seen.insert(x);
    }

    int ans = 0;
    for (int x : seen) {
        if (x > 0 && seen.count(-x)) {
            ans++;
        }
    }

    cout << ans << '\n';
    return 0;
}
```

### 第02次 T1 相邻数对

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给出 `n` 个不同的整数，统计有多少对数的差正好为 1。样例中 `(2,3)`、`(6,7)`、`(7,8)` 都算答案。

**知识点**

这题可以用排序，也可以用集合。排序后，两个值如果差 1，它们一定会在有序数组中相邻；因为中间如果还有整数，就不可能差 1。

**解法**

推荐排序扫描：

1. 读入所有整数。
2. 对数组升序排序。
3. 从第二个元素开始扫描，如果 `a[i] - a[i-1] == 1`，答案加一。

由于题目保证数字不同，不需要考虑重复值。如果题目没有保证不同，则排序后要先去重，或者改用集合判断。

**易错点**

- 问的是“数对数量”，不是最长连续段长度。
- 输入数可能原本无序，不能只比较输入中相邻的位置。
- 题目中的数互不相同，所以 `(x, x+1)` 最多贡献一次。

**复杂度**

排序时间复杂度 `O(n log n)`，扫描 `O(n)`，空间复杂度取决于排序实现和数组存储，为 `O(n)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<int> a(n);
    for (int& x : a) {
        cin >> x;
    }

    sort(a.begin(), a.end());

    int ans = 0;
    for (int i = 1; i < n; i++) {
        if (a[i] - a[i - 1] == 1) {
            ans++;
        }
    }

    cout << ans << '\n';
    return 0;
}
```

### 第03次 T1 门禁系统

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给出一串读者编号记录。对每条记录，输出这个编号到当前为止是第几次出现。比如输入 `1 2 1 1 3`，输出 `1 1 2 3 1`。

**知识点**

这是典型的“边读边统计”题。答案依赖当前记录之前已经出现过多少次，所以每读到一个编号，就立刻更新它的计数并输出当前次数。

题目保证读者编号是不超过 `n` 的正整数，因此可以直接开 `vector<int> cnt(n + 1)`。如果编号范围很大或不是整数，就应改用 `map` / `unordered_map`。

**解法**

1. 读入记录数 `n`。
2. 初始化计数数组 `cnt`，所有编号出现次数为 0。
3. 依次读取每个编号 `x`。
4. 执行 `++cnt[x]`，得到当前是第几次出现。
5. 按一行空格分隔输出。

这里要注意输出格式：所有答案在同一行，中间一个空格，行末换行。常见写法是根据 `i` 判断是否先输出空格，或根据是否最后一个元素决定输出空格还是换行。

**易错点**

- 不能先统计总次数再输出，因为题目要的是“每条记录出现时的次数”。
- 编号从 1 开始，数组大小至少要开到 `n + 1`。
- 输出是一行，不是每个次数单独一行。

**复杂度**

每条记录处理一次，时间复杂度 `O(n)`，计数数组空间复杂度 `O(n)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<int> cnt(n + 1, 0);
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        cnt[x]++;

        if (i > 0) {
            cout << ' ';
        }
        cout << cnt[x];
    }
    cout << '\n';

    return 0;
}
```
