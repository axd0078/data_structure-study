# Day 030：依赖时间表与二分可行性

## 学习内容

今天是中档模拟阶段的小复盘，两题都和“可行时间”有关。

训练计划的依赖关系满足 `p_i < i`，所以天然是拓扑顺序。最早开始时间从前往后推：没有依赖就是第 1 天，有依赖就等父科目结束后开始。若所有科目最早完成时间都不超过 `n`，再从后往前推最晚开始时间；父科目的最晚开始时间要受所有子科目的最晚开始时间约束。

垦田计划是二分答案。假设最终总耗时不超过 `T`，每块田超过 `T` 的部分都要缩短，代价为 `(t_i-T)*c_i`。如果总代价不超过资源 `m`，说明 `T` 可行；要求最小可行 `T`，在 `[k, max(t_i)]` 上二分即可。

## 题解

### 第28次 T2 训练计划

**题意抓手**

有 `m` 个训练科目，第 `i` 个科目耗时 `t_i`，最多依赖一个编号更小的科目。要求输出每个科目的最早开始时间；如果所有训练能在 `n` 天内完成，还要输出每个科目的最晚开始时间。

**知识点**

依赖编号总是小于当前编号，说明输入顺序就是拓扑顺序。最早时间正向 DP，最晚时间反向 DP。科目 `i` 的子科目最晚从 `L` 开始，则 `i` 最晚必须在 `L - t_i` 开始，才能保证子科目不被耽误。

**解法**

正向计算：

- 无依赖：`earliest[i]=1`；
- 依赖 `p`：`earliest[i]=earliest[p]+duration[p]`。

如果存在 `earliest[i]+duration[i]-1>n`，只输出最早时间。否则初始化 `latest[i]=n-duration[i]+1`，表示不考虑子科目时的最晚开始时间。再从 `m` 到 1 反向扫描，若 `i` 依赖父科目 `p`，则 `latest[p]=min(latest[p], latest[i]-duration[p])`。

**易错点**

- 如果不能完成全部训练，不输出第二行。
- 依赖科目训练结束后的下一天才能开始当前科目。
- 父科目可能有多个子科目，要取所有约束中的最小值。

**复杂度**

正向和反向各扫一遍，时间复杂度 `O(m)`，空间复杂度 `O(m)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

void print_line(const vector<int>& values) {
    for (int i = 0; i < (int)values.size(); ++i) {
        if (i) cout << ' ';
        cout << values[i];
    }
    cout << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;
    vector<int> parent(m), duration(m);
    for (int& value : parent) cin >> value;
    for (int& value : duration) cin >> value;

    vector<int> earliest(m, 1);
    for (int i = 0; i < m; ++i) {
        if (parent[i] != 0) {
            int p = parent[i] - 1;
            earliest[i] = earliest[p] + duration[p];
        }
    }
    print_line(earliest);

    bool feasible = true;
    for (int i = 0; i < m; ++i) {
        if (earliest[i] + duration[i] - 1 > n) {
            feasible = false;
        }
    }
    if (!feasible) {
        return 0;
    }

    vector<int> latest(m);
    for (int i = 0; i < m; ++i) {
        latest[i] = n - duration[i] + 1;
    }
    for (int i = m - 1; i >= 0; --i) {
        if (parent[i] != 0) {
            int p = parent[i] - 1;
            latest[p] = min(latest[p], latest[i] - duration[p]);
        }
    }
    print_line(latest);

    return 0;
}
```

### 第29次 T2 垦田计划

**题意抓手**

有 `n` 块田，每块初始开垦耗时 `t_i`，每减少 1 天需要 `c_i` 资源，最低只能降到 `k` 天。总耗时由最大耗时决定。给定总资源 `m`，求能达到的最小总耗时。

**知识点**

可行性单调：如果能把所有田都压到 `T` 天以内，那么压到更大的天数也一定可行。因此可以二分答案。检查 `T` 时，只需统计所有 `t_i>T` 的田需要的资源。

**解法**

二分区间为 `[k, max(t_i)]`。对中点 `mid`，计算：

```text
cost = sum(max(0, t_i - mid) * c_i)
```

若 `cost <= m`，说明 `mid` 可行，尝试更小的时间；否则资源不够，需要增大时间。

**易错点**

- `m` 可到 `1e9`，乘法和总代价用 `long long`。
- 最低答案不能低于 `k`。
- 检查代价超过 `m` 后可以提前退出，避免无意义累加。

**复杂度**

每次检查 `O(n)`，二分约 17 次，时间复杂度 `O(n log max(t_i))`，空间复杂度 `O(n)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    long long m;
    int k;
    cin >> n >> m >> k;
    vector<pair<int, int>> fields(n);
    int high = k;
    for (auto& [t, c] : fields) {
        cin >> t >> c;
        high = max(high, t);
    }

    int low = k;
    while (low < high) {
        int mid = (low + high) / 2;
        long long cost = 0;
        for (auto [t, c] : fields) {
            if (t > mid) {
                cost += 1LL * (t - mid) * c;
                if (cost > m) {
                    break;
                }
            }
        }
        if (cost <= m) {
            high = mid;
        } else {
            low = mid + 1;
        }
    }

    cout << low << '\n';
    return 0;
}
```
