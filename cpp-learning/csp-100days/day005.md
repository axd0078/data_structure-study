# Day 005：基础模拟与状态更新

## 学习内容

Day 005 继续做 T1 保分题。今天的三道题都不需要复杂数据结构，核心是把题目中的“状态变化规则”翻译成少量变量，并且保持扫描过程稳定。

今天要掌握三类模拟：

- **连续状态计分**：跳一跳里，跳到中心的分数依赖上一次是否也跳到中心。只需要记录“当前连续中心跳的本次分值”。
- **邻域平均**：卖菜里，第 `i` 家店第二天价格只依赖第一天的相邻价格，更新时不能把新价格反过来影响其他店。
- **分类累加**：小明上学里，每条记录按 `k` 分类处理，道路、红灯、黄灯、绿灯分别转化成应花费的时间。

做模拟题时，不要急着写循环，先问三个问题：

1. 当前答案依赖哪些历史信息。
2. 每条输入记录会怎样改变这些信息。
3. 是否存在“同一天/同一轮”的原始数据不能被覆盖。

今天的最低复杂度：

- 跳一跳：输入最多 30 个数，单次扫描，`O(n)` 时间，`O(1)` 空间。
- 卖菜：每个店只处理一次，可以流式输出，`O(n)` 时间，`O(1)` 额外空间。
- 小明上学：每条记录只分类累加一次，`O(n)` 时间，`O(1)` 空间。

## 题解

### 第13次 T1 跳一跳

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

输入是一串 `0/1/2`。`0` 表示游戏结束，且只出现在最后；`1` 表示跳到方块但没到中心，本次得 1 分；`2` 表示跳到中心，如果前一次不是中心或这是第一次跳中心，本次得 2 分，否则连续中心的得分依次为 4、6、8。

要求输出总分。

**知识点**

关键变量是 `centerScore`，表示“如果这一次跳到中心，本次应得几分”。它只在连续中心跳时增加：

- 遇到 `1`：总分加 1，并打断连续中心，`centerScore = 0`。
- 遇到 `2`：`centerScore += 2`，总分加 `centerScore`。
- 遇到 `0`：停止处理。

这种写法比记录上一跳类型更直接，因为连续中心跳的分值本身就是状态。

**解法**

1. 初始化 `total = 0`，`centerScore = 0`。
2. 循环读入 `value`。
3. 如果 `value == 0`，结束。
4. 如果 `value == 1`，`total += 1`，并清空连续中心状态。
5. 如果 `value == 2`，连续中心分值加 2，再累加到总分。

**易错点**

- `0` 不计分，只是终止符。
- `1` 会打断连续中心。
- 第一次跳到中心得 2 分，不是 4 分。
- 连续中心得分增加的是本次得分，不是总分直接翻倍。

**复杂度**

时间复杂度 `O(n)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int total = 0;
    int centerScore = 0;
    int value;
    while (cin >> value && value != 0) {
        if (value == 1) {
            total += 1;
            centerScore = 0;
        } else {
            centerScore += 2;
            total += centerScore;
        }
    }

    cout << total << '\n';
    return 0;
}
```

### 第14次 T1 卖菜

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

一排 `n` 个商店，每个商店第一天有一个价格。第二天每个商店的新价格等于它自己和相邻商店第一天价格的平均值，向下取整。

边界店只有一个相邻店：

- 第 1 家：`(a1 + a2) / 2`
- 第 n 家：`(a[n-1] + a[n]) / 2`
- 中间第 i 家：`(a[i-1] + a[i] + a[i+1]) / 3`

**知识点**

这题容易犯的错是边读边覆盖数组。如果把 `a[i]` 改成第二天价格，再计算 `a[i+1]`，就污染了后面的计算。所有结果都必须基于第一天原始价格。

可以存数组做，空间 `O(n)`。但本题只需要相邻三个原始值，因此可以用滑动窗口流式输出，额外空间做到 `O(1)`：

- 先读 `prev = a1` 和 `cur = a2`，输出第 1 家。
- 从第 3 个数开始，每读入 `next`，就能输出上一家 `cur` 的第二天价格。
- 扫描结束后，输出最后一家。

**解法**

1. 读入 `n`。
2. 读入前两个价格 `prev`、`cur`。
3. 输出第 1 家价格 `(prev + cur) / 2`。
4. 对第 3 到第 n 个价格，读入 `next`，输出 `(prev + cur + next) / 3`，再让窗口右移。
5. 最后输出 `(prev + cur) / 2`。

**易错点**

- 输出一整行，数字之间一个空格，末尾换行。
- 平均值用整数除法向下取整。
- 中间店用三个数平均，首尾店只用两个数平均。
- `n = 2` 时只有首尾两家，两个输出值都等于 `(a1+a2)/2`。

**复杂度**

时间复杂度 `O(n)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    int prev, cur;
    cin >> prev >> cur;

    bool printed = false;
    auto printValue = [&](int value) {
        if (printed) {
            cout << ' ';
        }
        cout << value;
        printed = true;
    };

    printValue((prev + cur) / 2);
    for (int i = 3; i <= n; i++) {
        int next;
        cin >> next;
        printValue((prev + cur + next) / 3);
        prev = cur;
        cur = next;
    }
    printValue((prev + cur) / 2);
    cout << '\n';
    return 0;
}
```

### 第15次 T1 小明上学

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给定红灯 `r` 秒、黄灯 `y` 秒、绿灯 `g` 秒。接下来有 `n` 条记录，每条记录是 `k t`：

- `k = 0`：经过一段道路，用时 `t`。
- `k = 1`：到达路口时看到红灯，还剩 `t` 秒，要等待 `t`。
- `k = 2`：到达路口时看到黄灯，还剩 `t` 秒，要等黄灯剩余 `t` 秒，再等下一轮红灯 `r` 秒。
- `k = 3`：到达路口时看到绿灯，可以直接通过。

输出总用时。

**知识点**

这是第 15 次 T1，不需要根据当前总时间重新推算信号灯状态。题目已经给出“小明到达时看到的颜色和倒计时”，因此每条记录独立转化成等待时间。

分类规则可以写成：

```text
k=0: +t
k=1: +t
k=2: +t+r
k=3: +0
```

因为 `r,y,g,t` 可到 `10^6`，虽然 `n <= 100`，用 `long long` 更稳。

**解法**

1. 读入 `r,y,g`。
2. 读入记录数 `n`。
3. 对每条记录分类累加。
4. 输出总时间。

**易错点**

- 黄灯不能直接通行，要等黄灯剩余时间和下一轮红灯。
- 绿灯不需要增加等待时间。
- 第 15 次 T1 不模拟路上耗时导致的红绿灯变化；那是后续 T2《小明放学》的难点。
- 道路记录 `k=0` 的 `t` 是行驶时间，不是等待时间，但同样要计入总用时。

**复杂度**

时间复杂度 `O(n)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    long long r, y, g;
    cin >> r >> y >> g;

    int n;
    cin >> n;

    long long total = 0;
    for (int i = 0; i < n; i++) {
        int k;
        long long t;
        cin >> k >> t;
        if (k == 0 || k == 1) {
            total += t;
        } else if (k == 2) {
            total += t + r;
        }
    }

    cout << total << '\n';
    return 0;
}
```
