# Day 037：命令行解析与字符串匹配

## 学习内容

今天开始进入早期 T3 的字符串类题目。它们算法不难，但要求严格按题意处理输入、停止条件和输出格式。

《命令行选项》的核心是“状态化解析”。格式串先转成一张表：某个选项是否合法、是否带参数。解析一条命令行时，从工具名之后开始扫描，遇到非法选项、缺少参数或普通参数就立即停止，剩下内容全部忽略。带参数选项如果出现多次，只保留最后一次参数；输出按选项字母升序。

《字符串匹配》是最基础的子串查找。大小写敏感时直接 `find`；大小写不敏感时，把模式串和待匹配行都转成小写再查找。注意输出原始行，而不是转小写后的行。

## 题解

### 第01次 T3 命令行选项

**题意抓手**

给定一个格式串，例如 `ab:m:`，表示 `-a` 不带参数，`-b` 和 `-m` 带参数。对每条命令行，跳过第一个工具名，从左到右解析合法选项，并输出实际出现的选项。

**知识点**

字符串分割、映射表、扫描停止条件。格式串可以用 `map<char,bool>` 保存，`true` 表示该选项需要参数。

**解法**

先扫描格式串，遇到字母就看后一位是否是冒号。处理命令行时用 `stringstream` 分出 token，从下标 1 开始：如果当前 token 不是 `-x` 形式，或 `x` 不在格式表中，则停止。若该选项带参数，必须存在下一个 token；下一个 token 不再按选项判断，而是直接作为参数消耗。最后用有序容器按字母序输出。

**易错点**

- 带参数选项的参数可以包含减号，所以 `-b -a` 中 `-a` 可以成为 `-b` 的参数。
- 解析停止后，后续 token 全部忽略，不要继续找合法选项。
- 同一个带参数选项多次出现时，保留最后一次参数。

**复杂度**

每条命令行最多 256 个字符，逐 token 扫描一次，时间复杂度 `O(L)`；输出用有序容器，最多 26 个选项，空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string spec;
    cin >> spec;
    map<char, bool> accepts;
    for (int i = 0; i < (int)spec.size();) {
        char option = spec[i];
        bool has_arg = (i + 1 < (int)spec.size() && spec[i + 1] == ':');
        accepts[option] = has_arg;
        i += has_arg ? 2 : 1;
    }

    int n;
    cin >> n;
    string line;
    getline(cin, line);
    for (int case_id = 1; case_id <= n; ++case_id) {
        getline(cin, line);
        stringstream ss(line);
        vector<string> tokens;
        string token;
        while (ss >> token) {
            tokens.push_back(token);
        }

        map<char, string> args;
        set<char> flags;
        int pos = 1;
        while (pos < (int)tokens.size()) {
            string cur = tokens[pos];
            if (cur.size() != 2 || cur[0] != '-' || !accepts.count(cur[1])) {
                break;
            }
            char option = cur[1];
            if (accepts[option]) {
                if (pos + 1 >= (int)tokens.size()) {
                    break;
                }
                args[option] = tokens[pos + 1];
                flags.insert(option);
                pos += 2;
            } else {
                flags.insert(option);
                pos += 1;
            }
        }

        cout << "Case " << case_id << ":";
        for (char option : flags) {
            cout << " -" << option;
            if (accepts[option]) {
                cout << ' ' << args[option];
            }
        }
        cout << '\n';
    }
    return 0;
}
```

### 第02次 T3 字符串匹配

**题意抓手**

给定模式串 `S`、大小写敏感开关和 `n` 行文本，输出所有包含 `S` 的原始文本行。

**知识点**

子串查找和大小写归一。C++ 中 `string::find` 返回 `string::npos` 表示没找到。

**解法**

若大小写敏感，直接用原模式串和原文本行匹配。若不敏感，先把模式串转成小写，每一行也转成小写后再判断；输出时仍输出输入的原行。

**易错点**

- 不敏感匹配只影响比较，不影响输出。
- 输入行不含空格，可以直接用 `cin >> line`。
- `tolower` 参数最好转成 `unsigned char`，避免字符符号扩展造成未定义行为。

**复杂度**

设总字符数为 `T`，每行做一次转换和查找，时间复杂度 `O(T)`，空间复杂度为当前字符串副本 `O(L)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

string lower_copy(string s) {
    for (char& ch : s) {
        ch = (char)tolower((unsigned char)ch);
    }
    return s;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string pattern;
    int sensitive;
    int n;
    cin >> pattern >> sensitive >> n;
    string query = sensitive ? pattern : lower_copy(pattern);
    while (n--) {
        string line;
        cin >> line;
        string target = sensitive ? line : lower_copy(line);
        if (target.find(query) != string::npos) {
            cout << line << '\n';
        }
    }
    return 0;
}
```
