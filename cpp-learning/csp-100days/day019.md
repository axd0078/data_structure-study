# Day 019：T2 模拟入门

## 学习内容

Day 019 开始把早期 T2 放进训练。T2 不一定比 T1 算法更难，但更考验“状态维护”和“覆盖关系”。今天两道题都属于小规模模拟：窗口要维护层级，画图要维护哪些单位格已经被覆盖。

**窗口**的状态是一组矩形从底到顶的顺序。每次点击时要从顶层往底层找第一个包含点击点的窗口；找到后输出窗口编号，并把这个窗口移动到最顶层。因为 `N,M <= 10`，用 `vector` 直接 `erase + push_back` 最清楚。

**画图**要统计多个矩形覆盖后的总面积。坐标范围不超过 100，可以把每个单位小格看成一个布尔值。矩形 `(x1,y1)` 到 `(x2,y2)` 覆盖的是：

```text
x = x1..x2-1
y = y1..y2-1
```

注意右边界和上边界不作为单位格起点参与循环。

## 题解

### 第01次 T2 窗口

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

有 `N` 个矩形窗口，输入顺序是从最下层到最上层。每次点击一个点：

- 如果点属于某个或多个窗口，选择最上层的那个，输出它的编号，并把它移动到最上层。
- 如果点不属于任何窗口，输出 `IGNORED`，层级不变。

窗口边界上的点也属于窗口。

**知识点**

把窗口数组维护为“从底到顶”的顺序。顶层就是数组末尾。点击时从末尾向前扫描，遇到第一个包含该点的窗口即可停止。命中后移动到顶层，可以先保存窗口，再从原位置删除，最后插入末尾。

**解法**

1. 定义窗口结构体，保存编号和矩形坐标。
2. 按输入顺序放入 `vector<Window>`。
3. 每次点击：
   - 从 `windows.size()-1` 向 0 扫描。
   - 判断 `x1 <= x <= x2 && y1 <= y <= y2`。
   - 没找到输出 `IGNORED`。
   - 找到则输出编号，并 `erase` 后 `push_back`。

**易错点**

- 边界点属于窗口，所以比较要用 `<=`。
- 输入窗口编号按原始输入顺序固定，移动层级不改变编号。
- 命中后要置顶；如果忘记更新层级，后续查询会错。
- 点击空白区域时不能改变任何层级。

**复杂度**

`N,M <= 10`。每次点击最多扫描 `N` 个窗口，移动一次窗口也是 `O(N)`，总时间复杂度 `O(MN)`，空间复杂度 `O(N)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

struct Window {
    int id;
    int x1, y1, x2, y2;
};

bool contains(const Window& w, int x, int y) {
    return w.x1 <= x && x <= w.x2 && w.y1 <= y && y <= w.y2;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    vector<Window> windows;
    for (int i = 0; i < n; ++i) {
        Window w;
        w.id = i + 1;
        cin >> w.x1 >> w.y1 >> w.x2 >> w.y2;
        windows.push_back(w);
    }

    while (m--) {
        int x, y;
        cin >> x >> y;

        int hit = -1;
        for (int i = (int)windows.size() - 1; i >= 0; --i) {
            if (contains(windows[i], x, y)) {
                hit = i;
                break;
            }
        }

        if (hit == -1) {
            cout << "IGNORED\n";
        } else {
            Window chosen = windows[hit];
            windows.erase(windows.begin() + hit);
            windows.push_back(chosen);
            cout << chosen.id << '\n';
        }
    }

    return 0;
}
```

### 第02次 T2 画图

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给出多个矩形，把这些矩形覆盖的区域涂色。重叠区域只计算一次，输出最终被涂色的单位面积数量。

**知识点**

当坐标是整数、范围很小时，可以把平面拆成单位格。单位格 `(x,y)` 表示左下角为 `(x,y)`、右上角为 `(x+1,y+1)` 的小正方形。

矩形 `(x1,y1)` 到 `(x2,y2)` 覆盖的单位格是半开范围：

```text
x1 <= x < x2
y1 <= y < y2
```

用 `bool painted[101][101]` 标记是否已经被覆盖，重复覆盖不会重复计数。

**解法**

1. 建立布尔网格，初始为 false。
2. 对每个矩形：
   - 枚举 `x=x1..x2-1`。
   - 枚举 `y=y1..y2-1`。
   - 标记 `painted[x][y]=true`。
3. 扫描网格，统计 true 的数量。

**易错点**

- 循环到 `< x2`、`< y2`，不是 `<=`。
- 重叠区域只算一次，所以不能直接把每个矩形面积相加。
- 坐标最大为 100，数组开到 101 或稍大都可以。

**复杂度**

坐标范围固定为 100。时间复杂度 `O(n * 100 * 100)` 的上界很小，空间复杂度 `O(100 * 100)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    bool painted[101][101] = {};
    while (n--) {
        int x1, y1, x2, y2;
        cin >> x1 >> y1 >> x2 >> y2;
        for (int x = x1; x < x2; ++x) {
            for (int y = y1; y < y2; ++y) {
                painted[x][y] = true;
            }
        }
    }

    int answer = 0;
    for (int x = 0; x <= 100; ++x) {
        for (int y = 0; y <= 100; ++y) {
            answer += painted[x][y];
        }
    }

    cout << answer << '\n';
    return 0;
}
```
