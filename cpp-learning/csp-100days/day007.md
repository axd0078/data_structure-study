# Day 007：二维符号与分类判断

## 学习内容

Day 007 从普通线性模拟过渡到“用状态或符号做判断”。今天两道题都不难，但都要求你把题目里的规则转换成可维护的变量。

- **报数**：关键是“报出的数”和“被跳过的数”不是一回事。循环中每个自然数都会轮到一个人，但只有没被跳过的数才计入已经报出的数量。
- **线性分类器**：直线把平面分成两侧。对每个点计算 `theta0 + theta1*x + theta2*y` 的正负号，就能知道它在哪一侧；如果所有 A 同号、所有 B 同号，且两类符号相反，就可以完美分隔。

复杂度选择：

- 报数数据范围小，顺序模拟最直接，时间 `O(last)`，其中 `last` 是游戏结束时检查到的最后一个自然数。
- 线性分类器每条直线都要检查所有点，时间 `O(nm)`，空间 `O(n)` 存训练点。

## 题解

### 第18次 T1 报数

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

四个人轮流从 1 开始报数。如果当前数是 7 的倍数，或十进制表示中含有数字 7，则当前人跳过。游戏在成功报出 `n` 个数后结束，跳过的数不计入这 `n` 个数。输出四个人各跳过几次。

**知识点**

需要同时维护两个计数：

- `current`：当前轮到的自然数，每轮都加 1。
- `reported`：已经成功报出的数，只在当前数没有被跳过时加 1。

当前人编号由 `(current - 1) % 4` 决定。

**解法**

1. 初始化四个人跳过次数为 0。
2. 从 `current = 1` 开始模拟。
3. 如果 `current` 是 7 的倍数或含数字 7，当前人跳过次数加 1。
4. 否则成功报数，`reported++`。
5. 当 `reported == n` 时停止。

**易错点**

- 被跳过的数不计入成功报出的 `n` 个数。
- 人员轮换按自然数轮换，不是按成功报出的数轮换。
- 含数字 7 包括 7、17、70、171 等。

**复杂度**

时间复杂度 `O(last)`，额外空间复杂度 `O(1)`。本题 `n <= 666`，顺序模拟足够且最稳。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

bool shouldSkip(int value) {
    if (value % 7 == 0) {
        return true;
    }
    while (value > 0) {
        if (value % 10 == 7) {
            return true;
        }
        value /= 10;
    }
    return false;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    array<int, 4> skipped{};
    int reported = 0;
    int current = 1;
    while (reported < n) {
        int player = (current - 1) % 4;
        if (shouldSkip(current)) {
            ++skipped[player];
        } else {
            ++reported;
        }
        ++current;
    }

    for (int count : skipped) {
        cout << count << '\n';
    }
    return 0;
}
```

### 第19次 T1 线性分类器

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给出 `n` 个带类别的点，以及 `m` 条候选直线。每条直线形如 `theta0 + theta1*x + theta2*y = 0`。判断每条直线能否把 A 类点和 B 类点完全分在两侧。

题目保证没有点恰好落在查询直线上。

**知识点**

对一个点 `(x, y)`，计算：

```text
value = theta0 + theta1*x + theta2*y
```

`value > 0` 和 `value < 0` 表示直线两侧。能完美分隔需要满足：

- 所有 A 类点符号相同。
- 所有 B 类点符号相同。
- A 类和 B 类的符号不同。

坐标和参数绝对值可到 `10^4`，乘法和加法建议使用 `long long`。

**解法**

1. 读入所有点。
2. 对每条直线，分别记录 A 类点符号和 B 类点符号。
3. 扫描所有点：
   - 第一次遇到某类点时记录符号。
   - 后续同类点如果符号不同，则该直线不合法。
4. 扫描结束后，如果两类符号相反，输出 `Yes`，否则输出 `No`。

**易错点**

- 不需要求直线斜率，直接用一般式代入。
- 不能只检查 A 类或只检查 B 类，两类内部都必须各自同侧。
- 输出是 `Yes` / `No`，大小写固定。
- 使用 `long long` 避免乘法边界。

**复杂度**

时间复杂度 `O(nm)`，额外空间复杂度 `O(n)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

struct Point {
    long long x;
    long long y;
    char type;
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    vector<Point> points(n);
    for (auto &point : points) {
        cin >> point.x >> point.y >> point.type;
    }

    while (m--) {
        long long theta0, theta1, theta2;
        cin >> theta0 >> theta1 >> theta2;

        int signA = 0;
        int signB = 0;
        bool ok = true;
        for (const auto &point : points) {
            long long value = theta0 + theta1 * point.x + theta2 * point.y;
            int sign = value > 0 ? 1 : -1;
            int &stored = point.type == 'A' ? signA : signB;
            if (stored == 0) {
                stored = sign;
            } else if (stored != sign) {
                ok = false;
                break;
            }
        }

        cout << (ok && signA != signB ? "Yes" : "No") << '\n';
    }

    return 0;
}
```
