# Day 012：线性扫描与边界裁剪

## 学习内容

Day 012 的两道题都能在线性时间内完成。它们训练的是把题面里的数学或几何定义转成稳定的循环：一个是按时间折现的累加，一个是矩形交集面积的累加。

**现值计算**要掌握浮点累乘。第 `k` 年现金流 `v_k` 的现值是 `v_k / (1+i)^k`，总现值就是所有年份相加。实现时不需要每次调用 `pow`，可以维护当前折现分母 `discount`，每处理一年后乘上 `1+i`。这样代码更短，也少做重复计算。

**田地丈量**要掌握区间交集。两个矩形的交集面积等于横向交集长度乘纵向交集长度。横向交集长度为：

```text
max(0, min(x2, a) - max(x1, 0))
```

纵向同理。只要矩形没有交集，其中一个长度就是 0，面积自然为 0。

今天的最低复杂度目标：

- 现值计算：`O(n)` 时间，`O(1)` 空间。
- 田地丈量：`O(n)` 时间，`O(1)` 空间。

## 题解

### 第28次 T1 现值计算

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

输入年份数 `n`、利率 `i`，以及 `v_0..v_n` 共 `n+1` 个现金流。要求计算：

```text
sum(v_k * (1+i)^(-k))    k = 0..n
```

也就是把未来每一年的现金流折算到第 0 年。

**知识点**

`(1+i)^(-k)` 等价于 `1 / (1+i)^k`。从 `k=0` 开始，当前分母是 `1`；每读完一个现金流，下一年的分母乘上 `1+i`。

这种维护递推量的写法比每次 `pow(1+i, -k)` 更直接，时间复杂度仍是线性的，常数更低，浮点误差也足够通过本题。

**解法**

1. 读入 `n` 和 `rate`。
2. 初始化 `total = 0`，`discount = 1`。
3. 循环 `k=0..n`：
   - 读入 `value`。
   - 累加 `value / discount`。
   - 更新 `discount *= 1 + rate`。
4. 输出 `total`。

**易错点**

- 一共有 `n+1` 个现金流，不是 `n` 个。
- `i` 是利率变量，C++ 中可以叫 `rate`，避免和循环变量混淆。
- 输出允许误差，`fixed << setprecision(6)` 足够。
- 现金流可能为负数，不要取绝对值。

**复杂度**

时间复杂度 `O(n)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    double rate;
    cin >> n >> rate;

    double total = 0.0;
    double discount = 1.0;
    for (int k = 0; k <= n; ++k) {
        double value;
        cin >> value;
        total += value / discount;
        discount *= 1.0 + rate;
    }

    cout << fixed << setprecision(6) << total << '\n';
    return 0;
}
```

### 第29次 T1 田地丈量

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

目标区域是矩形 `[0,a] x [0,b]`。输入 `n` 块田地，每块田地也是轴对齐矩形 `(x1,y1,x2,y2)`。要求计算所有田地与目标区域的交集面积总和。题目保证田地之间没有重叠面积，所以每块田地的交集面积直接相加即可。

**知识点**

一维区间 `[l1,r1]` 和 `[l2,r2]` 的交集长度是：

```text
max(0, min(r1,r2) - max(l1,l2))
```

矩形是两个方向上的区间乘积。横向长度乘纵向长度就是交集面积。

**解法**

1. 读入 `n,a,b`。
2. 对每个田地矩形：
   - `width = max(0, min(x2,a) - max(x1,0))`
   - `height = max(0, min(y2,b) - max(y1,0))`
   - `answer += width * height`
3. 输出 `answer`。

**易错点**

- 坐标可能为负，不能只判断是否在第一象限。
- 右上角可能超过目标矩形，要用 `min` 裁剪。
- 不相交时长度要压成 0，不能让负数参与面积计算。
- 面积虽然本题范围不大，但用 `long long` 是更稳的习惯。

**复杂度**

时间复杂度 `O(n)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    long long a, b;
    cin >> n >> a >> b;

    long long total = 0;
    for (int i = 0; i < n; ++i) {
        long long x1, y1, x2, y2;
        cin >> x1 >> y1 >> x2 >> y2;

        long long width = max(0LL, min(x2, a) - max(x1, 0LL));
        long long height = max(0LL, min(y2, b) - max(y1, 0LL));
        total += width * height;
    }

    cout << total << '\n';
    return 0;
}
```
