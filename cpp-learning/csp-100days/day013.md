# Day 013：状态编码与操作合并

## 学习内容

Day 013 的核心是“不要把过程模拟得太重”。有些题看起来要处理棋盘、处理一串操作，但只要找到合适的状态表示或合并规律，就能把实现压到很短。

**重复局面**要把二维棋盘编码成字符串。每个局面固定是 `8 x 8`，把 8 行拼成长度 64 的字符串后，就可以作为哈希表的键。每读到一个局面，计数加 1 并输出当前次数。

**坐标变换（其一）**只有平移操作。连续平移满足可加性：先加 `(dx1,dy1)`，再加 `(dx2,dy2)`，等价于一次加 `(dx1+dx2, dy1+dy2)`。因此不需要对每个点重复执行所有操作，只要先求总平移量，再处理每个点。

今天的最低复杂度目标：

- 重复局面：`O(n * 64)` 平均时间，`O(n * 64)` 空间保存出现过的局面。
- 坐标变换（其一）：`O(n+m)` 时间，`O(1)` 空间。

## 题解

### 第30次 T1 重复局面

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

输入 `n` 个国际象棋局面。每个局面由 8 行字符串表示，每行 8 个字符。对于第 `i` 个局面，输出它在前 `i` 个局面中已经出现了多少次。

**知识点**

二维棋盘可以转成一维字符串。只要转换规则一致，同一个棋盘一定得到同一个字符串，不同棋盘也会在某个位置字符不同。固定 `8 x 8` 的局面可以直接拼接：

```text
state = row1 + row2 + ... + row8
```

然后用 `unordered_map<string,int>` 统计出现次数。

**解法**

1. 读入 `n`。
2. 对每个局面：
   - 清空字符串 `state`。
   - 读入 8 行，依次拼到 `state`。
   - `++count[state]`，输出更新后的次数。

**易错点**

- 每个局面正好读 8 行，不要受 `n` 的循环影响读错层级。
- 不要只统计每一行；要把完整 64 个字符作为一个局面。
- 变量名不要写成 `map`，会遮住 STL 容器名。
- 哈希表平均复杂度更低；本题不需要按字典序输出。

**复杂度**

每个局面固定 64 个字符，时间复杂度 `O(n * 64)` 平均，空间复杂度 `O(k * 64)`，其中 `k` 是不同局面数。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    unordered_map<string, int> count;

    for (int step = 0; step < n; ++step) {
        string state;
        state.reserve(64);

        string row;
        for (int i = 0; i < 8; ++i) {
            cin >> row;
            state += row;
        }

        cout << ++count[state] << '\n';
    }

    return 0;
}
```

### 第31次 T1 坐标变换（其一）

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

有 `n` 个平移操作，每个操作把点 `(x,y)` 变成 `(x+dx_i,y+dy_i)`。之后给出 `m` 个初始坐标，要求输出每个点依次经过全部 `n` 个操作后的最终坐标。

**知识点**

平移操作可以合并。所有操作的总效果是：

```text
total_x = dx_1 + dx_2 + ... + dx_n
total_y = dy_1 + dy_2 + ... + dy_n
```

每个点最终就是：

```text
(x + total_x, y + total_y)
```

如果对每个点都重新跑一遍 `n` 个操作，复杂度会变成 `O(nm)`；先合并操作后只需要 `O(n+m)`。

**解法**

1. 读入 `n,m`。
2. 累加所有操作的 `dx,dy`。
3. 对每个初始点 `(x,y)`，输出 `(x+total_x, y+total_y)`。

**易错点**

- 坐标和位移绝对值可到 `1e5`，操作数最多 100，`int` 够用，但用 `long long` 更稳。
- 输入先是全部操作，再是全部坐标。
- 输出每个点一行，两个数之间一个空格。

**复杂度**

时间复杂度 `O(n+m)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    long long dx = 0;
    long long dy = 0;
    for (int i = 0; i < n; ++i) {
        long long x, y;
        cin >> x >> y;
        dx += x;
        dy += y;
    }

    for (int i = 0; i < m; ++i) {
        long long x, y;
        cin >> x >> y;
        cout << x + dx << ' ' << y + dy << '\n';
    }

    return 0;
}
```
