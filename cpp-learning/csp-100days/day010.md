# Day 010：T1 保分复盘

## 学习内容

Day 010 是前 10 天 T1 基础题的一次收束。今天两道题都体现了一个原则：题目定义看起来像“查询”或“代码检查”，真正实现时要把它压成一次线性扫描。

- **序列查询**：`f(x)` 在每个区间 `[A_i, A_{i+1})` 上恒等于 `i`，所以一段一段累加贡献，不要逐个枚举 `x`。
- **未初始化警告**：变量是否初始化只取决于它是否曾经作为左值出现过，常量 `a_0` 永远可用。

今天的最低复杂度：

- 序列查询：`O(n)` 时间，`O(1)` 额外空间，流式处理相邻边界。
- 未初始化警告：`O(k)` 时间，`O(n)` 空间记录初始化状态。

## 题解

### 第24次 T1 序列查询

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给定严格递增序列：

```text
0 = A_0 < A_1 < ... < A_n < N
```

对每个 `x in [0,N)`，`f(x)` 是不超过 `x` 的最大 `A_i` 的下标。要求计算：

```text
sum(A) = f(0) + f(1) + ... + f(N-1)
```

**知识点**

当 `x` 落在区间 `[A_i, A_{i+1})` 时，`f(x)` 恒等于 `i`。因此这一整段的贡献是：

```text
i * (A_{i+1} - A_i)
```

最后还有 `[A_n, N)`，贡献是：

```text
n * (N - A_n)
```

不需要枚举 `x`，因为 `N` 可到 `10^7`；也不需要存下整个数组，读到当前边界就能结算上一段。

**解法**

1. 初始化 `previous = 0`。
2. 依次读入 `A_1..A_n`。
3. 读到第 `index` 个边界 `current` 时，区间 `[previous,current)` 的 `f(x)` 为 `index-1`，累加贡献。
4. 更新 `previous = current`。
5. 所有边界读完后，累加最后一段 `[previous,N)` 的贡献 `n * (N-previous)`。

**易错点**

- 输入不包含 `A_0=0`，需要自己补。
- 区间是左闭右开 `[A_i,A_{i+1})`。
- 最后一段到 `N`，不是到 `N-1` 的长度手动加一；长度就是 `N-A_n`。
- 答案可能超过 `int`，用 `long long`。

**复杂度**

时间复杂度 `O(n)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    long long limit;
    cin >> n >> limit;

    long long previous = 0;
    long long ans = 0;
    for (int index = 1; index <= n; index++) {
        long long current;
        cin >> current;
        ans += 1LL * (index - 1) * (current - previous);
        previous = current;
    }
    ans += 1LL * n * (limit - previous);

    cout << ans << '\n';
    return 0;
}
```

### 第25次 T1 未初始化警告

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

有 `n` 个变量 `a_1..a_n`，常量统一记作 `a_0`。每条语句是：

```text
a_x = a_y
```

左值 `a_x` 执行后会被初始化。右值 `a_y` 如果是变量且之前没有作为左值出现过，就产生一次未初始化警告。常量 `a_0` 永远不产生警告。

**知识点**

只需要维护每个变量是否已经初始化：

- 初始时 `a_1..a_n` 都未初始化。
- `a_0` 是常量，可以看作始终已初始化。
- 每读一条语句，先检查右值，再标记左值。

检查顺序很重要：对于 `a_3 = a_3`，如果 `a_3` 此前未初始化，这条语句仍然要报警，然后才把 `a_3` 标记为已初始化。

**解法**

1. 创建 `initialized[n+1]`，初始全 false。
2. 对每条语句读入 `x,y`。
3. 如果 `y != 0 && !initialized[y]`，答案加 1。
4. 标记 `initialized[x] = true`。
5. 输出答案。

**易错点**

- `a_0` 是常量，不报警。
- 先检查右值，再初始化左值。
- 右值是否初始化只看它此前是否作为左值出现过，不受那条历史语句自身是否报警影响。
- `n,k` 可到 `100000`，用数组或 `vector<char>`。

**复杂度**

时间复杂度 `O(k)`，额外空间复杂度 `O(n)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, k;
    cin >> n >> k;
    vector<char> initialized(n + 1, 0);
    int warnings = 0;
    for (int i = 0; i < k; i++) {
        int x, y;
        cin >> x >> y;
        if (y != 0 && !initialized[y]) {
            ++warnings;
        }
        initialized[x] = 1;
    }
    cout << warnings << '\n';
    return 0;
}
```
