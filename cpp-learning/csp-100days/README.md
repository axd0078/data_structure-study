# 100 天 CSP 系统学习计划

## 模块简介

`cpp-learning/csp-100days` 用于把 CSP 认证真题拆成 100 天的系统学习路线。当前版本先完成路线骨架和题解占位：每天先明确学习主题和需要完成的题目，后续你指定某一天时，再补齐当天的详细知识讲解、题解、参考代码和本地测试环境。

## 覆盖范围

- 覆盖来源：`../csp-certification-problems/`。
- 覆盖题目：第 1-41 次 CSP 认证，每次 T1-T5，共 205 道题。
- 文件组织：`day001.md` 到 `day100.md`。
- 题解策略：当前只预留题解结构，不一次性补完所有题解。

## 阶段安排

| 阶段 | 天数 | 重点 |
|---|---:|---|
| T1/T2 保分基础 | Day001-Day020 | 输入输出、计数、数组、排序、基础模拟、前缀和 |
| 字符串、矩阵与中档建模 | Day021-Day045 | 字符串、矩阵、STL 容器、时间线模拟、分段贡献、二分、贪心 |
| 搜索、图论、DP 与数据结构 | Day046-Day070 | 搜索、图论、拓扑、动态规划、树和区间数据结构 |
| 系统模拟、协议解析与复杂状态 | Day071-Day090 | 系统类大模拟、协议解析、编码压缩、复杂状态维护 |
| T4/T5 综合建模 | Day091-Day100 | 高难综合题、部分分策略、整套复盘 |

## 按天补齐流程

之后你可以直接说“做 Day 001”或“补 Day 037”。补当天时按以下顺序推进：

1. 阅读当天题面和仓库中已有练习代码。
2. 扩写 `学习内容`，把当天题目涉及的基础知识讲详细。
3. 补齐每道题的题意抓手、知识点、解法、易错点、复杂度和 C++ 参考代码。
4. 按 `../csp-test-env/` 的约定补本地测试环境：题库注册、参考程序和 10 组测试数据。
5. 运行测试数据生成、单元测试、参考程序校验和练习范围校验。
