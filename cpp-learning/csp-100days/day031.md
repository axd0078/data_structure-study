# Day 031：矩阵结合律与变换前缀

## 学习内容

今天两题的共同点是“不要按题面顺序硬模拟”。

矩阵运算表面是 `(W dot (QK^T))V`，如果先算 `QK^T` 会得到 `n x n` 矩阵，`n` 可到 `10^4`，不仅慢也占内存。利用矩阵乘法结合律，把 `K^T V` 先算成 `d x d`，再算 `Q(K^T V)`，最后每行乘上 `W_i`，复杂度从接近 `O(n^2d)` 降到 `O(nd^2)`。

坐标变换（其二）中，拉伸是统一乘一个比例，旋转是累加角度；统一缩放和绕原点旋转可以交换顺序。因此一段操作可以被压缩成两个量：缩放乘积和角度和。用前缀积、前缀和回答区间查询即可。

## 题解

### 第30次 T2 矩阵运算

**题意抓手**

给定 `n x d` 的 `Q,K,V` 和长度为 `n` 的 `W`，计算 `(W dot (QK^T))V`。其中 `W dot` 表示把 `QK^T` 的第 `i` 行整体乘以 `W_i`。

**知识点**

矩阵乘法结合律。原式第 `i` 行第 `c` 列为：

```text
W_i * sum_h Q[i][h] * sum_j K[j][h] * V[j][c]
```

内层 `sum_j K[j][h] * V[j][c]` 正是 `(K^T V)[h][c]`。

**解法**

先计算 `K^T V`，这是一个 `d x d` 矩阵，复杂度 `O(nd^2)`。再对每一行 `Q_i` 乘这个 `d x d` 矩阵，得到输出行，最后乘 `W_i`。所有中间结果用 `long long`。

**易错点**

- 不能真的开 `n x n` 的 `QK^T`。
- 输入元素和 `W` 可能为负。
- 乘法累计量可达 `1e17` 级别，必须用 64 位。

**复杂度**

时间复杂度 `O(nd^2)`，空间复杂度 `O(nd + d^2)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, d;
    cin >> n >> d;
    vector<vector<long long>> q(n, vector<long long>(d));
    vector<vector<long long>> k(n, vector<long long>(d));
    vector<vector<long long>> v(n, vector<long long>(d));

    for (auto& row : q) for (auto& value : row) cin >> value;
    for (auto& row : k) for (auto& value : row) cin >> value;
    for (auto& row : v) for (auto& value : row) cin >> value;

    vector<long long> w(n);
    for (auto& value : w) cin >> value;

    vector<vector<long long>> ktv(d, vector<long long>(d, 0));
    for (int row = 0; row < d; ++row) {
        for (int col = 0; col < d; ++col) {
            for (int i = 0; i < n; ++i) {
                ktv[row][col] += k[i][row] * v[i][col];
            }
        }
    }

    for (int i = 0; i < n; ++i) {
        for (int col = 0; col < d; ++col) {
            long long value = 0;
            for (int mid = 0; mid < d; ++mid) {
                value += q[i][mid] * ktv[mid][col];
            }
            if (col) cout << ' ';
            cout << value * w[i];
        }
        cout << '\n';
    }

    return 0;
}
```

### 第31次 T2 坐标变换（其二）

**题意抓手**

有 `n` 个操作：类型 1 为坐标统一拉伸 `k` 倍，类型 2 为绕原点逆时针旋转 `theta`。每次查询给出区间 `[i,j]` 和点 `(x,y)`，求依次执行这些操作后的坐标。

**知识点**

统一缩放和旋转都以原点为中心，组合结果仍然是“缩放 `s` 倍 + 旋转 `theta` 弧度”。缩放系数相乘，旋转角度相加。

**解法**

预处理：

- `scale[p]`：前 `p` 个操作的缩放乘积；
- `angle[p]`：前 `p` 个操作的旋转角度和。

查询 `[l,r]` 时，缩放为 `scale[r]/scale[l-1]`，角度为 `angle[r]-angle[l-1]`。再套旋转公式。

**易错点**

- 旋转单位是弧度。
- 查询下标从 1 开始。
- 输出浮点数即可，评测允许误差；本地参考程序保留 10 位小数。

**复杂度**

预处理 `O(n)`，每次查询 `O(1)`，空间复杂度 `O(n)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;
    vector<double> scale(n + 1, 1.0), angle(n + 1, 0.0);

    for (int i = 1; i <= n; ++i) {
        int type;
        double value;
        cin >> type >> value;
        scale[i] = scale[i - 1];
        angle[i] = angle[i - 1];
        if (type == 1) scale[i] *= value;
        else angle[i] += value;
    }

    cout << fixed << setprecision(10);
    while (m--) {
        int l, r;
        double x, y;
        cin >> l >> r >> x >> y;
        double s = scale[r] / scale[l - 1];
        double theta = angle[r] - angle[l - 1];
        double nx = s * (x * cos(theta) - y * sin(theta));
        double ny = s * (x * sin(theta) + y * cos(theta));
        cout << nx << ' ' << ny << '\n';
    }

    return 0;
}
```
