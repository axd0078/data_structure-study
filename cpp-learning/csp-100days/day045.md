# Day 045：编码输出与递归计数

## 学习内容

今天收束 Day41-45 的字符串与结构化建模训练。第17次 T3《字符画》把图像块平均成背景色，再输出终端控制序列的十六进制转义；第18次 T3《化学方程式》把化学式递归解析成元素计数表，再比较等号左右是否守恒。

《字符画》的难点不在平均颜色，而在输出状态管理。每个输出字符块代表原图中 `p*q` 个像素的平均色。终端背景色如果和当前状态不同，才输出设置背景色的控制序列；一行结束时如果背景不是默认黑色，要输出重置序列。最后不是直接输出原始控制字符，而是把原始输出中的每个字节转成 `\xHH`。

《化学方程式》的难点是递归乘法。一个物质可能有前导系数，括号内也可能有下标系数，例如 `3Mg(OH)2`。解析时函数返回一个 `map<element,count>`，读到元素符号就加数量，读到括号就递归解析括号内公式并整体乘以后面的数字。等式左边所有物质计数相加，右边同理，最后比较两个 map。

这两题都不是靠套模板，而是靠“状态机 + 小函数”。字符画维护当前颜色状态；化学式维护当前位置和当前计数表。状态只有一种解释，代码才不容易乱。

## 题解

### 第17次 T3 字符画

**题意抓手**

给出一张由十六进制颜色表示的图片，把每个 `p*q` 像素块压缩成一个字符空格，并用 ANSI 背景色控制序列显示平均颜色。题目要求输出的是原始字节流的十六进制转义形式。

**知识点**

颜色格式解析、二维分块、输出状态压缩。颜色可能是 `#abcdef`、`#abc` 或 `#a`，短格式需要按 CSS 规则扩展到 8 位通道。

**解法**

先把所有像素转成 `(r,g,b)`。按块遍历：对块内 `p*q` 个像素的三个通道分别求平均，向下取整。维护当前背景色 `current`，初始为黑色。如果下一个块需要的颜色与 `current` 不同，输出原始控制序列：非黑色输出 `ESC[48;2;r;g;bm`，黑色输出 `ESC[0m`。随后输出一个空格。每行结束时，如果当前背景色不是黑色，输出重置序列并把状态改回黑色，再输出换行。最后遍历原始字节串，把每个字节输出成大写 `\xHH`。

**易错点**

- `#abc` 要扩展为 `#aabbcc`，`#a` 要扩展为灰度 `#aaaaaa`。
- 平均值是整数除法向下取整。
- 控制序列中的 ESC 是字节 `0x1B`，最终要输出为 `\x1B`。
- 行末必须重置背景色，否则下一行初始状态会错。
- 输出的十六进制字母要大写。

**复杂度**

设图片大小为 `m*n`。每个像素只参与一次求和，时间复杂度 `O(mn)`；空间复杂度 `O(mn)`，也可以边读边处理块但实现会复杂一些。

### 第18次 T3 化学方程式

**题意抓手**

给出若干化学方程式，判断每个方程式左右两边各种元素的原子个数是否相等。输出 `Y` 或 `N`。

**知识点**

递归下降解析、map 计数、系数乘法。化学式由元素、数字、括号组成。元素名以大写字母开头，后面可能跟小写字母；数字缺省为 1。

**解法**

把方程式按 `=` 分成左右，再按 `+` 分成多个物质。解析单个物质时，先读取可能存在的前导系数，然后解析后面的公式并整体乘以前导系数。

解析公式使用递归函数：遇到大写字母就读完整元素名，再读后续数字作为个数；遇到 `(` 就递归解析直到对应 `)`，然后读取括号后的数字，把括号内部计数整体乘上该倍数；遇到 `)` 或字符串结束则返回。左右两边分别累加所有物质的元素计数，最后比较两个 map 是否完全相同。

**易错点**

- 前导系数作用于整个物质，不只作用于第一个元素。
- 括号后的下标作用于括号内所有元素。
- 元素名可能有多个小写字母，例如 `Mg`、`Na`，不能只读一个字符。
- 缺省数字是 1，不是 0。
- 比较时元素缺失等价于计数 0，但用 map 累加后直接比较最简单。

**复杂度**

设一个方程式长度为 `L`，不同元素数为 `E`。解析时间复杂度 `O(L log E)`，使用哈希表可近似 `O(L)`；空间复杂度 `O(E + recursion_depth)`。

完整可编译参考程序见 `cpp-learning/csp-test-env/reference_solutions/18/3.cpp`。
