# Day 044：区间归并与磁盘映射

## 学习内容

今天进入中档题里比较“工程感”的两类建模。第15次 T3《CIDR 合并》把 IP 前缀看成二进制区间，做排序、包含删除和兄弟前缀合并；第16次 T3《损坏的 RAID5》把逻辑块映射到 RAID5 的数据盘、校验盘和盘内块位置。

《CIDR 合并》先要统一表示。无论输入是 `a.b.c.d/p`、`a.b.c/p` 还是 `a.b`，都转成 `(ip, len)`，其中 `ip` 是 32 位整数且主机位清零，`len` 是前缀长度。两个前缀的包含关系可以用区间判断，也可以用掩码判断。合并时只有两个同长度、相邻、且父前缀相同的前缀才能变成长度减一的父前缀。

《损坏的 RAID5》重点是映射公式。每个条带有 `s` 个块，一行 RAID 有 `n-1` 个数据条带和 1 个校验条带。第 `row` 行的校验盘是 `(n-1-row%n)`，逻辑块先定位到第几行、行内第几个数据条带、条带内偏移，再根据校验盘位置跳过校验盘得到实际磁盘号。若目标盘存在，直接输出；若目标盘缺失但只缺一盘，用同一行同一偏移的其他盘块异或恢复；否则输出 `-`。

这两题都适合把复杂规则写成函数：CIDR 写 `parse_prefix`、`contains`、`can_merge`；RAID 写 `locate(logical_block)`、`read_block(disk, index)`、`recover(row, offset)`。函数边界清楚后，主流程会非常短。

## 题解

### 第15次 T3 CIDR 合并

**题意抓手**

给出若干 IP 前缀，要求输出等价的最简前缀集合。最简包括两步：删除被其他前缀包含的前缀；把可以合并成父前缀的相邻兄弟前缀不断合并。

**知识点**

IPv4 整数化、二进制前缀、区间包含。一个 `len` 位前缀覆盖长度为 `2^(32-len)` 的连续地址区间。合并两个前缀时，必须长度相同、地址相邻，并且父前缀相同。

**解法**

先解析输入。缺省前缀长度由给出的段数决定，例如 `10.2` 表示 `10.2.0.0/16`；若显式给出 `/p`，就使用 `p`。无论哪种形式，都把主机位清零。

然后按 `(ip, len)` 排序。第一遍扫描删除被前一个保留前缀包含的项。第二步使用类似栈的归并：把前缀依次压入结果，每次检查栈顶两个前缀能否合并；能合并就弹出两个，压入父前缀，并继续向前检查，直到不能合并。

**易错点**

- 主机位必须清零，否则排序和包含都会出错。
- `0.0.0.0/0` 覆盖整个 IPv4 空间，位移时要避免 `1 << 32` 的 32 位溢出，用 `uint64_t`。
- 不能只看地址相邻，还要确认两个前缀属于同一个父前缀。
- 合并后可能继续和前面的前缀合并，所以要循环检查栈顶。

**复杂度**

设前缀数为 `N`。排序时间复杂度 `O(N log N)`，删除包含和栈式合并都是 `O(N)` 均摊；空间复杂度 `O(N)`。

### 第16次 T3 损坏的 RAID5

**题意抓手**

给定 RAID5 阵列参数、当前存在的磁盘数据和若干逻辑块查询。对每个逻辑块，如果能直接读取或通过校验恢复，就输出 8 位十六进制块内容；否则输出 `-`。

**知识点**

RAID5 条带映射和异或校验。每行有 `n` 个磁盘条带，其中一个是校验条带，校验盘随行号旋转。任意一块缺失时，可由同一行同一偏移的其他 `n-1` 块异或恢复。

**解法**

先根据任一存在磁盘的十六进制串长度得到每盘块数。查询逻辑块 `b` 时：

1. `row = b / ((n-1)*s)`，定位到第几行。
2. `inner = b % ((n-1)*s)`，`data_col = inner / s`，`offset = inner % s`。
3. 校验盘 `parity = n - 1 - row % n`。
4. 数据盘先等于 `data_col`，如果 `data_col >= parity`，实际盘号加一，跳过校验盘。
5. 盘内块号为 `row*s + offset`。

如果目标盘存在，直接截取对应 8 个十六进制字符。若不存在，检查是否只有一个盘缺失且其他盘对应块都存在，是则把其他盘块逐个异或恢复目标块；否则输出 `-`。

**易错点**

- 逻辑块数上界是 `blocks_per_disk * (n-1)`，超过直接输出 `-`。
- 十六进制块固定 8 个字符，输出要补足前导零并保持大写。
- 校验盘会随行号旋转，不是固定最后一盘。
- 恢复缺失数据块时，校验块也参与异或；恢复缺失校验块时同样可异或出校验块，但查询逻辑块通常定位到数据盘。

**复杂度**

设查询数为 `Q`，磁盘数为 `N`。直接读取 `O(1)`；恢复时需要异或至多 `N-1` 个块，所以总时间复杂度 `O(Q*N)`，空间复杂度为保存输入磁盘数据 `O(total_blocks)`。
