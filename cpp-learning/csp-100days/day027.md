# Day 027：二维前缀和与非零段变化

## 学习内容

今天的两题都要求把朴素模拟降到线性或近似线性。

邻域均值如果对每个像素都重新枚举邻域，复杂度会达到 `O(n^2 r^2)`，在 `n=600,r=100` 时没必要冒险。二维前缀和可以把任意矩形和降成 `O(1)`，再用 `sum <= t * count` 避免浮点平均数误差。

非零段划分看起来要枚举阈值 `p`，但数值上限只有 `10000`。可以从大到小激活等于当前值的位置，维护当前非零段数量。新增一个位置时，如果左右都不活跃，新增一段；如果左右都活跃，两个段被连接成一段；如果只有一边活跃，段数不变。

## 题解

### 第22次 T2 邻域均值

**题意抓手**

给定 `n x n` 灰度矩阵。对每个像素 `(i,j)`，取横纵距离都不超过 `r` 且仍在矩阵内的邻域。如果邻域平均值不超过阈值 `t`，这个像素算作较暗区域。输出较暗像素数。

**知识点**

二维前缀和：`prefix[i+1][j+1]` 表示左上角到 `(i,j)` 的矩形和。任意矩形 `[x1,x2] x [y1,y2]` 的和可以用四个前缀值相减得到。

**解法**

先建立二维前缀和。枚举每个像素，裁剪出邻域边界 `x1=max(0,i-r)`、`x2=min(n-1,i+r)`，列边界同理。用前缀和得到邻域总灰度 `sum` 和元素个数 `count`。判断平均值是否 `<=t` 时写成 `sum <= t * count`。

**易错点**

- 边界像素的邻域会被矩阵边界截断，元素个数不一定是 `(2r+1)^2`。
- 不要用浮点数比较平均值。
- 灰度和最大约 `600*600*255`，`int` 能放下，但写成通用代码时也可以用 `long long`。

**复杂度**

建前缀和 `O(n^2)`，每个像素查询 `O(1)`，总时间复杂度 `O(n^2)`，空间复杂度 `O(n^2)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, L, r, t;
    cin >> n >> L >> r >> t;

    vector<vector<int>> prefix(n + 1, vector<int>(n + 1, 0));
    for (int i = 0; i < n; ++i) {
        int row_sum = 0;
        for (int j = 0; j < n; ++j) {
            int value;
            cin >> value;
            row_sum += value;
            prefix[i + 1][j + 1] = prefix[i][j + 1] + row_sum;
        }
    }

    int ans = 0;
    for (int i = 0; i < n; ++i) {
        int x1 = max(0, i - r);
        int x2 = min(n - 1, i + r);
        for (int j = 0; j < n; ++j) {
            int y1 = max(0, j - r);
            int y2 = min(n - 1, j + r);
            int sum = prefix[x2 + 1][y2 + 1] - prefix[x1][y2 + 1] - prefix[x2 + 1][y1] + prefix[x1][y1];
            int count = (x2 - x1 + 1) * (y2 - y1 + 1);
            if (sum <= t * count) {
                ++ans;
            }
        }
    }

    cout << ans << '\n';
    return 0;
}
```

### 第23次 T2 非零段划分

**题意抓手**

任选正整数 `p`，把数组中所有小于 `p` 的数变成 0。问操作后非零连续段数量最多是多少。

**知识点**

阈值越低，保留的数越多。反过来看，从高到低降低阈值，就是不断“激活”等于当前值的位置。新增位置对段数的影响只取决于左右邻居是否已经激活。

**解法**

按数值建立桶 `pos[value]`，记录每个值出现的位置。然后从最大值降到 1 依次激活这些位置：

- 左右都未激活：新增一个非零段，`segments++`；
- 左右都已激活：连接两个非零段，`segments--`；
- 只有一边激活：扩展已有段，段数不变。

每处理完一个值，就用当前 `segments` 更新答案。

**易错点**

- 数组值可能为 0，0 永远不会被激活，因为 `p` 是正整数。
- 相同值对应同一个阈值状态，处理完整个桶后再更新答案最清晰。
- 原数组已是最优时，答案也会在扫描过程中被记录。

**复杂度**

数值上限 `V=10000`，时间复杂度 `O(n+V)`，空间复杂度 `O(n+V)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;
    vector<int> a(n + 1);
    int max_value = 0;
    for (int i = 1; i <= n; ++i) {
        cin >> a[i];
        max_value = max(max_value, a[i]);
    }

    vector<vector<int>> pos(max_value + 1);
    for (int i = 1; i <= n; ++i) {
        if (a[i] > 0) {
            pos[a[i]].push_back(i);
        }
    }

    vector<int> active(n + 2, 0);
    int segments = 0;
    int answer = 0;
    for (int value = max_value; value >= 1; --value) {
        for (int idx : pos[value]) {
            bool left = active[idx - 1];
            bool right = active[idx + 1];
            if (!left && !right) {
                ++segments;
            } else if (left && right) {
                --segments;
            }
            active[idx] = 1;
        }
        answer = max(answer, segments);
    }

    cout << answer << '\n';
    return 0;
}
```
