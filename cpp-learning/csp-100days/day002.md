# Day 002：线性扫描与简单统计

## 学习内容

Day 002 继续做 T1 保分题，主题是“线性扫描与简单统计”。这类题的共同点是：题意通常很短，算法也不复杂，但需要把数据读完整、处理顺序想清楚、输出格式写准确。

今天的三道题分别对应三种基础动作：

- **按坐标重排输出**：图像旋转不是复杂图形题，本质是矩阵下标变换。
- **连续段统计**：数列分段只关心相邻元素是否发生变化，扫一遍即可。
- **逐位拆数统计**：数位之和可以按字符串扫描，也可以对整数反复取模。

做这类题时，先问自己三个问题：

1. 是否需要保存全部数据？矩阵旋转需要按列输出，通常要保存矩阵；数列分段和数位之和不需要保存全部历史。
2. 答案在什么时候更新？看到新段时更新，看到新数字位时累加。
3. 输出形状是什么？矩阵题要控制行列和空格；单值题只输出一个整数。

今天建议手造这些边界：

- `1 x 1`、单行、单列矩阵。
- 全部元素都相同的数列。
- 每个相邻元素都不同的数列。
- `n = 0`、末尾有很多 `0`、最大值 `1000000000` 的数位之和。

## 题解

### 第04次 T1 图像旋转

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给定一个 `n x m` 的矩阵，输出它逆时针旋转 90 度后的矩阵。原矩阵有 `n` 行 `m` 列，旋转后会变成 `m` 行 `n` 列。

**知识点**

矩阵旋转题的关键是写清楚下标对应关系。逆时针旋转 90 度后，输出矩阵的行来自原矩阵的列；输出第一行应当是原矩阵最后一列，从上到下输出。

如果原矩阵为 `a[row][col]`，那么旋转后输出顺序是：

```text
for col = m-1..0:
    for row = 0..n-1:
        print a[row][col]
```

**解法**

1. 读入 `n, m`。
2. 保存整个矩阵 `a`。因为输出第 1 行时，需要访问原矩阵所有行的第 1 列，所以不能只保留当前读入行。
3. 从原矩阵最后一列开始枚举 `col`，作为输出矩阵的行。
4. 对每一列，从原矩阵第一行扫到最后一行，依次输出。
5. 控制每行内部空格，行末换行。

**易错点**

- 逆时针不是顺时针。逆时针输出第一行是原矩阵第 `m-1` 列，从上到下输出。
- 输出矩阵是 `m` 行 `n` 列，不是仍然 `n` 行 `m` 列。
- 每行数字之间有空格，但行尾不需要额外空格。
- `n,m` 可到 1000，最多 `10^6` 个数，二维 `vector<int>` 可以接受。

**复杂度**

读入和输出都要处理每个元素一次，时间复杂度 `O(nm)`，空间复杂度 `O(nm)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    vector<vector<int>> a(n, vector<int>(m));
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            cin >> a[i][j];
        }
    }

    for (int col = m - 1; col >= 0; col--) {
        for (int row = 0; row < n; row++) {
            if (row != 0) {
                cout << ' ';
            }
            cout << a[row][col];
        }
        cout << '\n';
    }

    return 0;
}
```

### 第05次 T1 数列分段

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给定一个整数数列，把连续相同的一段看成一个段，问一共有多少段。例如 `8 8 8 0 12 12 8 0` 有 5 段：`8 8 8`、`0`、`12 12`、`8`、`0`。

**知识点**

这是相邻比较题。一个新段开始的条件是：当前元素和前一个元素不同。第一个元素一定开启第一段。

**解法**

1. 读入 `n`。
2. 从左到右读入数列。
3. 如果是第一个数，答案加一。
4. 如果不是第一个数，并且当前数 `x` 不等于上一个数 `last`，说明新段开始，答案加一。
5. 更新 `last = x`。

这题不需要保存整个数组，只需要保留上一个数。

**易错点**

- 问的是连续段数量，不是不同数字种类数。`8 0 8` 是 3 段，不是 2 段。
- 第一个数一定贡献 1 段，不能漏掉。
- 可以用哨兵值初始化 `last`，但更稳的写法是用 `i == 0` 特判。

**复杂度**

时间复杂度 `O(n)`，只保留上一个数，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    int ans = 0;
    int last = -1;
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        if (i == 0 || x != last) {
            ans++;
        }
        last = x;
    }

    cout << ans << '\n';
    return 0;
}
```

### 第06次 T1 数位之和

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

给定一个十进制整数 `n`，输出它各位数字之和。比如 `20151220` 的答案是 `2+0+1+5+1+2+2+0=13`。

**知识点**

这题有两种常见写法：

- 把输入当作字符串，逐个字符累加 `ch - '0'`。
- 把输入当作整数，反复累加 `n % 10`，再执行 `n /= 10`。

字符串写法更不容易漏掉 `0` 这个输入。因为如果用整数拆位，`while (n > 0)` 对 `n = 0` 不会进入循环，需要额外特判；字符串 `"0"` 扫描后自然得到答案 0。

**解法**

1. 用 `string s` 读入整数。
2. 初始化 `ans = 0`。
3. 遍历字符串中每个字符 `ch`。
4. 数字字符转数值：`ch - '0'`。
5. 输出总和。

**易错点**

- 输入最大到 `1000000000`，用 `int` 也能存，但字符串更直接。
- `0` 的数位之和是 0。
- 字符转数字不能直接加 `ch`，要减去 `'0'`。

**复杂度**

设数字位数为 `L`，时间复杂度 `O(L)`，额外空间复杂度 `O(L)` 用于保存字符串；本题 `L` 很小。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    string s;
    cin >> s;

    int ans = 0;
    for (char ch : s) {
        ans += ch - '0';
    }

    cout << ans << '\n';
    return 0;
}
```
