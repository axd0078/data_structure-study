# Day 017：比例定位与几何计数

## 学习内容

Day 017 继续处理公式类 T1。**正态分布**不要求你真的查概率表，而是把标准化后的 `Z` 值定位到表格行列。**蒙特卡洛**也不要求生成随机数，只需要统计已经给出的点有多少落在圆内，再套估算公式。

正态分布题给出 `mu, sigma, n`，要算：

```text
z = (n - mu) / sigma
```

表格每一列步长是 `0.01`，因此把 `z` 放大 100 后得到百分位整数 `percent`。行由十分位决定，列由百分位个位决定：

```text
row = percent / 10 + 1
col = percent % 10 + 1
```

题目保证 `sigma` 是 100 的因子，所以 `(n-mu)*100/sigma` 是精确整数，直接用整数计算，避免浮点截断误差。

蒙特卡洛题只需判断点是否在圆内。圆心在原点、半径为 `a`，点 `(x,y)` 在圆内当且仅当：

```text
x*x + y*y <= a*a
```

最后输出 `4.0 * inside / n`。

## 题解

### 第38次 T1 正态分布

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

每个查询给出 `mu`、`sigma` 和 `n`，将正态分布变量转为标准正态分布：

```text
z = (n - mu) / sigma
```

题目要输出 `z` 在标准正态分布表里的行列位置。行从 `0.0` 开始编号为 1，列从 `0.00` 到 `0.09` 编号为 1 到 10。

**知识点**

把 `z` 乘以 100 后，整数 `percent` 的十位及以上决定行，个位决定列。例如：

- `z = 1.00`，`percent = 100`，行 `100/10+1=11`，列 `100%10+1=1`。
- `z = 3.45`，`percent = 345`，行 `34+1=35`，列 `5+1=6`。

由于 `sigma` 是 100 的因子，可以直接计算：

```text
percent = (n - mu) * 100 / sigma
```

**解法**

1. 读入查询数 `k`。
2. 每个查询读入 `mu,sigma,value`。
3. 计算 `percent = (value - mu) * 100 / sigma`。
4. 输出 `percent/10+1` 和 `percent%10+1`。

**易错点**

- 输入里的 `n` 是查询值，不是查询个数；代码里建议叫 `value`。
- 不要用 `double` 后强转整数，边界上可能因为精度误差少 1。
- 行列编号都从 1 开始。

**复杂度**

时间复杂度 `O(k)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int k;
    cin >> k;

    while (k--) {
        int mu, sigma, value;
        cin >> mu >> sigma >> value;

        int percent = (value - mu) * 100 / sigma;
        cout << percent / 10 + 1 << ' ' << percent % 10 + 1 << '\n';
    }

    return 0;
}
```

### 第39次 T1 蒙特卡洛

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

正方形区域内已经给出 `n` 个点，圆心为原点、半径为 `a`。统计有多少点在圆内或圆上，设数量为 `m`，输出：

```text
4 * m / n
```

这是题面给出的对 `pi` 的估计值。

**知识点**

点到原点距离不需要真的开平方。比较：

```text
sqrt(x*x + y*y) <= a
```

等价于：

```text
x*x + y*y <= a*a
```

这样既简单，也避免平方根带来的误差。

**解法**

1. 读入 `n,a`。
2. 初始化 `inside = 0`。
3. 对每个点 `(x,y)`：
   - 如果 `x*x + y*y <= a*a`，`inside++`。
4. 输出 `4.0 * inside / n`。

**易错点**

- 点在圆周上算圆内，所以使用 `<=`。
- 坐标可能是小数，使用 `double`。
- 最终结果是实数，不能做整数除法。
- 为抵抗浮点边界误差，可以在比较时加极小量 `1e-12`。

**复杂度**

时间复杂度 `O(n)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    double a;
    cin >> n >> a;

    int inside = 0;
    for (int i = 0; i < n; ++i) {
        double x, y;
        cin >> x >> y;
        if (x * x + y * y <= a * a + 1e-12) {
            ++inside;
        }
    }

    cout << fixed << setprecision(6) << 4.0 * inside / n << '\n';
    return 0;
}
```
