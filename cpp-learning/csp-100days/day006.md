# Day 006：边界格式与统计输出

## 学习内容

Day 006 的重点不是复杂算法，而是“输出格式”和“统计口径”。这类题经常代码很短，但扣分点集中在格式、并列规则、是否使用足够大的整数类型。

今天要掌握两类问题：

- **有序数据统计**：小中大给出的序列已经有序，可能升序也可能降序。最大值和最小值只在两端，中位数只在中间位置，不需要排序。
- **逐行汇总统计**：小明种苹果每棵树一行，先算这棵树剩余苹果数和疏果总数，再更新全局总量和最大疏果树编号。

今天的最低复杂度：

- 小中大：必须读入 `n` 个数，但只记录两端和中间位置，`O(n)` 时间，`O(1)` 额外空间。
- 小明种苹果：每个输入数必须处理一次，`O(NM)` 时间，`O(1)` 额外空间。

格式细节：

- 小中大的中位数如果是整数，不能输出 `.0`；如果是半整数，要输出 1 位小数。
- 小明种苹果如果多棵树疏果数量并列最大，输出编号最小的那棵，扫描时只在“严格更大”时更新即可。

## 题解

### 第16次 T1 小中大

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

输入 `n` 个已经有序的整数，序列可能升序，也可能降序。要求按“最大值 中位数 最小值”的顺序输出。

中位数规则：

- `n` 为奇数：中位数是中间那个数。
- `n` 为偶数：中位数是中间两个数的平均值。
- 如果中位数是整数，直接输出整数；如果不是整数，输出保留 1 位小数的结果。

**知识点**

题目已经保证有序，所以不能再排序浪费复杂度。最大值和最小值一定是首尾两个数中的较大者和较小者。中位数只需要中间一个或两个位置。

可以只记录这几个位置：

```text
leftMiddleIndex = (n + 1) / 2
rightMiddleIndex = (n + 2) / 2
```

这里使用 1-based 下标。`n` 为奇数时，两个下标相同；`n` 为偶数时，它们正好是中间两个数。

为了避免浮点误差，先计算 `twiceMedian = leftMiddle + rightMiddle`。如果它是偶数，中位数是整数；如果它是奇数，中位数就是 `.5`。

**解法**

1. 读入 `n`。
2. 计算两个中间位置。
3. 顺序读入所有数，只保存：
   - 第一个数 `first`
   - 最后一个数 `last`
   - 左中位数位置的值
   - 右中位数位置的值
4. `maximum = max(first, last)`，`minimum = min(first, last)`。
5. 根据 `twiceMedian` 的奇偶决定中位数输出格式。

**易错点**

- 序列可能是降序，不能默认第一个是最小值。
- 偶数长度时，中位数可能是 `.5`。
- 中位数为整数时不要输出 `1.0`。
- 数据绝对值可到 `10^7`，中间两个数相加仍安全，但用 `long long` 更稳。

**复杂度**

时间复杂度 `O(n)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    const int leftMiddleIndex = (n + 1) / 2;
    const int rightMiddleIndex = (n + 2) / 2;

    long long first = 0;
    long long last = 0;
    long long leftMiddle = 0;
    long long rightMiddle = 0;

    for (int i = 1; i <= n; i++) {
        long long value;
        cin >> value;
        if (i == 1) {
            first = value;
        }
        if (i == leftMiddleIndex) {
            leftMiddle = value;
        }
        if (i == rightMiddleIndex) {
            rightMiddle = value;
        }
        last = value;
    }

    long long maximum = max(first, last);
    long long minimum = min(first, last);
    long long twiceMedian = leftMiddle + rightMiddle;

    cout << maximum << ' ';
    if (twiceMedian % 2 == 0) {
        cout << twiceMedian / 2;
    } else {
        cout << fixed << setprecision(1) << twiceMedian / 2.0;
    }
    cout << ' ' << minimum << '\n';
    return 0;
}
```

### 第17次 T1 小明种苹果

- 原题链接：见上方题目表。
- 当前状态：题解已补齐。

**题意抓手**

有 `N` 棵苹果树，进行了 `M` 轮疏果。每棵树输入 `M+1` 个数：

- 第一个数 `a[i][0]` 是疏果前这棵树的苹果数。
- 后面 `M` 个数是每轮疏果变化，都是 `0` 或负数；负数的绝对值表示去掉的苹果数量。

要求输出三个数：

- `T`：所有树最后剩余苹果总数。
- `k`：疏果总数最多的树编号；如果并列，输出最小编号。
- `P`：第 `k` 棵树的疏果总数。

**知识点**

每一行可以独立统计：

```text
current = 初始苹果数
removed = 0
每个 delta:
    current += delta
    removed -= delta
```

因为 `delta` 是 `0` 或负数，所以 `-delta` 就是本轮去掉的苹果数。

扫描到一棵树后：

- 把 `current` 加到全局总剩余 `T`。
- 如果 `removed > bestRemoved`，更新最大疏果数量和树编号。
- 如果 `removed == bestRemoved`，不更新，因为题目要求并列时编号最小，先扫描到的编号更小。

**解法**

1. 读入 `N`、`M`。
2. 对第 `i` 棵树：
   - 读入初始苹果数。
   - 累加后面 `M` 个变化，得到剩余数和疏果总数。
3. 更新总剩余苹果数。
4. 用严格大于更新最大疏果树。
5. 输出 `T k P`。

**易错点**

- 疏果输入是负数，疏果数量要用 `-delta` 累加。
- 输出的 `P` 是去掉的苹果数量，不是剩余苹果数量。
- 并列最大时输出最小编号，所以不能在相等时更新。
- `N` 和 `M` 可到 1000，数值可到 `10^6`，总和建议用 `long long`。

**复杂度**

时间复杂度 `O(NM)`，额外空间复杂度 `O(1)`。

**参考代码**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    long long total = 0;
    int bestIndex = 1;
    long long bestRemoved = -1;

    for (int i = 1; i <= n; i++) {
        long long current;
        cin >> current;

        long long removed = 0;
        for (int j = 0; j < m; j++) {
            long long delta;
            cin >> delta;
            current += delta;
            removed -= delta;
        }

        total += current;
        if (removed > bestRemoved) {
            bestRemoved = removed;
            bestIndex = i;
        }
    }

    cout << total << ' ' << bestIndex << ' ' << bestRemoved << '\n';
    return 0;
}
```
