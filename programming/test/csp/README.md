# CSP 练习题实现方法总结

## 模块简介

`programming/test/csp` 按第 `1` 次到第 `41` 次 CSP 认证届次归档练习代码。当前目录同时保留了三类文件：

- 已经补有实现的练习入口。
- 已经补有实现的 T2 / T3 / T4 题目。
- 仍保留为空 `main` 的练习入口或占位骨架。

当前目录共有 `80` 个 `.cpp` 文件，其中 `37` 个文件已有实现，`43` 个文件仍为空 `main`。这和 `cpp-learning/csp-100days`、`cpp-learning/csp-test-env/reference_solutions` 的“题解 / 参考程序”状态并不完全相同：很多天的文档已经补齐，但这里的练习入口仍故意保留为空 `main`，用于自己重做。

## 文件结构

| 路径范围 | 说明 |
|------|------|
| `1/` ~ `14/` | 早期 T1 练习入口，其中 `1/1.cpp` ~ `14/1.cpp` 已有实现，`1/2.cpp` ~ `4/2.cpp` 仍为空 `main` |
| `15/` ~ `20/` | 第 15 到 20 次 T1 练习入口，其中 `15/1.cpp`、`16/1.cpp`、`18/1.cpp` 已有实现，其余仍为练习入口 |
| `21/` ~ `41/` | 后续各届 T1 / T2 / T3 / T4 归档；多个高题次文件已有实现，但绝大多数 T1 入口仍为空 `main` |
| `README.md` | 当前目录索引与状态说明 |

## 关键接口或算法说明

`9/1.cpp` 用值域较小这一条件建立 `1..1000` 的计数桶。顺序扫描每个候选值时，同时维护它左边和右边已有多少个元素，从而判断中间数是否存在。

`10/1.cpp` 只需要顺序累计当前朋友拿到的蛋糕总重量，达到阈值 `k` 就立即切到下一个朋友。最后即使剩余总量不足 `k`，也仍然要记成一位朋友。

`11/1.cpp` 当前实现不是搜索或 DP，而是直接按 `50 / 30 / 10` 元三档优惠做整除和取模，本质上是常数时间的公式题。

`12/1.cpp` 当前练习入口已经有代码，实现方式是“排序后扫相邻差值”。如果排序后发现重复值，答案立即可以压到 `0`。

`13/1.cpp` 通过 `times` 记录连续跳到中心的次数。遇到 `1` 时把连续状态清零，遇到 `2` 时按 `2 * (++times)` 计入本次得分。

`14/1.cpp` 把第一天价格存放在 `first` 中，把第二天结果写入 `second`，避免边计算边覆盖原数组。

`15/1.cpp` 直接按道路、红灯、黄灯、绿灯四类记录分类累加总时间，重点是明确第 15 次 T1 不需要根据当前总时间再反推灯色状态。

`16/1.cpp` 直接利用输入序列本身已经有序，只记录首尾和中间位置的值，不再额外排序；偶数长度时再决定是整数中位数还是带 `.5` 的输出格式。

`18/1.cpp` 用顺序模拟维护“轮到的自然数”和“已经成功报出的数量”两套状态，7 的倍数或十进制中包含 7 的数都要跳过。

`8/2.cpp` 先尝试在同一排内找连续空座，找不到再按全局座位编号从小到大零散分配，核心是二维座位状态和编号映射。

`9/2.cpp` 不是按税率区间手推逆函数，而是把税前工资到税后工资看成单调函数，先正向计算，再二分最小可行税前工资。

`10/2.cpp` 直接用 `vector` 承担顺序表功能，每次操作都按“找到学生当前位置 -> 删除 -> 按偏移插回”的流程模拟。

`22/2.cpp` 的关键是二维前缀和。判断平均值时直接比较 `sum` 和 `t * area`，可以避免先除法再比较带来的精度问题。

`31/2.cpp` 把一段连续操作压成“前缀缩放积 + 前缀旋转和”，查询时不需要逐条回放区间内的操作。

`35/2.cpp` 把字符替换规则视作函数复合，预处理 `2^bit` 次变换后的字符映射表，再按查询轮数的二进制位逐层跳转。

`41/3.cpp` 同时维护按左端点排序和按长度排序的空闲区间。删除队列时要做相邻区间合并，发送操作则要把“下一个位置”绕回到区间左端，形成循环队列。

## 注意事项

- `cpp-learning/csp-100days` 中很多天的题解已经补齐，但 `programming/test/csp` 下对应练习入口仍可能故意保留为空 `main`。
- 这类空 `main` 文件不代表题目未归档，只代表“练习入口保留给自己重做”。
- `38/3.cpp` 需要 `C++17` 编译，其余大多数文件按 `C++11` 即可。
- 本目录默认不保存本地 `.exe` 等编译产物，建议随编随删，避免污染源码目录。
