## 核心实现说明

### 线性统计类

`3/1.cpp`、`5/1.cpp`、`21/1.cpp`、`22/1.cpp`、`25/1.cpp` 和 `33/1.cpp` 都是顺序扫描驱动的实现。重点不在复杂数据结构，而在于统计口径是否一致，例如 `3/1.cpp` 要边读边输出，`5/1.cpp` 只在相邻值发生变化时计数，`33/1.cpp` 则要同时维护“总出现次数”和“不同归属数”两套结果。

### 区间与前缀和类

`23/1.cpp`、`29/1.cpp`、`36/2.cpp` 和 `39/2.cpp` 都依赖区间视角。`29/1.cpp` 通过 `max/min` 把矩形裁剪回合法区域，`36/2.cpp` 把答案拆成前缀最大值和后缀最大值两部分，`39/2.cpp` 进一步把大量阈值区间叠加转成差分统计。

### 模拟与状态维护类

`1/2.cpp`、`36/1.cpp`、`40/3.cpp` 和 `41/3.cpp` 重点在状态更新顺序。窗口置顶、网格移动、字符矩阵恢复、循环队列分配都不是单纯套算法模板，更需要先把当前状态、更新动作和输出时机梳理清楚。

### 搜索与分层推进类

`36/4.cpp` 和 `38/2.cpp` 都带有“按层扩展”的味道。`36/4.cpp` 用区间层维护当前步数能覆盖的最远范围，`38/2.cpp` 用 BFS 扩展马步状态，两者都要避免重复扫描已经处理过的位置。

### 编码与预处理类

`27/1.cpp`、`38/3.cpp` 和 `40/2.cpp` 都属于“先建好表示，再做还原或查询”的题。`27/1.cpp` 用各位权值的累乘数组还原混合进制每一位，`38/3.cpp` 按字段拆解 72 位消息并结合历史哈希表恢复代码串，`40/2.cpp` 则利用状态总数固定的特点提前整表预处理。

## 运行方式

建议在仓库根目录编译。

```bash
g++ -std=c++11 programming/test/csp/5/1.cpp -o test && ./test
g++ -std=c++11 programming/test/csp/27/1.cpp -o test && ./test
g++ -std=c++11 programming/test/csp/29/1.cpp -o test && ./test
g++ -std=c++11 programming/test/csp/33/1.cpp -o test && ./test
g++ -std=c++11 programming/test/csp/36/2.cpp -o test && ./test
g++ -std=c++17 programming/test/csp/38/3.cpp -o test && ./test
```

如果当前目录已经在 `programming/test` 下：

```bash
g++ -std=c++11 csp/21/1.cpp -o test && ./test
g++ -std=c++11 csp/41/3.cpp -o test && ./test
```

## 注意事项

- `9/1.cpp` 当前只有空的 `main` 骨架，这次文档仅把它记为待补文件，不当作已完成题解。
- `29/1.cpp` 计算相交面积时，宽和高都要先用 `max(0, ...)` 裁成非负值。
- `33/1.cpp` 中“出现于多少个集合”与“总共出现多少次”是两套不同统计，不能只用一个计数数组代替。
- `36/2.cpp` 这次已补全缺失括号并恢复到可编译状态，后续复盘时重点看前缀和与前后缀最大值如何拆开维护。
- `38/3.cpp` 需要 `C++17` 编译，且恢复摘要字段时依赖之前已经出现过的代码记录。
- `40/2.cpp` 适合固定状态空间、批量查询的题，重点是先做整表预处理而不是逐次重算。
- `41/3.cpp` 同时维护按左端点排序和按长度排序的空闲区间，删除队列时要注意相邻空闲段合并。
