# CSP 练习题实现方法总结

这个目录按 CSP 第几次认证保存练习代码，数字命名文件夹表示认证次数，例如 `38/3.cpp` 表示第 38 次认证的第 3 题。这里不只记录“用了什么算法”，更关注实现时怎么把题目描述落到 C++ 代码里：字段怎么拆、状态怎么存、更新顺序怎么保证、哪些地方容易写反。

## 实现型题的通用拆法

很多 CSP 题的难点不是算法名字，而是“把题意翻译成稳定的代码”。可以按下面顺序做。

### 1. 先画数据结构，不急着写循环

遇到位串、矩阵、窗口、队列、区间这类题，先把输入拆成字段。

例如 `38/3.cpp` 的简单消息：

```text
0 | 28 位接收方 | 28 位发送方 | 15 位位置
```

复杂消息：

```text
1 | 58 位完整代码 | 12 位散列值 | 1 位收发标志
```

字段确定以后，代码里就不要到处手写下标，封装一个取位函数：

```cpp
unsigned long long getBits(const string& s, int l, int len) {
    unsigned long long x = 0;
    for (int i = l; i < l + len; i++) {
        x = x * 2 + (s[i] - '0');
    }
    return x;
}
```

这样主流程会变成“取字段 -> 解字段 -> 更新状态”，不容易乱。

### 2. 编码题优先写一对正反函数

如果题目给了“怎么把对象编码成数字”，反解时不要硬猜，按原公式倒着写。

普通代码是 38 进制：

```cpp
sum = sum * 38 + 当前字符值;
```

逆过程就是从右往左：

```cpp
v = x % 38;
x /= 38;
```

典型代码是混合进制，原来最低三位是 `26, 26, 26`，再往前是 `10` 和 `36`，所以逆拆顺序就是：

```text
% 26, % 26, % 26, % 10, % 36, 剩余部分
```

这一类题最容易错在“进制顺序写反”。判断方法是看原公式里哪个字符乘的是 `26^0`，它就是第一个被 `%` 拆出来的。

### 3. 散列、摘要、校验值通常不能反推

`38/3.cpp` 里的散列值不能数学反解代码，只能用此前直接出现过的代码查表。

所以要维护历史表：

```cpp
unordered_map<int, string> hash12;
unordered_map<int, string> hash25;
```

每次遇到直接出现的代码，就计算它的 12 位和 25 位散列并覆盖旧值。覆盖是有意义的，因为题目要求散列冲突时使用最近收到的代码。

这类题的关键不是“解密”，而是“查历史”。看到哈希、摘要、校验、模数映射时，都要先问：题目是否保证可逆？如果没有，就应该建表、查表或维护候选集合。

### 4. 状态更新顺序要跟题意一致

实现题经常错在更新时机。

`38/3.cpp` 要先用旧历史表解当前消息，再把当前消息中直接出现的代码加入历史表：

```text
读消息
用当前已有历史解码
输出
把本条直接出现的代码加入历史
```

`1/2.cpp` 的窗口题也是类似：

```text
先按当前层级找命中窗口
输出窗口编号
再把它移动到顶层
```

如果先更新再判断，答案就会偏一轮。

### 5. 模拟题把“查找结构”和“顺序结构”分开

`41/3.cpp` 维护空闲区间时用了两套结构：

```cpp
map<long long, long long> freeByL;
set<Segment, SegmentCmp> freeByLen;
```

`map` 适合按左端点找相邻区间，方便释放时合并。

`set` 适合按长度找最短可用区间，方便分配。

同一个状态如果有两种查询方式，就不要强行只用一个容器。多维护一份索引，增删时同步更新，主逻辑会清楚很多。

### 6. 小状态空间先预处理

`40/2.cpp` 的状态只有 9 位，也就是 `0..511`。这种题不要每个输入都重复模拟完整过程，可以先把所有状态的答案算好：

```cpp
vector<int> ans(512);
for (int i = 0; i < 512; i++) {
    ans[i] = i;
    for (int j = m - 1; j >= 0; j--) {
        ans[i] = g(ans[i], k[j]);
    }
}
```

以后读到一个数，直接输出 `ans[num]`。

看到状态总数很小、操作次数可能很多时，优先考虑整表预处理。

### 7. 扫描大量区间时用差分

`39/2.cpp` 对每个模板位置得到一个合法阈值区间。如果每个区间都逐个枚举 `k`，复杂度会高。

差分写法是：

```cpp
diff[left]++;
diff[right + 1]--;
```

最后前缀和还原每个 `k` 被多少个区间覆盖。

看到“很多区间加一、最后问哪些点被覆盖”时，优先想到差分数组。

### 8. 矩阵变换题先写统一访问函数

`40/3.cpp` 用 `rot` 记录整体旋转状态，读写矩阵时都走：

```cpp
char getCell(int x, int y);
void setCell(int x, int y, char ch);
```

这样局部旋转、翻转、输出都可以写在“当前视角”下，不用在每个操作里重复考虑全局旋转。

矩阵题常见套路是：不要到处写坐标变换，集中到一个 `pos()` 函数里。

## `38/3.cpp` 的实现检查

当前主流程方向是正确的：

1. 简单消息用 `getBits(s, 1, 28)`、`getBits(s, 29, 28)`、`getBits(s, 57, 15)` 拆字段。
2. `value >= 2^25` 时按典型代码短表示解码，否则按 25 位散列查 `hash25`。
3. 复杂消息用 58 位数字直接 `decodeNormal`，另一个 12 位散列查 `hash12`。
4. 每条消息都是先输出，再把本条直接出现的代码 `addCode`。

有几个实现细节要记住：

```cpp
sum = sum * 38 + charToValue(c);
```

不要写成 `sum *= sum * 38 + ...`。

```cpp
for (int i = 10; i >= 0; i--)
```

普通代码数字反解时要从后往前填。

```cpp
while (!s.empty() && s.back() == ' ')
```

去掉的是尾部空格，条件不能写反。

散列函数用 `unsigned long long` 可以，因为无符号整数乘法自然保留低 64 位：

```cpp
unsigned long long product = num * 47055833459ULL;
return (product >> (64 - bits)) & ((1ULL << bits) - 1);
```

## 复盘清单

写完这类题后，可以按这个顺序自查：

1. 字段下标是否画清楚，左闭右闭还是左闭右开是否统一。
2. 编码和解码是否真的是互逆，最低位是不是先 `%` 出来。
3. `int` 会不会溢出，乘法、位移、平方是否需要 `long long`。
4. 查询历史和更新历史的顺序是否符合题目时间线。
5. 容器里的状态是否有多份索引，增删时有没有同步。
6. 散列冲突时是不是按题目要求覆盖或选择最近值。
7. 输出格式里的空格、换行、可省略字段是否处理完整。
