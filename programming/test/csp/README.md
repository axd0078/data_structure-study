## 核心实现说明

### 线性统计类

`3/1.cpp`、`5/1.cpp`、`21/1.cpp`、`22/1.cpp`、`24/1.cpp`、`25/1.cpp`、`30/1.cpp`、`33/1.cpp` 和 `35/1.cpp` 都是顺序扫描驱动的实现。重点不在复杂数据结构，而在于统计口径是否一致，例如 `3/1.cpp` 要边读边输出，`24/1.cpp` 要把整段区间的函数值一次性计入贡献，`30/1.cpp` 要先把棋盘压成稳定状态串再做映射计数，`33/1.cpp` 要同时维护“总出现次数”和“不同归属数”，`35/1.cpp` 则把字符种类统计和重复字符统计拆开处理。

### 区间与前缀和类

`22/2.cpp`、`23/1.cpp`、`24/2.cpp`、`29/1.cpp`、`31/1.cpp`、`31/2.cpp`、`36/2.cpp` 和 `39/2.cpp` 都依赖累计信息或区间视角。`22/2.cpp` 用二维前缀和快速求邻域总和，`24/2.cpp` 同时跟踪两套分段边界并按小区间累计误差，`29/1.cpp` 通过 `max/min` 把矩形裁剪回合法区域，`31/1.cpp` 先把全部平移量累加成总位移，`31/2.cpp` 再进一步把连续操作压成前缀缩放积与前缀旋转和，`36/2.cpp` 把答案拆成前缀最大值和后缀最大值两部分，`39/2.cpp` 则把大量阈值区间叠加转成差分统计。

### 模拟与状态维护类

`1/2.cpp`、`34/1.cpp`、`34/2.cpp`、`36/1.cpp`、`40/3.cpp` 和 `41/3.cpp` 重点在状态更新顺序或矩阵展开方式。窗口置顶、矩阵重排、矩阵转置、网格移动、字符矩阵恢复、循环队列分配都不是单纯套算法模板，更需要先把当前状态、维度变化、更新动作和输出时机梳理清楚。

### 搜索与分层推进类

`36/4.cpp` 和 `38/2.cpp` 都带有“按层扩展”的味道。`36/4.cpp` 用区间层维护当前步数能覆盖的最远范围，`38/2.cpp` 用 BFS 扩展马步状态，两者都要避免重复扫描已经处理过的位置。

### 依赖关系与时序推导类

`28/2.cpp` 关注的是“依赖先后顺序如何限制课程的最早和最晚开始时间”。当前写法先顺推每门课程最早何时能做，再在整体可行时逆序回推最晚起点，属于典型的正反两遍时序约束传播。

### 编码与预处理类

`27/1.cpp`、`28/1.cpp`、`29/2.cpp`、`33/2.cpp`、`35/2.cpp`、`38/3.cpp` 和 `40/2.cpp` 都属于“先整理表示或公式，再做计算”的题。`27/1.cpp` 用各位权值的累乘数组还原混合进制每一位，`28/1.cpp` 直接按折现公式累计现值，`29/2.cpp` 通过排序后配合二分检查目标天数是否可行，`33/2.cpp` 先统一大小写再做集合交并统计，`35/2.cpp` 则把单字符映射预处理成倍增跳表，用于快速处理多轮字符串变换。

### 骨架占位类

`9/1.cpp`、`34/3.cpp`、`37/3.cpp` 和 `39/3.cpp` 当前都只保留了最基础的 `main` 模板。这些文件已经完成题号归档，但还没有进入具体建模和实现阶段，适合后续按题面继续补题时直接沿用现有路径。

## 注意事项

- `9/1.cpp` 当前只有空的 `main` 骨架，这次文档仅把它记为待补文件，不当作已完成题解。
- `22/2.cpp` 关键是用二维前缀和做邻域求和，判断平均值时直接比较 `sum` 与 `t * area` 更稳妥。
- `24/1.cpp` 这题不要枚举每个 `x`，按区间 `[A[i], A[i+1])` 一次性累计贡献才是正解。
- `24/2.cpp` 要同时维护 `f(x)` 和 `g(x)` 的分段边界，按更近的断点切出小区间，才能把复杂度压下来。
- `28/1.cpp` 的关键是每一项现金流都按正确指数折现，浮点输出建议固定格式。
- `28/2.cpp` 如果最早开始时间已经导致某门课超出总天数，就只输出第一行最早时间，不再继续回推最晚时间。
- `29/1.cpp` 计算相交面积时，宽和高都要先用 `max(0, ...)` 裁成非负值。
- `29/2.cpp` 当前写法把任务按时间排序后配合 `upper_bound` 定位超期部分，二分检查时要重点看可行性函数是否单调。
- `30/1.cpp` 把 `8 x 8` 棋盘压成固定长度状态串后再统计，可以避免直接比较二维结构带来的代码噪声。
- `31/1.cpp` 只有平移操作时不需要保存整段历史，直接累计总位移向量即可把单次查询降到 `O(1)`。
- `31/2.cpp` 查询区间时直接用前缀缩放积和前缀角度差恢复变换，避免逐条操作重放。
- `33/1.cpp` 中“出现于多少个集合”与“总共出现多少次”是两套不同统计，不能只用一个计数数组代替。
- `33/2.cpp` 当前实现先统一转小写再做交并集，复盘时重点看“大小写归一 + 去重统计”的处理顺序。
- `34/1.cpp` 这类矩阵重排题先压平再输出通常更稳定，复盘时重点看一维下标如何连续推进。
- `34/2.cpp` 当前转置实现会完整重建一次矩阵，之后还要立刻交换 `n`、`m`，否则查询下标会基于旧维度出错。
- `34/3.cpp` 当前仍是待补骨架，文档中按“已建档、待实现”处理。
- `35/1.cpp` 当前写法在某个字符第三次出现时才标记重复条件成立，复盘时要注意频次判断是在自增前还是自增后触发。
- `35/2.cpp` 的重点是把字符映射做成 `2^bit` 次跳转表，避免对每次查询真的循环执行 `k` 轮替换。
- `36/2.cpp` 这次已补全缺失括号并恢复到可编译状态，后续复盘时重点看前缀和与前后缀最大值如何拆开维护。
- `37/3.cpp` 当前同样只有骨架，后续补题时建议连同题意和运行示例一起补齐。
- `38/3.cpp` 需要 `C++17` 编译，且恢复摘要字段时依赖之前已经出现过的代码记录。
- `39/3.cpp` 目前也只是骨架占位，不应误解为已完成的 HTTP 头信息题解。
- `40/2.cpp` 适合固定状态空间、批量查询的题，重点是先做整表预处理而不是逐次重算。
- `41/3.cpp` 同时维护按左端点排序和按长度排序的空闲区间，删除队列时要注意相邻空闲段合并。
